## Why

当前 `ProjectDetailPage.vue` 已承载项目基础信息、8 阶段状态、阶段资料清单、资料状态/适用性/责任人操作、阶段推进和业务日志等多个区域。后续如果继续在单文件中叠加文件联动、在线表单和权限相关能力，页面状态依赖和维护成本会继续上升。

本 change 通过行为保持型前端结构拆分，将项目详情页拆成更清晰的页面模块，为后续复杂能力预留可维护结构，同时不新增业务能力、不改变后端接口、不改变用户可见业务结果。

## What Changes

- 拆分 `digital-platform-web/src/pages/ProjectDetailPage.vue` 中的主要展示和操作区域。
- 建议拆出项目基础信息/标题区域、阶段资料清单、资料项操作子区域、阶段推进和业务日志等组件。
- 可按最小范围抽取资料责任人选择/展示、状态文案、追溯展示或格式化 helper。
- 保持现有页面路由、API 调用、刷新逻辑、错误提示、边界说明文案和用户可见业务结果不变。
- 不修改后端业务代码、不修改数据库迁移、不改变任何业务状态机、接口字段、错误码或权限规则。
- 不新增文件上传/下载、文件管理平台联动、在线表单、消息提醒、超期提醒、项目成员权限、复杂权限或其他业务能力。
- 更新 Web README，说明项目详情页已模块化但业务能力保持不变。

## Capabilities

### New Capabilities

无。本变更不引入新的业务 capability，只对既有前端项目详情页做结构性重构。

### Modified Capabilities

- `project-core-frontend`: 增加项目详情页模块化拆分的实现约束和行为保持要求，确保拆分后项目详情页现有展示、交互、API 调用和边界说明不回退。

## Impact

影响范围限定在 `digital-platform-web` 前端项目详情页相关 Vue 组件、少量前端 helper、样式和 Web README。后端 API、数据库迁移、OpenSpec 总路线图 `define-digital-platform-v1`、项目/阶段/资料状态机、业务日志规则、权限规则均不受影响。
