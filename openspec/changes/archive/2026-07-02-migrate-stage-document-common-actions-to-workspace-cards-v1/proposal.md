## Why

项目工作区 shell 已经建立了上方蓝色模块和产出卡片，但现有 64 项项目资料的通用上传、提交、审核、退回等能力仍集中在下方旧资料清单，导致未来主操作入口和当前执行入口分离。

本 change 规划把旧资料清单里的通用功能迁移到上方项目工作区产出卡片，使产出卡片成为后续资料工作的主入口；下方旧资料清单先降级为兼容区，迁移稳定后再清理，最后才进入 v20260629 71 项模板切换评估。

## What Changes

- 规划上方项目工作区产出卡片承载现有 20260625 64 项旧资料清单的通用操作和内容。
- 第一批迁移能力包括责任人展示、责任人分配/清空、附件列表、附件上传、附件下载、附件删除、提交资料、审核通过、审核退回、返工重提/返工完成、标记不适用、恢复适用，以及阻塞原因、权限原因、状态说明展示。
- 下方旧资料清单从主操作入口降级为兼容区；本 change 不隐藏、不物理删除旧资料清单组件，隐藏或删除必须通过后续独立 change 处理。
- `1.1 / 1.2 / 1.3` 立项阶段继续保持现有在线表单和 `1.2` 专用评价审批，不回退为普通文件上传，不新增普通提交入口绕过在线表单。
- 当前阶段不是切换 v20260629 71 项模板，不是旧项目迁移，不是新增复杂流程引擎。
- 当前阶段不新增资料表结构，不写 migration，不把新增 71 项候选落库，不切换新项目默认模板。

## Capabilities

### New Capabilities

- 无。

### Modified Capabilities

- `project-core-frontend`: 产出卡片 MUST 承载旧资料清单通用操作，并成为现有项目资料通用操作的主入口。
- `stage-document-checklist`: 旧资料清单 MUST 降级为兼容区，不能和上方产出卡片形成两个主入口。
- `project-core`: 产出卡片触发的资料操作 MUST 使用同一套后端状态、权限、业务日志和阶段推进判断。
- `technical-architecture`: 技术边界 MUST 禁止第二套状态机、禁止复杂流程引擎、禁止 v20260629 71 项模板切换、禁止数据库 migration。

## Impact

- 影响后续实现中的项目详情页、项目工作区产出卡片、旧资料清单组件、阶段资料相关 API 调用和权限展示。
- 后续实现应复用现有阶段资料接口、权限字段、状态字段、附件接口、业务日志和阶段推进判断。
- 本规划 change 不改业务代码，不改数据库，不写 migration，不提交旧项目迁移，不切换新项目默认模板，不归档或处理其他 active change。
