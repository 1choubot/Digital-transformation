## Why

当前项目详情仍以 64 项阶段资料清单为主，业务人员需要在资料列表中理解流程位置、产出、责任和阻塞原因，阅读成本偏高。领导已明确希望项目详情按“8 阶段 -> 流程图蓝色节点 -> 对应产出”的方式组织，因此本 change 的目标是重塑项目详情信息架构，而不是单纯做 UI 美化。

本 change 是全局项目详情信息架构调整 + 立项阶段第一批完整落地。全局部分只建立 8 阶段/蓝色节点工作区骨架；第一版只要求立项阶段节点、在线表单和业务规则完整实现。其他阶段暂不要求补齐完整蓝色节点映射和在线表单。

同时，立项阶段业务口径已经变化：项目创建需要轻量化，`1.1`、`1.2`、`1.3` 需要转为在线表单产出，且 `1.2 项目立项审批表` 需要从旧三审批节点调整为“两个评价 + 一个最终审批”模型。

## What Changes

- 建立“项目工作区 / 阶段节点视图”规划：项目列表和项目总览仍用于选择项目，进入单个项目后，项目详情变为项目工作区。
- 项目工作区左侧建立 8 个阶段导航骨架，右侧展示被选中节点的节点工作区。
- 节点工作区第一屏展示节点状态、关联产出、责任人、阻塞原因和可操作按钮；填写表单通过“填写/编辑”按钮进入，不直接把编辑表单作为第一屏。
- 蓝色节点只作为导航和业务语境入口，不单独拥有完成状态；节点状态必须从关联产出、在线表单、评价/审批记录和精准返工状态派生。
- 第一版只完整落地立项阶段节点：项目输入、项目市场调研、项目立项审批、项目立项通知。
- 立项阶段 `1.1 项目需求表`、`1.2 项目立项审批表`、`1.3 项目立项通知` 均规划为在线表单产出，并分别映射到现有模板文件。
- 其他 7 个阶段第一版可以展示阶段级占位、旧资料清单入口或后续待配置状态；后续阶段节点和产出映射通过后续 change 逐阶段补齐。
- 项目创建字段轻量化：创建时只要求项目名称、客户、客户联系方式；项目经理、项目模式、参与中心、项目编号、计划时间和立项日期不再作为创建必填。
- `1.1`、`1.2` 责任人由营销中心负责人分配，分配给谁就由谁填写/提交；`1.3` 默认由营销中心负责人处理。
- `1.2` 从旧的商务审批、技术审批、总经理审批三节点审批模型调整为营销评价、研发评价、总经理最终审批。
- `1.3` 增加前置门禁：必须在 `1.2` 总经理审批通过后才能填写或提交。
- 保留 64 项资料底座、资料责任人、状态、`completionMode`、精准返工、阶段推进、项目编号门禁、工作台、业务日志和权限体系。

## Capabilities

### New Capabilities

- None. 本 change 在既有项目核心、前端、阶段资料、业务日志和技术架构能力上调整需求，不新增独立 capability。

### Modified Capabilities

- `project-core`: 调整项目创建必填字段、项目详情工作区组织方式、`1.2` 评价/审批模型、`1.3` 前置门禁和第二阶段补录边界。
- `project-core-frontend`: 调整新建项目页、项目详情页信息架构、立项阶段节点工作区、`1.2` 评价/审批展示和 `1.3` 门禁展示。
- `stage-document-checklist`: 调整立项阶段 `1.1`、`1.2`、`1.3` 的在线表单完成方式、责任人规则、`1.2` 特殊完成规则和退回返工规则。
- `business-operation-log`: 补充轻量创建、在线表单、评价、审批、退回返工和 `1.3` 提交等日志动作规划，并保持节点视图不放宽权限。
- `technical-architecture`: 增加阶段节点视图和在线表单架构规划，调整 `1.2` 从 approval nodes 到 evaluation + final approval 的架构口径，并说明兼容策略。

## Impact

- 后续实现会影响 `digital-platform-api` 的项目创建校验、项目详情聚合/节点视图接口、阶段资料清单、在线表单、`1.2` 状态机、阶段推进/项目编号门禁和业务日志。
- 后续实现会影响 `digital-platform-web` 的新建项目页、项目详情页、立项阶段节点工作区、在线表单入口、评价/审批入口和门禁提示。
- 后续可能需要新增在线表单 schema/data/submission 存储能力，并需要在实现前设计旧三节点审批数据的迁移或解释方式。
- 本 change 不实现业务代码、不修改数据库或 migration、不运行测试库写入、不处理 `file-platform-integration-v1`、不处理 `define-digital-platform-v1`、不修改 `D:\file-server-local`。

## Out of Scope

- 不实现其他阶段的完整节点表单。
- 不一次性补齐其他 7 个阶段的全部蓝色节点映射。
- 不要求其他阶段产出在本 change 全部在线表单化。
- 不做日报、周报或自动汇总日报。
- 不恢复文件管理平台联动。
- 不做前端 UI 美化重构之外的视觉设计。
- 不处理 `define-digital-platform-v1`。
- 不做账号管理或用户管理。
- 不做通知推送、站内信、短信或邮件。
- 不物理删除旧字段，除非后续实现明确需要并另行规划。
