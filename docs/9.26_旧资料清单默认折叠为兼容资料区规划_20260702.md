# 9.26 旧资料清单默认折叠为兼容资料区规划

日期：2026-07-02

## 1. 背景

`map-legacy-contract-review-documents-to-workspace-cards-v1` 完成后，当前运行 20260625 64 项资料已全部由上方项目工作区 workspace card 覆盖，覆盖率为 64/64。下方旧资料清单已经不再承担资料主操作入口职责，但仍需要保留为旧模板兼容查看区和定位辅助区。

## 2. 当前目标

本 change `collapse-legacy-stage-document-checklist-to-compatibility-section-v1` 的目标是将下方旧资料清单从页面主内容降级为默认折叠的兼容资料区。

实现后：

- 上方项目工作区仍是唯一主操作区。
- 下方区域标题使用“兼容资料区”或“旧资料兼容区”等兼容语义。
- 下方区域默认折叠，不再进入页面后直接铺开旧资料清单。
- 折叠状态显示简短摘要，说明当前资料已迁移到上方项目工作区处理，本区域仅用于旧模板兼容查看。
- 用户仍可展开兼容资料区查看旧资料清单内容。
- 展开后仍保留只读状态、附件摘要、阻塞原因、兼容提示和“到上方产出卡片处理”的定位入口。

## 3. 非目标

本 change 明确不做：

- 不物理删除旧资料清单组件。
- 不完全隐藏旧资料清单入口。
- 不切换 v20260629 71 项模板。
- 不迁移旧项目，不补初始化旧项目。
- 不改数据库 schema，不写 migration。
- 不新增后端接口、第二套状态机、流程引擎或资料执行规则。
- 不恢复旧资料清单中已迁移资料的主操作按钮。

## 4. 验收口径

- 项目详情页默认不直接铺开下方旧资料清单。
- 兼容资料区可展开和收起。
- 展开后旧资料清单仍可访问。
- 已迁移资料仍不显示旧清单主操作按钮。
- “到上方产出卡片处理”的定位入口继续可用。
- `1.1 / 1.2 / 1.3` 在线表单和 `1.2` 专用评价审批不受影响。
- `3.3 / 5.4` 已在上方兼容卡片处理，旧清单展开后也只作为兼容区展示。
- 桌面和移动布局不重叠、不溢出。

## 5. 边界

本 change 只改前端项目详情页和旧资料清单展示方式，不改后端 API、不改数据库、不改变当前 64 项运行模板和 v20260629 71 项目标模板边界。
