# 智能制造项目整体推进流程（20260624）

版本：v0.1 草案  
来源：`智能制造项目管理流程图20260624.pdf` 及 `.tmp_pdf_review_20260624/flow_20260624_tile_01_0_1540.png` 至 `flow_20260624_tile_08_9823_11364.png`  
整理日期：2026-06-24  
适用范围：20260624 版项目流程理解、资料归属整理、后续 OpenSpec 规划和权限模型讨论

## 1. 编制说明

20260624 版流程图仍采用 8 个阶段：

1. 立项阶段
2. 方案设计阶段
3. 合同签订阶段
4. 详细设计阶段
5. 生产制作阶段
6. 预验收阶段
7. 终验收阶段
8. 结题阶段

与 20260610 版相比，新版不是重排阶段主干，而是把阶段内部的业务节点拆得更细，尤其补充了报价、合同审核、发票、采购申请、采购合同审核、设计变更、随机资料移交和资料服务器核查等节点。后续建模时必须区分“资料产出”和“业务节点”，不能把付款、发票、合同审核、采购审核、设计变更和资料核查都当成同一种普通阶段资料。

流程图横向泳道为：

- 阶段
- 流程-营销中心
- 流程-研发中心
- 流程-制造中心
- 流程-运营中心
- 过程文件或产出
- checklist/备注

红色波浪形块表示过程文件或产出；浅色波浪形块表示非必需或按客户要求的产出；绿色流程块表示中心或角色执行的动作；YES/NO 表示评审、审批、验收或审核的通过/回退。

## 2. 总体主线

```text
项目输入
  -> 项目市场调研
  -> 项目立项审批
  -> 项目立项通知
  -> 方案设计准备
  -> 项目方案分析
  -> 方案设计
  -> 内部方案评审
  -> 客户方案评审
  -> 成本估算/价格估算
  -> 报价
  -> 技术协议
  -> 销售合同
  -> 合同审核
  -> 预付款发票与付款
  -> 详细设计阶段启动会
  -> 详细设计
  -> 内部/客户设计评审
  -> 图纸审查与客户会签
  -> 生产制作阶段采购申请
  -> 采购准备、采购合同与采购合同审核
  -> 供应商生产、来料检验、入库、领料
  -> 厂内安装调试、设计变更、自验收
  -> 预验收、发货款发票
  -> 发货、现场安装调试、终验收
  -> 随机资料移交
  -> 产品培训
  -> 尾款发票与付款、资料整理归集与资料服务器核查
  -> 项目结题
```

## 3. 阶段流程

### 3.1 立项阶段

| 流程节点 | 责任中心 | 关键动作 | YES/NO 回退 | 过程文件或产出 | checklist/备注 |
|---|---|---|---|---|---|
| 项目输入 | 营销中心 | 营销人员获得市场信息后做项目输入，启动项目流程 | 无 | 无 | 项目输入是流程启动点，不是红色产出。 |
| 项目市场调研 | 营销中心组织，研发中心参与 | 组织了解客户需求，填写项目需求表 | 后续立项审批未通过时回到调研/需求完善 | 项目需求表 | 需求调研由营销人员组织沟通；研发中心负责填写生成《项目需求表》；需明确项目基本信息、工作环境条件、现工艺、期望目标。 |
| 项目立项审批 | 营销中心发起，商务/技术/总经理审批 | 填写项目立项审批表，进行商务评价审批、技术评价审批、总经理审批 | NO 回到前序需求完善；YES 进入立项通知 | 项目立项审批表 | 《项目立项审批表》由营销人员填写；审批签字完成后扫描上传电子档；《项目需求表》作为审批附件。 |
| 项目立项通知 | 营销中心 | 编写项目立项通知 | 无 | 项目立项通知 | 《项目立项通知》由营销总监编写，并发布至公司工作总群。 |

### 3.2 方案设计阶段

| 流程节点 | 责任中心 | 关键动作 | YES/NO 回退 | 过程文件或产出 | checklist/备注 |
|---|---|---|---|---|---|
| 方案设计准备 | 研发中心/项目经理 | 指定项目组成员、建立项目工作群、编写方案设计阶段工作计划 | 无 | 方案设计工作计划 | 项目组成员由研发总监指定；项目经理是项目第一责任人并拥有项目管理最大权利；项目经理和技术负责人可以是同一人；项目工作群由项目经理建立，群名为项目编号 + 项目名称。 |
| 项目方案分析 | 研发中心 | 填写项目方案分析表、编写产品功能框图、结构 3D 建模、绘制布局图、绘制工艺时序图 | 无 | 项目方案分析表、产品功能框图、3D模型、布局图、工艺时序图 | 项目经理组织方案实现分析，必要时组织客户人员参加；《项目方案分析表》和《产品功能框图》由技术负责人编写；工艺时序图为浅色非必需，按客户要求。 |
| 方案设计 | 研发中心 | 绘制节拍表、制作演示动画、编写电气功能框图、编写软件功能框图、编写项目方案PPT | 无 | 节拍表、演示动画、电气功能框图、软件功能框图、项目方案PPT | 节拍表、演示动画为浅色非必需；《电气功能框图》由电气工程师编写；《软件功能框图》由软件工程师编写；《项目方案PPT》由技术负责人编写。 |
| 内部方案评审 | 项目经理组织，研发中心记录 | 组织内部方案评审，填写方案评审记录表 | NO 回到方案设计；YES 进入客户方案评审 | 方案评审记录表 | 内部方案评审由项目经理按项目流程组织；评审次数视方案设计情况不固定，直到评审合格；《方案评审记录表》由技术负责人编写。 |
| 客户方案评审 | 营销中心组织，研发中心记录 | 组织客户方案评审，填写方案评审记录表 | NO 回到方案设计或方案调整；YES 进入成本估算 | 方案评审记录表 | 客户方案评审由营销人员组织；《方案评审记录表》由技术负责人编写。 |
| 成本估算/价格估算 | 研发中心、制造中心、运营中心 | 研发填写物料明细和研发费用，制造填写物料价格和安装调试运输费用，运营填写管理费用、税费和利润 | 总经理审批 NO 时回到估算调整；YES 进入报价 | 成本估算表 | 研发中心各专业组成员编写本人部分成本估算零部件清单；技术负责人汇总并填写研发费用后发送制造中心；制造中心核价并补充安装调试运输费用；运营中心补充管理费用、税费和利润后交总经理批准。 |
| 报价 | 营销中心 | 编制并发送标准格式报价单 | 客户或总经理不通过时回到报价/估算调整；YES 进入合同签订 | 报价单 | 《报价单》由营销人员编写，经总经理批准后，由营销人员发送至客户。 |

### 3.3 合同签订阶段

| 流程节点 | 责任中心 | 关键动作 | YES/NO 回退 | 过程文件或产出 | checklist/备注 |
|---|---|---|---|---|---|
| 编写技术协议 | 研发中心 | 编写技术协议 | 无 | 技术协议 | 《技术协议》由技术负责人编写。 |
| 签订技术协议 | 营销中心组织，研发中心配合 | 组织和客户签订技术协议 | NO 回到完善技术协议；YES 进入销售合同 | 技术协议 | 双方签字盖章后，技术协议作为销售合同的一部分；完善《技术协议》后再签销售合同。 |
| 拟订销售合同 | 营销中心 | 编写销售合同 | 无 | 销售合同 | 《销售合同》由营销人员编写。 |
| 销售合同审核 | 运营中心/合同审核相关人员 | 填写合同审核记录表 | NO 回到拟订销售合同；YES 进入签订销售合同 | 合同审核记录表 | 这是合同审核业务节点，不宜只作为普通资料项处理；需后续确认审核中心、审核人和是否形成独立模板。 |
| 签订销售合同 | 营销中心 | 和客户签订销售合同 | NO 回到合同修改；YES 进入预付款 | 销售合同 | 签订后的销售合同是阶段关键产出。 |
| 项目预付款交付 | 营销中心、运营中心/财务 | 催促客户支付项目预付款，开具预付款发票 | 付款未完成时停留在本节点 | 发票 | 由财务人员开具发票，营销人员协调客户付款；发票属于财务节点产出，不应简单混入普通技术资料清单。 |

### 3.4 详细设计阶段

| 流程节点 | 责任中心 | 关键动作 | YES/NO 回退 | 过程文件或产出 | checklist/备注 |
|---|---|---|---|---|---|
| 召开项目启动会 | 制造中心 | 组织召开项目启动会，编写项目启动书 | 无 | 项目启动书 | review 复核后，项目启动书归入详细设计阶段资料，作为详细设计阶段启动资料。 |
| 制定详细设计阶段工作计划 | 研发中心/项目经理 | 指定项目组成员、建立项目工作群、编写详细设计阶段工作计划 | 无 | 详细设计工作计划 | 项目组成员由研发总监指定；项目经理为第一责任人；项目经理和技术负责人可以是同一人；项目工作群沿用方案设计阶段建立的群。 |
| 详细设计 | 研发中心 | 结构 3D 建模，绘制电气原理图、电气接线图、电气布置图，编写控制逻辑流程图、自动化程序、软件开发说明文档、UI界面设计PPT、软件代码 | 无 | 3D模型、电气原理图、电气接线图、电气布置图、控制逻辑流程图、自动化程序、软件开发说明文档、UI界面设计PPT、软件代码 | 与 20260610 版详细设计资料项基本一致。 |
| 内部设计评审 | 项目经理组织，研发中心记录 | 填写设计评审记录表 | NO 回到详细设计；YES 进入客户设计评审 | 设计评审记录表 | 内部设计评审由项目经理按工作计划组织；评审次数视详细设计情况不固定，直到评审合格；《设计评审记录表》由技术负责人编写。 |
| 客户设计评审 | 营销中心组织，研发中心记录 | 组织客户设计评审，填写设计评审记录表 | NO 回到详细设计；YES 进入产品平面图/零部件清单 | 设计评审记录表 | 客户设计评审由营销人员组织；客户评审次数视详细设计情况不固定，直到评审合格；《设计评审记录表》由技术负责人编写。 |
| 绘制产品平面图 | 研发中心 | 绘制产品平面图 | 无 | 产品平面图 | 作为客户设计评审后的设计资料。 |
| 编写零部件清单 | 研发中心 | 编写产品零部件清单 | 无 | 产品零部件清单 | 各专业组成员编写本人部分零部件清单；《产品零部件清单》由技术负责人汇总。 |
| 内部图纸审查 | 项目经理组织，图纸审查人员执行 | 填写图纸审查记录表 | NO 回到设计修改；YES 进入客户图纸会签 | 图纸审查记录 | 图纸审查由项目经理按项目流程组织；图纸审查不能代替图纸签字流程；图纸审查人员由研发总监指定；图纸审查结束后图纸必须签字。 |
| 客户图纸会签 | 营销中心组织，项目经理归档 | 组织客户进行图纸会签 | NO 回到设计修改；YES 进入采购申请 | 客户会签记录 | 客户图纸会签由营销人员组织；《客户会签记录》由项目经理扫描存档。 |

### 3.5 生产制作阶段

| 流程节点 | 责任中心 | 关键动作 | YES/NO 回退 | 过程文件或产出 | checklist/备注 |
|---|---|---|---|---|---|
| 采购申请 | 研发中心发起，制造中心接收 | 提交采购申请流程 | YES 进入采购准备 | 采购申请表 | review 复核后，采购申请表归入生产制作阶段资料；技术负责人将各专业组图纸和零部件清单汇总，形成完整图纸，通过钉钉采购流程下发至制造中心。 |
| 采购准备 | 制造中心 | 寻找供应商，比价 | 无 | 比价表 | 《比价表》由采购人员编写。 |
| 拟订采购合同 | 制造中心 | 编写采购合同 | 无 | 采购合同 | 《采购合同》由采购人员编写。 |
| 采购合同审核 | 运营中心/合同审核相关人员 | 填写合同审核记录表 | NO 回到拟订采购合同；YES 进入签订采购合同 | 采购合同审核记录表 | 图中输出名为“合同审核记录表”，为避免与销售合同审核混淆，文档暂记为“采购合同审核记录表”，需业务确认。 |
| 签订采购合同 | 制造中心 | 签订采购合同 | NO 回到采购合同修改；YES 进入生产制作 | 采购合同 | 签订节点不应重复生成普通资料项，可作为采购合同状态或附件版本处理。 |
| 生产制作 | 制造中心 | 监督供应商生产制作 | 无 | 无 | 这是制造过程节点，不是红色资料产出。 |
| 准备随机资料 | 研发中心 | 编写产品装配作业指导书、产品使用说明书、产品维护保养手册、产品培训PPT | 无 | 作业指导书、产品使用说明书、产品维护保养手册、产品培训PPT | 这些资料在 20260624 版位于生产制作阶段，较 20260610 版前移；均由技术负责人编写。 |
| 来料检验 | 制造中心/质检 | 填写检验单 | NO 回到采购/供应商处理；YES 进入材料入库 | 检验单 | 《检验单》由质检中心或制造中心检验人员编写，需确认组织口径。 |
| 材料入库 | 制造中心/仓库 | 组织供应商材料入库，填写入库单 | 无 | 入库单 | 由采购人员组织材料入库，《入库单》由库管人员编写。 |
| 领料 | 制造中心/仓库 | 组织领料，填写领料单 | 无 | 领料单 | 由制造中心组织领料，《领料单》由库管人员编写。 |
| 厂内安装调试 | 制造中心 | 编写安装调试记录 | NO 进入设计变更或整改；YES 进入自验收 | 安装调试记录 | 《安装调试记录》由制造中心人员编写。 |
| 设计变更 | 研发中心 | 整改方案 3D 建模、绘制产品平面图、编写零部件清单、填写技术通知单 | 变更后回到厂内安装调试或自验收 | 3D模型、产品平面图、零部件清单、技术通知单 | 设计变更不是单一普通资料项，可能涉及一组受控资料和变更审批；《零部件清单》和《技术通知单》由设计变更人员编写。 |
| 自验收 | 制造中心 | 编写自验收报告 | NO 回到厂内安装调试或设计变更；YES 进入预验收 | 自验收报告 | 《自验收报告》由制造中心人员编写。 |

### 3.6 预验收阶段

| 流程节点 | 责任中心 | 关键动作 | YES/NO 回退 | 过程文件或产出 | checklist/备注 |
|---|---|---|---|---|---|
| 预验收 | 营销中心组织，制造中心记录 | 组织客户工厂预验收，编写预验收单 | NO 回到整改/安装调试；YES 进入发货款支付 | 预验收单 | 产品预验收由营销中心按工作计划组织；《预验收单》由制造中心人员编写。 |
| 项目发货款交付 | 营销中心、运营中心/财务 | 催促客户支付项目发货款，开具发货款发票 | 付款未完成时停留在本节点 | 发票 | 由财务人员开具发票，营销人员协调客户付款。 |

### 3.7 终验收阶段

| 流程节点 | 责任中心 | 关键动作 | YES/NO 回退 | 过程文件或产出 | checklist/备注 |
|---|---|---|---|---|---|

## 4. 建模注意事项

1. 20260624 版仍保留 8 阶段，不应为了新增节点直接扩展阶段数量。
2. 资料清单不能继续直接套用 v20260610 的 54 项模板；新版中有新增、改名、阶段移动和责任中心变化。
3. 报价单、合同审核记录表、采购申请表、采购合同审核记录表、发票不是普通技术资料，应在后续模型中区分财务、采购和审批类节点。
4. 设计变更产生的是一组受控资料和变更动作，不应只映射为一个普通“技术通知单”资料项。
5. 随机资料移交和资料服务器核查在流程中非常关键，但图中不是红色普通文件产出，是否落成资料项或检查单需要业务确认。
6. review 复核后，64 项普通资料阶段分布为 3/15/4/17/17/2/4/2：项目启动书归入详细设计阶段，采购申请表归入生产制作阶段，发货单和现场安装调试记录归入终验收阶段，尾款发票归入结题阶段。
