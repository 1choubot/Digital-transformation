## Why

20260624 版项目管理流程图已经将阶段内产出文件、财务/采购/审批节点和资料核查节点拆得更细，需要将当前运行模板从 `v20260610` 的 54 项切换到 `v20260624` 的 64 项，避免继续沿用旧模板或误按 66 行规划表导入。

本 change 直接在现有 change id 内完成模板数据、初始化、模拟数据重置脚本和 smoke 校验更新，不新建实现 change。

## What Changes

- 新增 `v20260624` 阶段资料模板规划，目标口径为 64 个流程图直接产出文件。
- 按 review 复核后的阶段分布落地 64 项：立项 3、方案设计 15、合同签订 4、详细设计 17、生产制作 17、预验收 2、终验收 4、结题 2。
- 新增 64 项资料规划表，记录稳定资料编号、阶段、资料名称、`ownerDepartment`、`reviewDepartment`、审核/确认对象备注、模板默认必填、适用条件、提交方式和来源备注。
- 新增内置 `v20260624` 64 项模板数据，并将当前 active 模板版本切换到 `v20260624`。
- 增加模拟数据 reset 脚本，用于清理旧项目阶段资料快照、附件记录、附件物理文件、阶段审批历史、阶段推进/审批/资料业务日志和旧模板，并重置项目流程状态后重新初始化。
- 前端阶段资料汇总展示全部资料、适用资料、适用必填资料和非必填/条件性资料口径；阶段推进门禁仍只使用适用且必填资料齐套。
- 明确阶段推进仍沿用现有机制：当前阶段适用且必填资料全部审核通过，并且阶段审批通过后才允许推进。
- 明确 `7.P1 随机资料移交` 和 `8.P1 资料服务器核查` 暂不纳入普通阶段资料模板，除非后续业务确认它们形成独立文件。
- 明确本 change 不处理 66 行口径，不导入 `7.P1` / `8.P1` 为普通资料模板。
- 明确本 change 不做合同审批流、付款流、采购审批流、设计变更自动触发、资料服务器核查流程或多中心协同任务流。
- 明确当前数据库是模拟数据，可重置旧项目资料并初始化 `v20260624` 64 项模板，不要求保留旧模拟数据兼容方案。
