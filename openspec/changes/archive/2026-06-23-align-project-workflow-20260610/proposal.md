## Why

20260610 版智能制造项目管理流程图已经成为数字化平台后续项目主流程依据，需要将系统项目流程说明和阶段资料模板口径对齐到新版流程。当前旧版 48 项资料模板不再符合新版产出物清单，本 change 只先建立 OpenSpec 规格，供后续实现窗口执行。

## What Changes

- 以 `智能制造项目管理流程图20260610.pdf` 作为系统项目管理主流程依据，并引用 `docs/9.7_智能制造项目整体推进流程_20260610.md` 和 `docs/9.2_阶段资料清单与责任角色表_20260610.md` 作为文字整理依据。
- 保持现有 8 阶段名称和顺序不变：立项、方案设计、合同签订、详细设计、生产制作、预验收、终验收、结题。
- 阶段资料清单切换为 20260610 新版产出物，后续实现时旧版 48 项资料模板直接废弃，不再作为程序运行依据。
- **BREAKING**：不兼容旧模拟项目数据，不做历史项目补初始化，不做旧资料项迁移，不做资料模板 `v1` / `v20260610` 共存。
- 后续实现时新建项目直接初始化 20260610 新版资料项，齐套摘要、阶段推进门禁、我的资料任务和项目总览均以新版资料项为计算基础。
- 本 change 不包含日报周报、文件管理平台联动、消息提醒、复杂审批流、跳阶段、阶段回退或资料多版本共存。
- 本窗口只创建 OpenSpec 文档，不写业务代码、不执行 apply、不归档。

## Capabilities

### New Capabilities

- 无。

### Modified Capabilities

- `project-core`: 将项目流程依据更新为 20260610 新流程图，保持 8 阶段主干和当前阶段齐套门禁推进口径不变。
- `stage-document-checklist`: 将阶段资料模板来源更新为 20260610 新版资料清单，废弃旧 48 项模板，并明确新建项目初始化新版资料项。
- `project-core-frontend`: 要求项目详情、齐套摘要和缺失必填资料展示以后端新版资料项为准，前端不得写死旧资料名或旧数量。

## Impact

- 后端：后续实现会影响阶段资料模板读取来源、模板版本号、资料项数量校验、新项目资料初始化、齐套摘要、阶段推进门禁、我的资料任务和项目总览查询。
- 前端：后续实现会影响项目详情页阶段资料清单、齐套摘要、缺失必填资料展示和相关边界文案。
- 数据：当前项目仍为开发阶段模拟数据，旧 48 项资料模板直接废弃；不做旧项目迁移、历史项目补初始化或兼容展示。
- API：本 change 不要求新增 API 路径；资料状态机、适用/不适用、责任人分配、附件能力保持现有行为，只更换资料项模板口径。
- 排除范围：不实现日报周报、文件管理平台联动、在线表单扩展、消息提醒、复杂审批流、跳阶段、回退、历史兼容或模板共存。
