## ADDED Requirements

### Requirement: 方案设计资料清单数量保持
阶段资料清单 MUST 保持 v20260629 / 71 项资料数量不变，并 MUST 只调整方案设计阶段资料项在内部节点中的归属和完成口径。

#### Scenario: 不改变资料总数
- **WHEN** 系统规划方案设计阶段内部流程
- **THEN** 系统 MUST 保持 v20260629 / 71 项资料数量不变
- **AND** 系统 MUST NOT 因研发成本估算、制造成本估算、财务成本估算、商务标或技术标新增独立资料项
- **AND** 系统 MUST NOT 将 C15 方案评审记录表（内部方案评审）和 C16 方案评审记录表（客户方案评审）合并为一个资料项

#### Scenario: 不新增大阶段资料组
- **WHEN** 系统展示或计算方案设计阶段资料清单
- **THEN** 系统 MUST 继续将相关资料归属于既有“方案设计阶段”
- **AND** 系统 MUST NOT 为 9 个内部节点创建新的大阶段资料组

### Requirement: 方案设计准备资料映射
阶段资料清单 MUST 将方案设计工作计划作为方案设计准备节点产出，并 MUST 第一版按文件上传处理。

#### Scenario: 方案设计工作计划文件上传
- **WHEN** 项目经理处理方案设计准备节点
- **THEN** 系统 MUST 要求上传方案设计工作计划
- **AND** 即使模板目录存在 `方案设计工作计划表-模板.xlsx`，第一版也 MUST 按文件上传处理
- **AND** 系统 MUST NOT 将方案设计工作计划表规划为在线表单

### Requirement: 项目方案分析资料映射
阶段资料清单 MUST 将项目方案分析表和产品功能框图归属于项目方案分析节点。

#### Scenario: 项目方案分析表在线表单
- **WHEN** 技术负责人处理项目方案分析节点
- **THEN** 系统 MUST 将项目方案分析表规划为在线表单
- **AND** 系统 MUST 使用 `项目方案分析表-模板.xlsx` 生成对应模板文件

#### Scenario: 产品功能框图文件上传
- **WHEN** 技术负责人处理项目方案分析节点
- **THEN** 系统 MUST 将产品功能框图作为文件上传产出
- **AND** 系统 MUST 保留产品功能框图在项目方案分析节点

#### Scenario: 工艺时序图目标归属
- **WHEN** 系统规划方案设计阶段资料项归属
- **THEN** 工艺时序图的目标口径 MUST 归入方案设计节点
- **AND** 系统 MUST NOT 因从项目方案分析节点调整到方案设计节点而改变 71 项资料数量

### Requirement: 方案设计 8 个产出映射
阶段资料清单 MUST 在方案设计节点下要求技术负责人提交 8 个方案设计产出。

#### Scenario: 方案设计节点 8 个上传产出
- **WHEN** 技术负责人处理方案设计节点
- **THEN** 系统 MUST 要求提交工艺时序图、节拍表、布局图、3D模型、演示动画、电气功能框图、软件功能框图和项目方案PPT
- **AND** 这些产出第一版 MUST 全部按文件上传处理
- **AND** 系统 MUST NOT 要求主管机械工程师、主管电气工程师或主管软件工程师分别提交

#### Scenario: 方案设计节点完成口径
- **WHEN** 方案设计 8 个产出未全部完成
- **THEN** 系统 MUST 不得将方案设计节点视为完成
- **AND** 系统 MUST 不得进入内部方案评审节点

### Requirement: C15 C16 方案评审记录表独立产出和模板复用
阶段资料清单 MUST 保留 C15 方案评审记录表（内部方案评审）和 C16 方案评审记录表（客户方案评审）两个独立产出/资料项，并 MUST 只复用同一个 `方案评审记录表-模板.xlsx`。

#### Scenario: C15 内部方案评审记录
- **WHEN** 技术负责人提交内部方案评审记录
- **THEN** 系统 MUST 将记录归属于 C15 方案评审记录表（内部方案评审）
- **AND** 系统 MUST 使用 `方案评审记录表-模板.xlsx` 规划在线表单和生成模板文件
- **AND** 系统 MUST 将该记录标识为内部方案评审上下文

#### Scenario: C16 客户方案评审记录
- **WHEN** 技术负责人提交客户方案评审记录
- **THEN** 系统 MUST 将记录归属于 C16 方案评审记录表（客户方案评审）
- **AND** 系统 MUST 使用同一个 `方案评审记录表-模板.xlsx` 规划在线表单和生成模板文件
- **AND** 系统 MUST 将该记录标识为客户方案评审上下文
- **AND** 系统 MUST NOT 将 C15 和 C16 合并为一个资料项

#### Scenario: 多次评审记录保留历史
- **WHEN** 内部方案评审或客户方案评审发生多次提交、审批或退回
- **THEN** 系统 MUST 分别保留 C15 和 C16 的评审记录版本和审批历史
- **AND** 系统 MUST NOT 物理删除旧版本来表达最新状态

#### Scenario: C15 C16 推进关系
- **WHEN** C15 内部方案评审经研发中心负责人审批通过
- **THEN** 系统 MUST 进入 C16 客户方案评审节点
- **AND** C16 客户方案评审经研发中心负责人审批通过后系统 MUST 进入研发成本估算节点

### Requirement: C17 成本估算表三段文件协作
C17 成本估算表 MUST 继续作为一个主资料项，并 MUST 在该资料项内表达研发、制造、财务/运营三段文件上传和审批历史。

#### Scenario: 成本估算表不拆在线表单
- **WHEN** 系统规划 C17 成本估算表
- **THEN** 即使模板目录存在 `成本估算表-模板.xlsx`，第一版也 MUST 按三段文件上传处理
- **AND** 系统 MUST NOT 拆分成本估算 Excel 的公式或多 sheet 为在线表单字段

#### Scenario: 三段文件归属 C17
- **WHEN** 技术负责人、采购负责人或财务会计提交成本估算文件
- **THEN** 研发成本估算文件、制造成本估算文件和财务/运营成本估算文件 MUST 均归属于 C17 成本估算表内部段落
- **AND** 系统 MUST 保留每段文件历史和审批历史

#### Scenario: 后一段可下载前一段
- **WHEN** 采购负责人或财务会计制作后续成本估算文件
- **THEN** 系统 MUST 允许其按权限下载前一段成本估算文件作为基础
- **AND** 系统 MUST NOT 覆盖或删除前一段文件

#### Scenario: C17 完成口径
- **WHEN** 财务/运营成本估算尚未通过总经理最终审批
- **THEN** 系统 MUST 不得将 C17 成本估算表视为最终完成
- **AND** 系统 MUST 将未完成段落作为方案设计阶段推进阻塞原因

### Requirement: 报价单和投标书资料映射
阶段资料清单 MUST 将报价单作为报价流程上传产出，并 MUST 将商务标和技术标作为投标书下的两个上传槽。

#### Scenario: 报价单文件上传
- **WHEN** 商务负责人处理报价流程
- **THEN** 系统 MUST 要求上传报价单
- **AND** 因模板目录中没有报价单模板，第一版 MUST 按文件上传处理
- **AND** 系统 MUST NOT 承诺生成报价单模板文件

#### Scenario: 投标书双上传槽
- **WHEN** 系统处理投标流程
- **THEN** 系统 MUST 使用投标书产出承载商务标和技术标两个必填上传槽
- **AND** 商务标 MUST 由商务负责人上传
- **AND** 技术标 MUST 由技术负责人上传
- **AND** 系统 MUST NOT 新增“商务标”和“技术标”两个独立资料项

#### Scenario: 投标书完成口径
- **WHEN** 商务标或技术标任一上传槽未完成
- **THEN** 系统 MUST 不得允许提交投标总经理审批
- **AND** 系统 MUST 展示缺失上传槽作为投标节点阻塞原因

### Requirement: 方案设计资料专用完成口径
纳入方案设计内部状态机管理的资料项和上传槽 MUST 使用专用节点状态派生完成状态，普通资料审核状态不得形成第二套完成口径。

#### Scenario: 专用资料不使用普通确认替代节点审批
- **WHEN** 项目方案分析、C15 内部方案评审、C16 客户方案评审、C17 成本估算、报价或投标资料处于专用节点流程中
- **THEN** 普通资料确认、退回或提交状态 MUST NOT 替代专用节点审批状态
- **AND** 资料完成状态 MUST 由对应专用节点的最终状态派生

#### Scenario: 普通资料审核入口不适用
- **WHEN** 用户查看纳入方案设计专用状态机的资料项或上传槽
- **THEN** 系统 MUST 将其普通资料审核入口视为不适用
- **AND** 后端直接收到普通审核请求时 MUST 拒绝或返回专用流程错误

#### Scenario: 齐套和推进使用专用完成口径
- **WHEN** 系统计算方案设计阶段齐套摘要或阶段推进门禁
- **THEN** 系统 MUST 使用方案设计内部节点状态和专用完成口径
- **AND** 系统 MUST NOT 因普通资料状态与专用节点状态不一致而产生双重阻塞或双重通过
