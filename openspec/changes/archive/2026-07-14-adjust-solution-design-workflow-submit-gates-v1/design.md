## Decisions

### Decision 1: 重提门禁使用“当前有效文件”而非严格 current revision

退回后，节点 revision 递增仍保留，用于记录新一轮处理历史；但提交门禁不再要求每个已上传槽必须等于当前 revision。旧 current file 若仍是该槽当前有效文件，且未被删除、未被新的上传覆盖，就可满足重提门禁。

重新上传时，仍按现有上传替换规则产生新文件，并把新文件设为 current file；新文件优先于旧文件满足后续提交、下载和派生规则。

该规则必须覆盖 C17 成本估算链路：财务成本估算被总经理退回后，研发成本估算、制造成本估算、财务/运营成本估算三个上传槽的旧 current file 均可继续满足对应节点重提门禁，除非用户重新上传覆盖。

Rationale: 退回很多时候只要求重新确认或补充说明，强制用户重复上传同一文件增加操作成本，也制造无意义文件版本。

### Decision 2: 报价/投标分支选择前移到财务总经理审批通过动作

财务成本估算总经理审批通过时，后端必须同时接收并校验 `quotation` 或 `tender` 分支选择。审批通过、分支选择、日志写入和下一节点激活需要保持同一事务语义。

`quotation_or_tender` 节点不再展示“未选择分支后等待选择”的主状态；进入该节点时已经有分支，前端直接显示报价流程或投标流程。若审批请求缺少分支，后端应拒绝审批通过并返回明确业务错误。

旧报价/投标节点选择接口在财务审批已选择分支后必须拒绝重复选择，返回明确业务错误，不改变分支状态，也不写新的分支选择成功 operation log。

Rationale: 业务判断报价或投标发生在总经理确认财务成本估算时，提前记录可减少一个独立选择步骤，并避免工作台产生额外待办。

### Decision 3: 方案设计 8 个产出引入按槽位豁免状态

每个 C07-C14 上传槽可记录：

- 是否豁免。
- 豁免操作人。
- 豁免时间。
- 豁免原因或备注。

提交门禁按每个槽位 `current file exists OR exemption active` 判断。用户对已豁免槽位重新上传文件时，系统必须自动取消该槽位“无需上传”豁免，写入 operation log，并以新 current file 为准。

Rationale: “无需上传”是单项产出业务判断，不应通过删除资料、减少资料数量或改变 C07-C14 编码实现。

### Decision 4: 派生资料、待办、日志和自动推进都消费同一门禁结果

- C04-C19 派生完成：C07-C14 对应资料在方案设计节点批准后，若对应槽位已上传或已豁免，均可派生为完成或等价满足状态。
- 自动推进：仍只在 C04-C19 齐套且报价/投标节点完成后触发；豁免只影响 C07-C14 的满足判断，不跳过报价/投标和成本估算节点。
- 工作台待办：已上传或已豁免的产出不得继续生成“待上传/待提交”处理项；未上传且未豁免的产出仍应出现在技术负责人待办中。
- operation log：重提复用旧文件、财务审批选择分支、旧选择接口拒绝重复选择、产出标记无需上传/取消无需上传、重新上传自动取消豁免都必须记录或保持可审计结果；拒绝类操作不得写成功选择日志。

Rationale: 如果提交门禁、派生资料、待办和日志各自计算，会导致前端显示可提交但阶段齐套不通过，或工作台重复产生待办。

## Risks / Trade-offs

- [Risk] 放宽 revision 门禁可能误用过旧文件。 → Mitigation: 只允许 current file 复用，不允许历史非 current 文件绕过门禁。
- [Risk] 报价/投标选择前移会影响已有 API DTO。 → Mitigation: 规划时明确保持返回字段兼容，必要时新增字段而不删除旧字段；旧选择接口固定为拒绝重复选择且不改变状态。
- [Risk] “无需上传”与现有不适用语义混淆。 → Mitigation: 豁免只作用于方案设计 8 个产出上传槽，不减少资料项，也不改全局阶段资料适用性模型。
- [Risk] 工作台待办和阶段派生遗漏新门禁。 → Mitigation: 每批实现都必须包含待办、派生齐套和自动推进回归测试。
- [Risk] 前移报价/投标选择改变 operation log 顺序。 → Mitigation: 明确新日志语义并测试关键日志存在，不依赖无意义的旧选择待办。

## Migration Plan

1. Batch 1 实现退回后复用旧文件重提，优先减少当前操作阻塞，不引入 UI 新状态。
2. Batch 2 实现财务成本估算总经理审批时选择报价/投标，迁移报价/投标节点的前端展示和后端门禁。
3. Batch 3 实现 C07-C14 “无需上传”豁免状态、前端入口、派生齐套和待办。
4. 每批完成后运行方案设计 workflow 回归、OpenSpec 校验和 diff check。
5. 全部实现完成后再归档和提交。

Rollback strategy: 每批应保持可独立回退；如果某批引入门禁差异，优先恢复对应门禁判断和 DTO 字段，不回滚无关批次。
