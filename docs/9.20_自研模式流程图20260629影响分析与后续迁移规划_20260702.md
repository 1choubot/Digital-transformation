# 9.20 自研模式流程图 20260629 影响分析与后续迁移规划

## 1. 输入来源与当前基线

输入流程图：`智能制造项目（自研模式）管理流程图20260629.pdf`。

对照基线：

- 当前运行资料模板仍以 20260625 64 项为准，代码版本为 `v20260625`。
- 当前完成规则仍以 `completionMode` 为主，统计为 `submit_only 33`、`approval_required 24`、`conditional_submit 7`、`conditional_approval 0`。
- 当前项目入口为项目总览，单项目入口为项目工作区。
- 当前项目工作区已完整支持立项阶段 `1.1 / 1.2 / 1.3` 在线表单、责任人、评价审批、返工和前置门禁。
- 当前其他 7 个阶段仍可使用占位和旧资料清单辅助入口，不要求全部在线表单化。

数量口径：

- 当前系统运行基线：20260625 64 项资料模板，不因本分析变更。
- 20260629 PDF 图面红色产出数：67 项。这是按流程图表面红色产出块直接统计的数量。
- 按业务修正后的逻辑产出数：71 项。这是后续模板调整候选数量，不是当前系统运行数量。
- 67 到 71 的差异来自 4 处流程图省略：领导流程图把“准备”和“签订”共用一个成品产出，实际业务应拆成“草稿 + 成品”。
- 后续不应长期支持这些伪多对一；迁移时应按“蓝色节点 -> 独立产出/草稿产出 -> 成品产出”建模。`成本估算表` 是当前确认的例外，可按多人/多节点协作同一表处理。

## 2. 版本结论

1. 20260629 仍保留 8 阶段：立项、方案设计、合同签订、详细设计、生产制作、预验收、终验收、结题。
2. 20260629 不推翻项目总览入口；项目总览仍是跨项目主入口。
3. 20260629 不推翻项目工作区框架；8 阶段导航、蓝色节点、节点产出和旧资料清单辅助区仍是后续承载结构。
4. 20260629 不推翻立项阶段 `1.1 / 1.2 / 1.3` 当前实现；立项阶段在线表单专用入口和 `1.2` 专用评价/审批暂不调整。
5. 后续重点不是重建主架构，而是补齐后 7 阶段蓝色节点和产出映射，并逐项确认疑似新增、改名、阶段移动和 completionMode 变化。
6. 20260629 流程图不得直接替换当前 20260625 64 项运行模板；任何模板或规则调整都必须另开独立 change。
7. 20260629 图面红色产出按 67 项理解；业务修正后的逻辑产出按 71 项理解，其中 4 项草稿产出建议正式纳入后续模板调整候选。
8. 当前真实多节点协作同一产出只保留 `成本估算表`；技术协议、销售合同、采购合同等准备/签订共用产出应判定为流程图漏画草稿产出，而不是可长期支持的多对一模型。

## 6. 后续实施建议

### 第一批：方案设计阶段蓝色节点和产出映射

- 目标：补齐方案设计阶段蓝色节点、节点产出和旧资料清单入口关系。
- 重点：`2.1-2.15` 映射、内部/客户评审 NO 回退展示、成本估算/价格估算/报价节点语义。
- 暂不处理：投标书模板新增，除非业务先确认。

### 第二批：合同签订阶段蓝色节点和产出映射

- 目标：补齐技术协议、销售合同、合同审核、预付款发票和项目启动通知相关节点。
- 重点：将准备技术协议和准备销售合同建模为草稿候选，签订节点继续对应成品技术协议和销售合同；判断多角色审核落在草稿产出还是成品签订确认上。
- 暂不处理：付款流、发票审批流、未收款启动审批流。

### 第三批：详细设计阶段蓝色节点和产出映射

- 目标：补齐详细设计准备、详细设计、内部/客户设计评审、图纸审查、客户会签节点。
- 重点：保持 `4.14 产品平面图`、`4.15 产品零部件清单` 为 `submit_only`，`4.16 图纸审查记录` 为 `approval_required` 的当前口径，除非业务另行确认。

### 第四批：生产制作阶段蓝色节点和产出映射

- 目标：补齐采购、采购合同、生产、随机资料、来料检验、入库领料、厂内安装调试、设计变更、自验收节点。
- 重点：单独确认生产制作阶段 `技术协议草稿/技术协议`、`采购合同草稿/采购合同`、`合格供应商评价表`、`生产记录表` 是否新增或调整模板项。
- 暂不处理：通用采购审批流、供应商管理系统或生产执行系统。

### 第五批：预验收、终验收、结题阶段蓝色节点和产出映射

- 目标：补齐预验收、发货款、发货通知、发货、现场安装调试、终验收、随机资料移交、产品培训、尾款、资料整理归集和结题节点。
- 重点：确认 `资料移交清单` 是否纳入资料模板；确认发货通知是否只是过程节点。
- 暂不处理：付款流、发票审批流、资料服务器自动核查或文件平台联动。
