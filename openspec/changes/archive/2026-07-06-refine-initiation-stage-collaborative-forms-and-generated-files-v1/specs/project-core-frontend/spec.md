## ADDED Requirements

### Requirement: 新建项目客户联系人前端呈现

项目核心前端 MUST 在新建项目、项目详情、项目列表、项目总览和立项项目输入节点中展示客户联系人，并 MUST 将其与客户联系方式保持同级业务字段。新建项目提交时客户联系人为空 MUST 阻止提交并提示；旧项目客户联系人为空时页面不得崩溃。

#### Scenario: 新建项目填写客户联系人
- **WHEN** 用户打开新建项目页面
- **THEN** 前端 MUST 展示客户联系人字段
- **AND** 客户联系人 MUST 位于客户联系方式前
- **AND** 客户联系人为空时前端 MUST 阻止提交并展示提示

#### Scenario: 项目页面展示客户联系人
- **WHEN** 前端展示项目列表、项目总览、项目详情或立项项目输入节点
- **THEN** 页面 MUST 能展示客户联系人
- **AND** 旧项目客户联系人为空时页面 MUST 不崩溃

### Requirement: 1.2 协同填写前端体验

项目核心前端 MUST 支持 `1.2 项目立项审批表` 商务负责人和技术负责人协同填写；不同负责人 MUST 看到和操作其职责范围内的填写区域，并能看到双方完成状态。

#### Scenario: 商务负责人填写区域
- **WHEN** 商务负责人打开 `1.2 项目立项审批表`
- **THEN** 前端 MUST 允许填写基础模块和商务模块
- **AND** 基础模块 MUST 按模板表头层级展示顶层表头信息、客户基本信息和项目基本信息
- **AND** 项目名称和项目号/项目编号 MUST 展示在顶层表头信息中，不得归入客户基本信息
- **AND** 基础模块 MUST NOT 展示模板外客户调查或项目调查字段
- **AND** 技术负责人 MUST NOT 被强行展示为基础模块表头字段
- **AND** 前端 MUST 提供项目编号填写或确认入口

#### Scenario: 技术负责人填写区域
- **WHEN** 技术负责人打开 `1.2 项目立项审批表`
- **THEN** 前端 MUST 允许填写技术模块
- **AND** 前端 MUST NOT 允许技术负责人修改基础模块和商务模块

#### Scenario: 双方完成状态可见
- **WHEN** 用户查看 `1.2 项目立项审批表`
- **THEN** 前端 MUST 展示商务部分完成状态
- **AND** 前端 MUST 展示技术部分完成状态
- **AND** 双方未完成时前端 MUST 表达评价审批尚未启动或不可处理

#### Scenario: 工作台分别展示填写待办
- **WHEN** 商务负责人或技术负责人存在 `1.2` 填写任务
- **THEN** 我的工作台 MUST 分别展示商务填写待办和技术填写待办
- **AND** 待办 MUST 能进入对应项目和对应填写区域
- **AND** `1.2` MUST NOT 展示普通资料责任人分配入口或生成普通资料责任人待办

#### Scenario: 已提交部分不可继续编辑
- **WHEN** 商务部分或技术部分已经提交
- **THEN** 前端 MUST 禁用对应字段和提交按钮
- **AND** 前端 MUST 展示需要退回后重新填写的阻塞原因

### Requirement: 1.2 项目编号前置前端门禁

项目核心前端 MUST 将项目编号入口前置到 `1.2` 填写阶段，并 MUST 在 `1.2` 提交前校验项目编号存在且后端唯一性校验通过。

#### Scenario: 1.2 阶段显示项目编号入口
- **WHEN** 商务负责人填写 `1.2` 基础模块
- **THEN** 前端 MUST 显示项目编号填写或确认入口
- **AND** 文案 MUST 表达该编号会进入 `1.2` 和后续 `1.3 项目立项通知`
- **AND** 项目详情顶部 MUST NOT 再展示独立“保存项目编号”入口

#### Scenario: 缺少项目编号阻止提交
- **WHEN** 商务负责人尝试提交 `1.2` 商务部分或整体进入评价审批
- **THEN** 前端 MUST 校验项目编号不为空
- **AND** 若项目编号为空，前端 MUST 展示阻塞提示并不得假装提交成功

#### Scenario: 1.3 自动带出项目编号
- **WHEN** 营销中心负责人打开 `1.3 项目立项通知`
- **THEN** 前端 MUST 自动展示项目编号、项目名称、客户单位和客户联系人相关主数据
- **AND** 前端 MUST NOT 要求在 `1.3` 阶段重新生成项目编号

### Requirement: 立项产出文件查看方向

项目核心前端 MUST 记录在线表单提交后查看模板生成文件的方向，但本 change MUST NOT 展示已生成文件的承诺或伪造下载入口。

#### Scenario: 在线表单作为填写入口
- **WHEN** 用户处理 `1.1`、`1.2` 或 `1.3`
- **THEN** 前端 MUST 继续展示在线表单填写入口
- **AND** 前端 MUST NOT 将填写入口改为普通附件上传

#### Scenario: 后续默认查看生成文件
- **WHEN** 后续实现模板文件生成能力
- **THEN** 有权限查看者 SHOULD 默认查看生成后的模板文件
- **AND** 前端 SHOULD 支持文件预览、下载或版本查看入口

#### Scenario: 本 change 不伪造文件生成
- **WHEN** 本 change 完成实现
- **THEN** 前端 MUST NOT 展示 Excel、Word、PDF 已生成的假状态
- **AND** 前端 MUST NOT 提示已经接入文件平台
