# 9.19 前端工作区体验统一规划

## 背景

当前前端已完成“项目总览 + 项目工作区 + 立项阶段在线表单”的主架构：项目总览作为项目入口，项目工作区承载单项目 8 阶段导航，立项阶段已具备蓝色节点、节点产出、在线表单和 `1.2` 评价/审批入口。

当前剩余问题不是业务规则缺口，而是界面语言、操作层级、组件边界和旧资料清单兼容区仍不统一。项目总览、项目工作区、我的工作台、用户管理和旧资料清单之间的页面头部、按钮、状态、空态、错误提示和组件职责还存在风格差异；如果继续叠加第二阶段补录或文件平台联动，会增加页面混乱和维护成本。

## 目标

- 统一项目总览作为跨项目入口的体验。
- 统一项目工作区作为单项目主操作区的体验。
- 保留左侧 8 阶段导航，右侧固定组织为阶段说明、蓝色节点列表、节点产出区、在线表单/评价审批动作区。
- 将下方旧资料清单降级为辅助兼容区，建议折叠或弱化，不作为立项阶段主操作区。
- 统一我的工作台与项目工作区深链体验，任务入口进入工作区后定位到对应阶段、节点或产出，但不自动打开在线表单。
- 统一新建项目、项目总览、项目工作区、我的工作台、用户管理的页面头部、按钮、状态、空态和错误提示语义。
- 收敛通用 UI 样式和组件边界，减少 `styles.css` 继续无序膨胀。
- 继续明确旧 `ProjectListPage.vue` 不作为用户可见入口。

## 范围

本 change 是前端体验统一规划，不新增业务规则。第一版实施时主要影响 `digital-platform-web/src/**`，并复用现有接口字段组合 UI：

- `/api/projects/overview-dashboard`
- `/api/projects/:id`
- `/api/projects/:id/workspace`
- `/api/projects/:id/stage-document-checklist`
- `/api/projects/:id/stage-documents/:documentId/online-form`

## 边界

- 不新增后端接口。
- 不改数据库，不写 migration。
- 不新增业务权限。
- 不做第二阶段项目经理、项目模式、立项日期补录。
- 不补齐其他 7 阶段蓝色节点。
- 不恢复文件平台联动。
- 不处理 `file-platform-integration-v1` 或 `define-digital-platform-v1`。
- 不改变立项阶段 `1.1 / 1.2 / 1.3` 已确认的在线表单、责任人、评价审批和返工业务规则。

## 验收重点

- `/projects` 和主导航进入项目总览。
- 项目工作区保持左侧 8 阶段导航和右侧阶段工作区层级。
- 点击蓝色节点先展示节点产出，不直接打开在线表单。
- 立项阶段主操作位于上方节点产出卡片。
- 下方旧资料清单为辅助兼容区，其他 7 阶段临时操作入口仍可访问。
- 工作台深链进入项目工作区后保留定位，不自动打开在线表单。
- 桌面和移动 viewport 下导航、节点、产出区、旧清单和动作区不重叠、不溢出。
