## Why

当前阶段资料权限主要依赖责任人部门推断“本中心相关资料”。未分配责任人的资料没有责任人部门，导致中心负责人无法查看或分配本中心默认负责的未分配资料，例如研发中心负责人无法处理研发中心默认负责但尚未分配责任人的 3D 模型等资料。

本 change 为现有 `v20260610` 54 项阶段资料模板补充结构化 `ownerDepartment` / `reviewDepartment`，并让权限、工作台和前端展示按结构化归属中心识别资料范围。

## What Changes

- 为阶段资料模板和项目级阶段资料快照增加可空 `ownerDepartment` 与 `reviewDepartment`。
- 继续使用现有 `v20260610` 54 项资料模板，不切换到 v20260624，不写入 20260624 的 64 个文件产出。
- 为 `v20260610` 54 项模板补充结构化归属中心，字段值使用现有 `BUSINESS_DEPARTMENT` 常量。
- 新建项目初始化资料时保存 `ownerDepartment` / `reviewDepartment`，旧数据通过 schema 兼容和默认值补齐逻辑处理。
- 阶段资料清单、工作台和相关接口返回结构化归属中心字段。
- 中心负责人可以查看、分配 `ownerDepartment = 本人部门` 的未分配资料，并审核 `reviewDepartment = 本人部门` 且 `status = submitted` 的资料。
- 普通员工仍只看到自己负责的资料；项目经理仍可查看和分配本项目全部资料；总经理仍可查看全部资料。
- 系统管理员、总经理助理不因归属中心字段获得业务资料、附件或审核权限。
- 前端展示默认责任中心/审核中心中文名，并继续按后端返回的权限字段控制按钮。
- 本 change 不实现发票、付款、合同审核、采购申请、设计变更、成本估算协作等 20260624 复杂节点。
- 完成后不归档，不提交，等待审查会话 review。

## Capabilities

### New Capabilities

- 无。

### Modified Capabilities

- `stage-document-checklist`: 为 `v20260610` 阶段资料模板和项目级资料快照增加结构化归属中心，并按归属中心调整中心负责人资料可见、分配、审核、附件访问和工作台审核待办来源。
- `project-core`: 按结构化归属中心调整中心负责人项目可见性、阶段推进权限和 `stage_advance` 工作台待办来源。
- `project-core-frontend`: 项目详情和工作台展示结构化归属中心，并按后端权限字段控制资料分配、资料审核和附件按钮。

## Impact

- 后端：影响阶段资料模板常量、模板规范化、数据库 schema 初始化/兼容、项目资料初始化、项目可见性、阶段推进权限、阶段资料查询、资料权限判断、责任人分配、资料审核、资料适用性、附件权限和我的工作台。
- 前端：影响项目详情阶段资料清单展示、责任中心/审核中心展示、按钮权限判断和受限视图。
- 数据：新增可空 `owner_department` / `review_department` 字段；旧模拟数据允许字段为空或按 `v20260610` 模板补默认值。
- API：阶段资料清单和工作台资料项返回新增 `ownerDepartment` / `reviewDepartment` 字段；不新增 v20260624 模板 API。
- 排除范围：不修改 `define-digital-platform-v1`，不切换模板版本，不实现 20260624 复杂业务节点，不归档不提交。
