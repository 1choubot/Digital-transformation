# 项目节点页面架构与通用能力设计方案

# Codex执行规范版

## 1. 执行目标

本文档用于指导 Codex 对当前数字化管理平台前端源码进行后续重构。

执行目标：

> 在保持现有业务能力不丢失的前提下，将项目节点改造成"独立节点页面 +
> 通用平台能力"的架构。

最终实现：

    ProjectDetailLayout

            |
            |
    NodePageRouter

            |
            |
    节点独立 Vue 页面

            |
            |
    通用业务能力组件

------------------------------------------------------------------------

# 2. 当前源码状态（必须理解）

当前已经完成：

-   ProjectDetailPage.vue 已拆分；
-   ProjectDetailLayout.vue 已作为项目工作区容器；
-   ProjectProcessTree.vue 已实现项目流程导航；
-   NodePageRouter.vue 已实现节点页面动态加载；
-   nodePages.js 已支持 nodeCode 到 Vue 页面映射；
-   在线表单能力已经存在。

因此：

## 禁止重新实现以下功能

禁止：

-   重写项目导航；
-   重写节点状态系统；
-   删除 NodePageRouter；
-   删除在线表单能力。

本阶段目标是架构调整，不是重新开发。

------------------------------------------------------------------------

# 3. 核心架构规则

## 规则1：节点页面必须独立

每个业务节点必须拥有独立 Vue 页面。

例如：

    ProjectInputPage.vue
    MarketResearchPage.vue
    ProjectApprovalPage.vue
    ProjectNoticePage.vue

禁止：

所有节点继续进入：

    GenericProjectNodePage.vue

或者：

    ProjectWorkspaceOutputPanel.vue

统一判断。

------------------------------------------------------------------------

# 4. 节点页面职责规则

节点页面负责：

-   页面布局；
-   页面标题；
-   业务区域组合；
-   特殊业务展示。

例如：

    MarketResearchPage.vue

负责：

-   市场调研页面结构；
-   调研说明；
-   表单区域；
-   附件区域。

------------------------------------------------------------------------

节点页面禁止负责：

-   通用保存逻辑；
-   状态刷新逻辑；
-   通用权限判断；
-   通用附件处理。

------------------------------------------------------------------------

# 5. 在线表单处理规则

## 5.1 在线表单属于平台能力

当前：

    InitiationOnlineFormEditor.vue

名称存在阶段耦合。

实际职责：

    节点在线表单渲染能力

------------------------------------------------------------------------

## 5.2 调整方向

建议调整为：

    NodeOnlineFormEditor.vue

职责：

只负责：

-   根据 schema 渲染字段；
-   绑定 formData；
-   输入校验；
-   触发表单事件。

------------------------------------------------------------------------

禁止：

在线表单组件中增加：

-   市场调研业务逻辑；
-   审批业务逻辑；
-   节点状态处理。

------------------------------------------------------------------------

# 6. 后端表单模型约束

后端已经采用：

    form_schema_json
    +
    form_data_json

模式。

保持该模式。

禁止：

-   为不同节点创建不同表结构；
-   将 Vue 页面名称存入数据库。

------------------------------------------------------------------------

正确方式：

后端返回：

    nodeCode
    +
    form schema
    +
    form data

前端决定：

    nodeCode

    ↓

    Vue 页面

------------------------------------------------------------------------

# 7. 节点状态规则

节点状态属于平台通用能力。

状态：

    PENDING
    PROCESSING
    WAIT_APPROVAL
    COMPLETED
    RETURNED
    FAILED

适用于：

-   项目输入；
-   市场调研；
-   方案设计；
-   审批；
-   验收。

------------------------------------------------------------------------

禁止：

在节点页面中实现：

``` javascript
if(nodeCode==='xxx') {
    updateStatus()
}
```

------------------------------------------------------------------------

# 8. 状态刷新规则

所有节点统一流程：

    用户操作

    ↓

    调用业务接口

    ↓

    后端更新节点状态

    ↓

    刷新节点数据

    ↓

    更新左侧导航状态

------------------------------------------------------------------------

建议抽离：

    composables/node/

    ├── useNodeForm.js
    ├── useNodeStatus.js
    ├── useNodeRefresh.js
    └── useNodeActions.js

------------------------------------------------------------------------

# 14. 验收标准

完成后必须满足：

## 页面

-   不同 nodeCode 显示不同 Vue 页面；
-   页面布局由节点决定；
-   在线表单仍然可填写。

------------------------------------------------------------------------

## 数据

-   form_schema_json 不改变；
-   form_data_json 不改变；
-   API协议尽量保持兼容。

------------------------------------------------------------------------

最终形成：

    节点页面独立

    +

    在线表单复用

    +

    状态能力通用

    +

    业务组件复用

的项目管理平台架构。
