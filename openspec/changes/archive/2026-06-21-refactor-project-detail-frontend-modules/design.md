## Context

当前前端项目详情页已经承载项目基础信息、8 阶段状态、阶段资料清单、资料状态流转、适用性操作、资料责任人分配、阶段齐套摘要、阶段推进和业务日志。`ProjectDetailPage.vue` 作为页面容器持续膨胀，会让后续文件联动、在线表单和权限相关能力更难定位状态依赖、刷新边界和错误处理。

本 change 是前端结构性重构，不新增业务能力。目标是在保持现有用户可见结果、API 调用、错误提示和边界说明不变的前提下，把项目详情页拆分为职责更清晰的组件和少量 helper。后端接口、数据库迁移、业务状态机、权限规则和业务日志规则均不在本 change 范围内。

## Goals / Non-Goals

**Goals:**

- 梳理 `ProjectDetailPage.vue` 当前区域、状态依赖、刷新链路和错误提示。
- 将项目详情页拆为页面容器和若干区域组件，降低单文件复杂度。
- 将阶段资料清单展示、资料项操作、责任人选择/展示、阶段推进和业务日志拆出为可维护模块。
- 保留现有页面路由、API 客户端方法、接口字段、刷新顺序和用户可见业务结果。
- 保留现有边界说明文案，包括手工状态不代表文件上传、齐套摘要基于手工状态和人工适用性、资料责任人不代表权限控制。
- 更新 Web README，说明项目详情页已模块化但业务能力不变。
- 通过前端 build 和轻量页面行为 smoke 验证关键入口不回退。

**Non-Goals:**

- 不实现文件上传/下载、文件管理平台联动、在线表单、消息提醒、超期提醒、项目成员权限、复杂权限或新的业务入口。
- 不修改后端业务代码、API 路径、请求/响应字段、错误码、权限规则或数据库迁移。
- 不改变资料状态流转、资料适用性规则、责任人分配规则、阶段推进规则或业务日志规则。
- 不重写整个前端应用，不引入新的 UI 框架，不做大范围视觉样式重做。
- 不修改、完成或归档 `define-digital-platform-v1`。

## Decisions

### 让 `ProjectDetailPage.vue` 保持页面容器职责

`ProjectDetailPage.vue` 继续作为项目详情页的路由级页面，负责读取 `projectId`、持有页面级加载状态、调用既有 API、协调刷新项目详情、阶段资料清单、责任人候选用户和业务日志。区域组件通过 props 接收数据，通过 emits 通知页面容器执行既有操作。

这种方式避免在重构中移动 API 调用语义，也减少拆分后组件之间直接互相调用接口造成的刷新链路分散。替代方案是让每个子组件自行调用接口，但这会在第一版重构中放大状态一致性风险。

### 按页面区域拆分组件，而不是按每个字段拆分

第一版优先拆分为项目基础信息/标题区域、阶段资料清单、资料项操作/追溯/责任人子区域、阶段推进和业务日志等模块。可根据现有代码形态进一步拆出责任人选择控件或展示 helper，但不把每个小字段都拆成独立组件。

这种拆分粒度能降低单文件复杂度，同时避免过度组件化导致 props 过多、代码跳转成本过高。

### 资料清单组件保留现有展示口径和操作入口

阶段资料清单组件必须继续展示阶段齐套摘要、缺失适用必填资料、资料项基础字段、状态、追溯字段、适用性、责任人和责任人变更追溯。资料项操作组件或 helper 只承接既有提交、确认、退回、标记不适用、恢复适用、分配/清空责任人的 UI 和事件，不引入新的状态机或权限判断。

资料项操作成功后的刷新仍由页面容器执行，确保清单、齐套摘要和业务日志保持现有同步行为。

### 阶段推进和业务日志独立为只读/操作边界明确的模块

阶段推进组件只接收当前阶段、阶段清单摘要和推进状态，触发既有阶段推进事件，不提供目标阶段选择、回退、跳阶段或批量推进。业务日志组件只展示既有日志加载状态、空状态、错误状态和日志字段，不提供日志写入、编辑、删除、筛选导出或复杂分页入口。

### helper 抽取保持小范围

允许将重复的状态文案、追溯展示、责任人显示、资料项操作可用性判断等前端纯展示逻辑抽为 helper。helper 不应包含后端请求、副作用、权限扩展或业务状态机变更。

### 验证以行为不回退为核心

本 change 的验证重点不是新增场景，而是确认重构后关键页面行为仍存在：项目详情页可加载、阶段资料清单可展示、资料状态/适用性/责任人操作入口仍存在、阶段推进入口仍存在、业务日志仍展示、错误提示和边界说明仍保留。前端 build 是必须验证项；如可行，增加源码/轻量 smoke 检查关键组件、文案和事件链路。

## Risks / Trade-offs

- [Risk] 组件拆分后 props/emits 过多，反而增加理解成本。→ 以页面区域为拆分单位，页面容器保留协调职责，避免过度细分。
- [Risk] 操作成功后的刷新链路被拆散，导致清单、齐套摘要或日志不同步。→ 所有写操作仍通过页面容器调用既有处理函数，并集中执行现有刷新逻辑。
- [Risk] 文案或错误提示在迁移模板时丢失。→ 将边界说明、错误提示和关键按钮作为 smoke 验证项。
- [Risk] 样式拆分时引入可视回退。→ 不做大范围样式重做，只在必要时迁移或补充组件 class。

## Migration Plan

本 change 不需要数据库迁移或后端部署顺序。实现时先梳理现有 `ProjectDetailPage.vue` 区域，再按组件边界小步移动模板和纯展示逻辑，保持每一步前端 build 可通过。回滚时可将组件内容合回 `ProjectDetailPage.vue`，不会影响后端接口或数据。

## Open Questions

无。第一版固定为前端项目详情页行为保持型结构拆分，不设计文件联动、在线表单、权限矩阵或新的业务能力。
