## Context

当前系统已经有项目工作区 shell：左侧 8 阶段导航、阶段内蓝色模块、上方产出卡片和下方旧资料清单兼容区。shell 第一版中，非立项产出卡片的处理入口只负责定位到下方旧资料清单，通用上传、提交、审核、退回等执行能力仍由旧资料清单承载。

本 change 的设计目标是为下一步迁移建立边界：先把现有 64 项项目资料的通用操作迁移到上方产出卡片；迁移稳定后再降级和清理下方旧资料清单；最后才评估 v20260629 71 项目标模板默认启用。

## Goals / Non-Goals

**Goals:**

- 将上方产出卡片规划为现有项目资料通用操作的主入口。
- 第一版只迁移当前已存在项目资料的通用能力。
- 复用现有接口、权限、状态字段、附件能力、业务日志和阶段推进判断。
- 明确下方旧资料清单从主入口降级为兼容区的路线。
- 保护 `1.1 / 1.2 / 1.3` 立项在线表单和 `1.2` 专用评价审批。

**Non-Goals:**

- 不切换 v20260629 71 项模板为新项目默认模板。
- 不迁移旧项目，不补初始化旧项目，不改写旧项目 64 项资料状态。
- 不把新增 71 项候选落库。
- 不新增资料表结构，不写 migration。
- 不创建第二套上传、提交、审核、退回规则。
- 不新增复杂流程引擎或通用 BPM。
- 本 change 不隐藏、不物理删除旧资料清单组件；隐藏或删除旧资料清单必须通过后续独立 change 处理。

## Decisions

### Decision 1: 三阶段推进

采用三阶段路线：

1. Phase 1：上方产出卡片承载旧资料清单通用操作。
2. Phase 2：下方旧资料清单降级并逐步清理。
3. Phase 3：切换 v20260629 71 项模板。

理由：当前最大风险不是 71 项模板切换，而是同一套资料操作同时存在上方 shell 和下方旧清单两个入口。先统一主入口，再处理旧清单清理，最后再做模板切换，可以降低运行时状态和验收复杂度。

### Decision 2: 只迁移现有项目资料的通用能力

第一版只对已经存在 documentId 或稳定 documentCode 绑定的项目资料开放通用操作。无现有资料绑定的候选产出可以继续显示为 shell 占位或候选项，但不能创建实际资料记录，也不能展示可执行的假按钮。

理由：这能保证迁移过程读取和写入的仍是当前 20260625 64 项资料状态，不提前引入 v20260629 71 项模板的落库和迁移问题。

### Decision 3: 产出卡片复用旧资料清单的后端能力

产出卡片触发责任人分配、附件上传/下载/删除、提交、审核通过、审核退回、返工重提/返工完成、标记不适用、恢复适用时，必须复用现有阶段资料接口、权限、状态字段和业务日志。

理由：迁移目标是前端主入口迁移，不是重新设计资料状态机或执行规则。所有完成状态、权限判断和阶段推进结果必须来自同一套后端状态。

### Decision 4: 立项阶段保持专用入口

`1.1 项目需求表`、`1.2 项目立项审批表`、`1.3 项目立项通知` 继续使用现有在线表单和 `1.2` 专用评价审批。产出卡片可以作为进入这些现有能力的入口，但不得新增普通文件上传或普通提交入口绕过在线表单。

理由：立项阶段已经具备专用在线表单、评价审批、返工和前置门禁，回退为通用文件上传会破坏既有业务语义。

### Decision 5: 旧资料清单先降级再清理

迁移前下方旧资料清单仍可操作；迁移中上方产出卡片作为主入口，下方显示辅助/兼容提示。本 change 只允许把旧资料清单降级为兼容区，不允许物理删除旧资料清单组件，不允许完全移除旧资料清单入口。隐藏或删除旧资料清单必须通过后续独立 change 处理，并且以前置覆盖验收通过为条件。

理由：先保留兼容区可以降低切换风险；清理必须以功能覆盖和验收结果为条件。

## Risks / Trade-offs

- [Risk] 上方产出卡片和下方旧资料清单同时可操作，用户不清楚主入口。→ Mitigation：迁移中明确上方为主入口，下方显示兼容提示，并逐步弱化下方操作入口。
- [Risk] 前端重复实现权限或状态判断，导致上下入口状态不一致。→ Mitigation：产出卡片只使用后端返回的权限、状态、阻塞原因和状态说明，不在前端创建第二套判断。
- [Risk] 无资料绑定的 v20260629 候选产出被误当成可执行资料。→ Mitigation：无 documentId 或稳定 documentCode 绑定时只展示 shell 占位，不执行上传、提交、审核、退回。
- [Risk] 立项在线表单被误迁移为普通资料操作。→ Mitigation：`1.1 / 1.2 / 1.3` 保持专用入口和专用审批，不新增绕过入口。

## Migration Plan

1. Phase 1 实现上方产出卡片通用操作迁移，覆盖责任人、附件、提交、审核、返工、不适用、阻塞原因、权限原因和状态说明。
2. Phase 2 将下方旧资料清单降级为兼容区；本 change 不隐藏、不物理删除旧清单组件。
3. Phase 3 在旧清单清理完成后，通过独立 change 评估并实施 v20260629 71 项模板默认启用。

每个阶段都必须独立验证 API check、Web build、OpenSpec validate、浏览器/人工验收、立项在线表单保护、71 模板未切换、旧项目未迁移，以及桌面/移动布局不重叠、不溢出。

## Open Questions

- 下方旧资料清单降级后的最终形态是隐藏、折叠还是删除，需要在后续独立 change 中结合覆盖验收结果决定。
- 上方产出卡片每类操作的具体交互布局，需要在 implementation 中结合现有组件和移动端空间确认。
