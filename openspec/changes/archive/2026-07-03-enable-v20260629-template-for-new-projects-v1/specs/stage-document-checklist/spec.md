## ADDED Requirements

### Requirement: v20260629 新项目默认阶段资料模板

阶段资料清单能力 MUST 支持将 v20260629 71 项目标资料模板启用为新建项目默认模板，同时 MUST 保持旧项目既有 20260625 64 项资料记录不变。

#### Scenario: 新建项目初始化 v20260629 71 项
- **WHEN** 本 change implementation 完成后用户创建新项目
- **THEN** 系统 MUST 默认按 v20260629 71 项目标资料模板初始化项目级阶段资料
- **AND** 新项目资料项数量 MUST 为 71 项
- **AND** 新项目资料项 MUST 保存 v20260629 模板版本或等价稳定版本标识

#### Scenario: 旧项目继续保持 64 项
- **WHEN** 系统存在已按 20260625 64 项初始化的旧项目
- **THEN** 系统 MUST NOT 因启用 v20260629 新项目默认模板而迁移、补初始化或改写旧项目资料
- **AND** 旧项目 MUST 继续返回其已有 64 项项目级阶段资料

#### Scenario: 旧合同审核兼容资料不进入 71 项
- **WHEN** 系统初始化 v20260629 新项目资料
- **THEN** 系统 MUST NOT 初始化 `3.3 合同审核记录表（销售合同）` 或 `5.4 采购合同审核记录表`
- **AND** 系统 MUST NOT 将 `LC33` 或 `LC54` 计入 v20260629 71 项模板
- **AND** `LC33 / LC54` MUST 继续只作为旧项目兼容 workspace 输出

#### Scenario: 非立项阶段默认文件上传
- **WHEN** v20260629 新项目初始化非立项阶段资料
- **THEN** 非立项资料 MUST 默认使用文件上传或附件上传能力承载
- **AND** 系统 MUST NOT 因模板启用自动新增复杂在线表单、专用审批流或流程引擎

#### Scenario: 立项在线表单继续沿用
- **WHEN** v20260629 新项目包含 `1.1 项目需求表`、`1.2 项目立项审批表` 或 `1.3 项目立项通知`
- **THEN** 系统 MUST 继续沿用当前在线表单、责任人、评价审批、返工和前置门禁规则
- **AND** 系统 MUST NOT 将这三项回退为普通文件上传或普通提交审核入口

#### Scenario: 模板启用不要求数据库 migration
- **WHEN** 团队实现 v20260629 新项目默认模板启用
- **THEN** 实现 MUST NOT 修改数据库 schema 或写 migration
- **AND** 模板版本和资料项区分 MUST 复用现有模板版本、项目资料记录和初始化结构
