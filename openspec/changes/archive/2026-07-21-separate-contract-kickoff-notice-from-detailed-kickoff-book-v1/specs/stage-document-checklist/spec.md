## MODIFIED Requirements

### Requirement: v20260629 新项目默认阶段资料模板

阶段资料清单能力 MUST 支持将 v20260629 71 项目标资料模板启用为新建项目默认模板，同时 MUST 保持旧项目既有 20260625 64 项资料记录不变。

#### Scenario: 新建项目初始化 v20260629 71 项
- **WHEN** 本 change implementation 完成后用户创建新项目
- **THEN** 系统 MUST 默认按 v20260629 71 项目标资料模板初始化项目级阶段资料
- **AND** 新项目资料项数量 MUST 为 71 项
- **AND** 新项目资料项 MUST 保存 v20260629 模板版本或等价稳定版本标识
- **AND** C25 / `4.1` MUST 初始化为详细设计阶段 `项目启动书`

#### Scenario: 旧项目继续保持 64 项
- **WHEN** 系统存在已按 20260625 64 项初始化的旧项目
- **THEN** 系统 MUST NOT 因启用 v20260629 新项目默认模板而迁移、补初始化或改写旧项目资料
- **AND** 旧项目 MUST 继续返回其已有 64 项项目级阶段资料

#### Scenario: 旧合同审核兼容资料不进入 71 项
- **WHEN** 系统初始化 v20260629 新项目资料
- **THEN** 系统 MUST NOT 初始化 `3.3 合同审核记录表（销售合同）` 或 `5.4 采购合同审核记录表`
- **AND** 系统 MUST NOT 将 `LC33` 或 `LC54` 计入 v20260629 71 项模板
- **AND** `LC33 / LC54` MUST 继续只作为旧项目兼容 workspace 输出

#### Scenario: 非立项阶段默认文件上传
- **WHEN** v20260629 新项目初始化非立项阶段资料
- **THEN** 非立项资料 MUST 默认使用文件上传或附件上传能力承载
- **AND** 系统 MUST NOT 因模板启用自动新增复杂在线表单、专用审批流或流程引擎

#### Scenario: 立项在线表单继续沿用
- **WHEN** v20260629 新项目包含 `1.1 项目需求表`、`1.2 项目立项审批表` 或 `1.3 项目立项通知`
- **THEN** 系统 MUST 继续沿用当前在线表单、责任人、评价审批、返工和前置门禁规则
- **AND** 系统 MUST NOT 将这三项回退为普通文件上传或普通提交审核入口

#### Scenario: 模板启用不要求数据库 migration
- **WHEN** 团队实现 v20260629 新项目默认模板启用
- **THEN** 实现 MUST NOT 修改数据库 schema 或写 migration
- **AND** 模板版本和资料项区分 MUST 复用现有模板版本、项目资料记录和初始化结构

### Requirement: 合同 workflow 资料清单映射
阶段资料清单能力 MUST 支持合同签订专用 workflow 与 v20260629 既有 71 项资料的映射，并 MUST 保持资料总数不变。

#### Scenario: 准备和签订资料映射
- **WHEN** 合同 workflow 同步准备协议和合同、签订协议和合同结果
- **THEN** 技术协议准备线 MUST 映射到既有 C20 稳定资料编码
- **AND** C20 展示名 MUST 修正为 `技术协议`
- **AND** 销售合同准备线 MUST 映射到既有 C22 稳定资料编码
- **AND** C22 展示名 MUST 修正为 `销售合同`
- **AND** 技术协议扫描件 MUST 映射到既有技术协议客户侧成品或扫描件资料编码
- **AND** 销售合同扫描件 MUST 映射到既有销售合同客户侧成品或扫描件资料编码
- **AND** C21/C23 MUST 在签订协议和合同节点完成动作成功后派生为完成
- **AND** 客户退回技术协议或销售合同时，对应 C21 或 C23 扫描件完成结果 MUST 失效，直到重新上传扫描件并完成签订节点
- **AND** 系统 MUST NOT 为准备线和扫描件线新增资料项
- **AND** C20/C22 旧元数据名中的“草稿” MUST 只作为命名修正来源，不得作为最终展示名、上传槽名或 workflow 文案

#### Scenario: C25 回归详细设计项目启动书
- **WHEN** 系统初始化或展示 v20260629 资料清单
- **THEN** C25 / `4.1` MUST 为详细设计阶段 `项目启动书`
- **AND** C25 MUST 归属详细设计 `project_kickoff_meeting` / `项目启动会`
- **AND** C25 责任人和部门 MUST 保持制造中心负责人相关口径
- **AND** C25 MUST NOT 被合同 workflow 预付款最终动作派生完成
- **AND** 后续详细设计 workflow 才负责完成 C25；本 change MUST NOT 开始实现详细设计 workflow

#### Scenario: 合同项目启动通知不改变资料数量
- **WHEN** 系统生成合同项目启动通知文件
- **THEN** 生成文件 MUST 作为合同 workflow generated file 查看形态
- **AND** 生成文件 MUST NOT 被计入新的资料项
- **AND** 生成文件 MUST NOT 改名、完成或占用 C25 / `4.1`
- **AND** 生成文件版本变化 MUST NOT 新增、删除、拆分或改名资料项

#### Scenario: 71 项资料数量不变
- **WHEN** 系统初始化或校验 v20260629 新项目资料模板
- **THEN** 系统 MUST 继续保持 71 项资料数量
- **AND** 合同签订 workflow 不得新增第 72 项资料
- **AND** 合同签订 workflow 不得改变立项、方案设计、生产制作、预验收、终验收或结题阶段资料数量

#### Scenario: C25 不形成合同双入口
- **WHEN** 用户查看合同签订阶段或详细设计阶段资料主流程入口
- **THEN** C25 MUST 保持详细设计阶段项目启动会的 `项目启动书` 资料项
- **AND** C25 MUST NOT 在合同阶段作为独立项目启动通知节点暴露
- **AND** C25 MUST NOT 被合同预付款节点展示为项目启动通知生成结果
- **AND** `4.1` 运行资料编码 MUST NOT 被当作合同阶段非主流程资料入口重复展示

#### Scenario: 预付款发票不成为通用付款流程
- **WHEN** 项目预付款支付节点进入完成支付或总经理放行路径
- **THEN** 既有预付款发票资料项 MUST NOT 被扩展为通用付款审批或发票流转状态机
- **AND** C24 `发票（预付款）` MUST 保持普通/条件性资料项
- **AND** C24 MUST NOT 出现在合同 workflow 3 个主流程节点中，也不得成为第 4 个合同 workflow 节点
- **AND** 本 change 的合同 workflow 页面 MUST NOT 为 C24 提供非主流程资料辅助区入口
- **AND** 后续如需关联预付款发票，MUST 通过独立 change 明确资料责任人、适用性和完成口径
