## ADDED Requirements

### Requirement: 立项资料项数量和在线表单入口保持

阶段资料清单能力 MUST 保持 `1.1 项目需求表`、`1.2 项目立项审批表`、`1.3 项目立项通知` 为原有资料项；本 change MUST NOT 新增、删除或改名资料项，也 MUST NOT 改变 v20260629 / 71 项资料数量。

#### Scenario: 不新增资料项
- **WHEN** 系统处理立项阶段协同填写和文件产出方向
- **THEN** `1.1` MUST 仍是原资料项
- **AND** `1.2` MUST 仍是原资料项
- **AND** `1.3` MUST 仍是原资料项
- **AND** 系统 MUST NOT 因协同填写新增商务子资料或技术子资料

#### Scenario: 71 项数量保持不变
- **WHEN** 系统初始化或校验 v20260629 项目资料模板
- **THEN** 资料数量 MUST 继续保持 71 项
- **AND** 系统 MUST NOT 因模板文件产出增加 Excel、Word 或 PDF 资料项

#### Scenario: 在线表单仍为填写入口
- **WHEN** 用户处理 `1.1`、`1.2` 或 `1.3`
- **THEN** 阶段资料清单 MUST 继续表达这些资料以在线表单为填写入口
- **AND** 普通资料提交、普通审核或普通退回 MUST NOT 绕过立项在线表单和 `1.2` 专用评价审批

### Requirement: 1.2 协同填写不拆分资料项

阶段资料清单能力 MUST 将 `1.2` 商务部分和技术部分视为同一资料项内的协同填写状态，而不是新增两个资料项或第二套资料状态机。

#### Scenario: 商务技术部分属于同一 1.2
- **WHEN** 商务负责人填写基础模块和商务模块
- **AND** 技术负责人填写技术模块
- **THEN** 系统 MUST 仍将这些内容归属于同一个 `1.2 项目立项审批表` 资料项
- **AND** 系统 MUST NOT 创建 `1.2-商务`、`1.2-技术` 或等价新增资料项
- **AND** 系统 MUST NOT 将 `1.2` 当作普通资料责任人分配资料项

#### Scenario: 双方完成后 1.2 才可进入评价
- **WHEN** 阶段资料清单计算 `1.2` 完成或待办状态
- **THEN** 系统 MUST 同时考虑商务部分完成状态和技术部分完成状态
- **AND** 任一部分未完成时 `1.2` MUST NOT 进入现有评价审批节点

### Requirement: 模板文件作为后续产出查看形态

阶段资料清单能力 MUST 将在线表单提交后生成模板文件记录为后续产出查看形态；本 change MUST NOT 生成文件或接入文件平台。

#### Scenario: 后续产出文件不改变资料项
- **WHEN** 后续为 `1.1`、`1.2` 或 `1.3` 生成模板文件
- **THEN** 生成文件 MUST 作为原资料项的产出查看形态
- **AND** 生成文件 MUST NOT 被计入新的资料项数量

#### Scenario: 本 change 不调用文件平台
- **WHEN** 本 change 完成实现
- **THEN** 阶段资料清单 MUST NOT 调用文件管理平台 API
- **AND** 阶段资料清单 MUST NOT 创建文件平台目录、文件映射或归档记录
