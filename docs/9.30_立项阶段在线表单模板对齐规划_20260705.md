# 9.30 立项阶段在线表单模板对齐规划 20260705

## 背景

前两个立项阶段验收已经完成：新项目默认 v20260629 / 71 项模板运行基线已稳定，立项阶段商务/技术负责人、评价退回路径、总经理项目结束路径已经完成并通过复测。下一步需要对齐 `1.1 项目需求表`、`1.2 项目立项审批表`、`1.3 项目立项通知` 的在线表单真实模板。

当前系统的 1.1 / 1.2 / 1.3 在线表单是简化版，适合先跑通流程，但还没有覆盖真实业务模板的字段、分组、评分项和通知正文。该问题需要先通过 OpenSpec 固化字段口径、责任人口径和项目编号门禁调整，再进入后续实现。

## 模板来源

- `智能制造项目管理文件模板/项目需求表-模板.xlsx`
- `智能制造项目管理文件模板/项目立项审批表-模板.xlsx`
- `智能制造项目管理文件模板/关于确定项目名称及编号的通知-模板.docx`

本规划只对齐在线表单内容结构，不把上述模板理解为本轮要生成 Excel、Word、PDF 或文件平台文件。

## 范围

本 change 规划：

- 让 1.1 项目需求表在线表单对齐真实市场调研结构化模板。
- 让 1.2 项目立项审批表在线表单对齐基础信息、商务评分、技术评分和三方意见结构。
- 让 1.3 立项通知在线表单对齐固定标题、固定正文、项目信息表格和落款。
- 调整项目编号顺序：`1.2 总经理最终通过 -> 允许填写/生成项目编号 -> 1.3 自动带出项目编号并提交 -> 立项阶段齐套`。
- 明确旧简化在线表单数据视为模拟/过渡数据：本 change 不迁移旧表单数据，不保证旧简化字段继续在 UI 中按旧模板展示；旧项目或旧数据如需继续处理，返工或重新填写时统一按新 `1.1` / `1.2` / `1.3` 模板填写。

本 change 不做：

- 不生成 Excel / Word / PDF。
- 不接文件平台。
- 不新增资料项。
- 不改变 v20260629 / 71 项模板数量。
- 不新增技术负责人填写任务。
- 不改变当前 1.2 商务评价、技术评价、总经理审批流。
- 不改变项目结束、退回、返工主流程。
- 不改业务代码、不改数据库 schema、不写 migration。

## 责任人口径

- `1.1 项目需求表`：商务负责人填写。
- `1.2 项目立项审批表`：商务负责人填写整张表，包括技术模块评分。
- `1.3 项目立项通知`：营销中心负责人填写/确认。
- 技术负责人第一版只作为项目技术协同/后续承接人，不新增表单填写任务。

## 1.1 项目需求表字段规划

### 基础信息

- 项目名称。
- 客户名称。
- 交流时间。
- 交流次数。
- 交流地点。
- 交流方式。
- 我方人员。
- 甲方人员。

### 环境要求

- 工作温度。
- 储存温度。
- 工作湿度。
- 储存湿度。
- 噪音。
- IP 防护等级。
- 防腐等级。
- 海拔高度。
- 防爆要求。

### 场地情况

- 可用场地尺寸。
- 电源。
- 气源。
- 液压源。
- 吊装设备。

### 工件描述

- 工件外形尺寸。
- 工件质量。
- 工件材质。
- 工件数量。
- 是否有图纸 / 图纸提供说明（对应模板“如有图纸请提供”）。

### 作业工艺

- 做什么。
- 怎么做。
- 是否有工艺文件 / 工艺文件提供说明（对应模板“如有工艺文件请提供”）。

### 目标

- 自动化环节。
- 节拍。
- 人机交互模式。
- 价格。
- 工期。

## 1.2 项目立项审批表字段规划

### 基础信息

- 项目名称。
- 项目号。
- 客户名称。
- 项目联系人。
- 客户联系方式。
- 项目负责人。
- 项目负责人联系方式。

### 商务模块 7 项

- 甲方属性。
- 甲方企业信息。
- 身份角色。
- 公司竞争优势。
- 商务形式及背调。
- 商务关系层级。
- 项目情况。

### 技术模块 4 项

- 特殊环境要求。
- 行业门槛。
- 技术成熟度。
- 可借鉴案例。

每项评分至少包含：

- 条款内容。
- 评价标准。
- 分值 0-5。
- 信息收集说明。
- 责任人。

评分项必须预置真实模板中的条款内容和评价标准文本，不允许只生成空白评分表。信息收集说明列必须按模板保留；当前模板行内未填写说明文本的项，也必须显式保留该字段，不得省略。

## 1.3 项目立项通知字段规划

固定标题：

- 关于确定项目名称及编号的通知。

固定正文：

- 各部门：
- 为便于公司项目生产准备、事前申请、费用填报、成本归集、物资采购等工作开展。现将各项目的项目名称、项目编号确定如下：
- 请各部门严格按照项目名称、项目编号进行生产准备、费用填报、事前申请、成本归集、物资采购等工作。

表格字段：

- 序号。
- 项目编号。
- 项目名称。
- 客户单位。
- 立项日期。

落款：

- 重庆凯尔夫智能测控技术有限责任公司。
- 日期。

第一版是在线通知内容展示和提交，不生成 docx。项目编号、项目名称、客户单位优先从项目主数据带出。立项日期可填写或默认提交日期，具体实现阶段再固化。

## 关键设计决策

- 在线表单 schema 需要支持分组。
- `1.2` 需要支持评分项或表格型结构。
- `1.3` 需要支持只读/自动带出字段和通知预览。
- 旧简化在线表单数据视为模拟/过渡数据。本 change 不迁移旧表单数据，不保证旧简化字段继续在 UI 中按旧模板展示；旧项目或旧数据如需继续处理，返工或重新填写时统一按新 `1.1` / `1.2` / `1.3` 模板填写。
- 现有在线表单数据仍走当前表单数据存储；是否需要 schema 扩展由后续实现阶段确认。
