## 第三部分：1.2 项目立项审批表 — 三节点审批流程

**基本信息：**
- completionMode: `approval_required`
- 审批流程（3 节点）：
  - **节点 1：商务评价审批** — 审批人：营销中心负责人（`marketing_center` / `center_manager`）
  - **节点 2：技术评价审批** — 审批人：研发中心负责人（`rd_center` / `center_manager`）
  - **节点 3：总经理审批** — 审批人：总经理（`general_manager`）
- 节点 1 和节点 2 **可并行审批**
- 节点 3 在节点 1+2 均通过后**自动激活**
- 全部节点通过后，1.2 自动变为 `confirmed`
- 退回时自动标记 1.1 返工

### 1.2 节点状态一览

| 状态值 | 含义 |
|--------|------|
| `waiting_document_submission` | 等待 1.2 资料本身提交 |
| `pending` | 待审批（当前节点可操作） |
| `approved` | 已通过 |
| `returned_blocked_by_rework` | 已退回（阻塞，需 1.1 返工清除后恢复） |
| `waiting_prerequisite` | 等待前置节点完成（仅总经理节点使用） |
| `invalidated` | 已失效（商务/技术退回导致总经理节点失效） |

---

### 场景 3.1：1.2 提交 → 激活商务 + 技术待审

**前置条件：** 项目已创建，立项阶段资料清单已初始化。以项目经理或有提交权限的用户登录。

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 进入项目详情 → 阶段资料清单 | 1.2 状态为 `not_submitted` |

| 3 | 点击"提交 1.2" | 提交成功 |
| 4 | 验证审批面板变化 | 商务评价审批 → `pending`；技术评价审批 → `pending`；总经理审批 → `waiting_prerequisite` |
| 5 | 验证操作日志 | 记录"提交资料并启动 1.2 多节点审批" |

## 第九部分：1.2 多节点审批完整状态转移矩阵

### 节点状态转移规则

| 从状态 | 触发操作 | 到状态 | 附加效果 |
|--------|---------|--------|---------|
| `waiting_document_submission` | 1.2 资料提交 | `pending`(商务+技术) | `waiting_prerequisite`(总经理) |
| `pending` | 商务/技术审批通过 | `approved` | 如果并行节点均已通过 → 总经理自动从 `waiting_prerequisite` → `pending` |
| `pending` | 总经理审批通过 | `approved` | 如果全部 3 节点通过 → 1.2 自动 `confirmed` |
| `pending` | 商务/技术退回 | `returned_blocked_by_rework` | 1.1 标记返工；总经理 invalidate（如非等待中） |
| `pending` | 总经理退回 | `returned_blocked_by_rework` | 1.1 标记返工；商务/技术保持 approved |
| `returned_blocked_by_rework` | 1.1 返工清除 | `pending` | 如所有退回节点恢复 → 可继续审批 |
| `waiting_prerequisite` | 商务+技术均通过 | `pending` | 自动触发 |
| `invalidated` | 商务+技术再次审批通过 | 需重新检查 `activateGeneralReviewIfReady` | — |
