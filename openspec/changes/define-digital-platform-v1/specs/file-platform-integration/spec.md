## ADDED Requirements

### Requirement: 项目文件夹树创建

项目在数字化管理平台创建后，系统 MUST（必须）调用文件管理平台创建或绑定项目文件夹树。

#### Scenario: 成功创建项目文件夹树

- **WHEN** 项目在数字化管理平台创建成功
- **THEN** 系统必须请求文件管理平台创建项目文件夹树，并保存返回的文件夹映射关系

#### Scenario: 文件夹树创建失败

- **WHEN** 文件管理平台无法创建项目文件夹树
- **THEN** 系统必须保留项目记录，将文件夹同步状态标记为失败，并允许有权限用户重试同步

### Requirement: 资料文件归档上传

系统 MUST（必须）通过文件管理平台归档已提交表单和上传文件，最终归档文件不能只保存在数字化管理平台中。

#### Scenario: 归档表单生成文件

- **WHEN** 标准表单提交并生成归档文件
- **THEN** 系统必须将生成文件上传到该项目、阶段和资料项对应的文件平台文件夹

#### Scenario: 归档上传文件

- **WHEN** 用户从数字化管理平台上传非模板文件
- **THEN** 系统必须将文件上传到文件管理平台映射的目标文件夹

### Requirement: 来源元数据传递

数字化管理平台归档任何文件时，系统 MUST（必须）向文件管理平台传递来源元数据。

#### Scenario: 归档时传递元数据

- **WHEN** 数字化管理平台归档文件
- **THEN** 系统必须传递来源系统、来源动作、来源用户 ID、来源用户姓名、来源部门、项目编号、项目名称、阶段标识、阶段名称、资料项标识、资料项名称和目标文件夹 ID

#### Scenario: 保留真实提交人

- **WHEN** 文件由平台集成账号代替用户归档
- **THEN** 文件管理平台日志必须能通过传递的元数据识别真实提交人

### Requirement: 文件状态查询

系统 MUST（必须）查询文件平台中的文件列表和归档状态，用于展示项目阶段和资料项文件状态。

#### Scenario: 查询资料项文件

- **WHEN** 用户打开已有归档文件的资料项
- **THEN** 系统必须获取或展示该资料项关联的文件列表

#### Scenario: 刷新阶段齐套情况

- **WHEN** 系统重新计算阶段资料齐套情况
- **THEN** 系统必须结合数字化平台资料状态和文件平台归档状态判断必需文件是否存在

### Requirement: 通过文件平台下载

系统 MUST（必须）将文件查看按文件下载处理，并必须使用文件管理平台的下载能力和权限判断。

#### Scenario: 下载有权限文件

- **WHEN** 用户请求查看或下载已归档文件
- **THEN** 系统必须向文件管理平台请求下载，并且只有文件平台授权后才允许下载

#### Scenario: 下载被文件平台拒绝

- **WHEN** 文件管理平台拒绝下载请求
- **THEN** 系统必须展示权限失败结果，并且不能绕过文件平台权限判断

### Requirement: 集成边界

系统 MUST（必须）将项目过程逻辑保留在数字化管理平台，将文件存储逻辑保留在文件管理平台。

#### Scenario: 文件平台收到归档请求

- **WHEN** 文件管理平台收到数字化管理平台的归档请求
- **THEN** 文件管理平台必须存储文件、应用文件权限并记录文件日志，且不需要理解完整 8 阶段项目流程

#### Scenario: 数字化平台展示文件状态

- **WHEN** 数字化管理平台展示资料状态
- **THEN** 数字化管理平台必须使用文件平台文件状态，但不能复制文件权限规则
