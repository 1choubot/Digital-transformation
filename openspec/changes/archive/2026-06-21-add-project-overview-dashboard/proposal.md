## 为什么

当前平台已经具备项目、阶段、阶段资料清单、齐套摘要、责任人分配和我的资料任务，但用户仍需要逐个进入项目详情才能了解跨项目进展。第一版“项目总览看板”用于提供跨项目查看入口，让管理者和项目负责人快速看到项目数量、风险状态、当前阶段齐套情况和个人待办资料数量。

该能力基于现有项目、阶段、资料齐套和我的资料任务数据做只读汇总展示，不新增业务状态，也不改变阶段推进、资料状态或权限规则。

## 变更内容

- 新增 `GET /api/projects/overview-dashboard` 后端查询接口。
- 接口必须要求登录态，只做 `requireAuth`，不要求 `requirePlatformAdmin`。
- 第一版返回所有项目的总览数据，不做项目成员权限、项目经理权限、复杂权限或轻角色校验。
- 返回汇总指标：`totalProjects`、`activeProjects`、`completedProjects`、`riskProjects`、`myPendingStageDocumentTasks`。
- 返回项目总览卡片/列表数据，包含项目基础字段、当前阶段字段、当前阶段齐套摘要、当前阶段未完成适用必填资料列表、创建人和计划时间。
- 当前阶段异常缺失、当前阶段资料清单未初始化等情况不得导致接口崩溃，必须通过 `currentStageIssue` 等可读字段展示问题。
- 复用现有阶段齐套摘要口径，只统计当前阶段适用必填资料；`requiredTotal = 0` 时完成百分比为 100。
- 支持基础筛选：`status`、`currentStageOrder`、`keyword`。
- 非法 `status` 返回 `INVALID_PROJECT_STATUS_FILTER`，非法 `currentStageOrder` 返回 `INVALID_STAGE_ORDER`。
- 第一版不做分页，按后端固定规则稳定排序。
- 前端新增“项目总览”页面和主导航入口，展示汇总指标、项目总览列表/卡片、筛选控件和项目详情跳转。
- “我的待办资料”指标可点击进入我的资料任务页。
- README 补充接口、筛选规则、页面能力和边界说明。
- 明确第一版不做文件上传/下载、文件平台联动、在线表单、消息提醒、超期提醒、截止日期、大屏图表、复杂权限、分页、导出、批量操作、业务日志、阶段推进规则修改或历史项目资料清单自动初始化。

## 能力范围

### 新增能力

- 无。本变更基于现有项目核心、阶段资料清单和项目核心前端能力扩展，不新增独立 capability。

### 修改能力

- `project-core`: 增加项目总览看板后端查询接口、汇总指标、项目卡片字段、筛选、排序和只读边界。
- `stage-document-checklist`: 增加项目总览看板复用当前阶段适用必填资料齐套摘要的口径和异常资料清单处理要求。
- `project-core-frontend`: 增加“项目总览”页面、主导航入口、汇总指标展示、项目卡片/列表、筛选和跳转要求。

## 影响

本变更影响后端项目查询路由、项目/阶段/资料查询仓储、前端路由/导航、API 客户端、项目总览页面和 README。无需新增数据库表或迁移，不改变现有项目创建、阶段推进、资料状态流转、适用性、责任人分配、我的资料任务和业务日志写入逻辑。
