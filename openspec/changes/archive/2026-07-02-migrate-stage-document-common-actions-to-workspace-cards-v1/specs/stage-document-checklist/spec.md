## ADDED Requirements

### Requirement: 旧资料清单降级为兼容区

阶段资料清单能力 MUST 支持从项目详情页主操作入口降级为兼容区；当上方产出卡片承载现有资料通用操作后，下方旧资料清单 MUST NOT 与上方产出卡片形成两个主入口。

#### Scenario: 迁移前旧资料清单仍可操作
- **WHEN** 某项通用资料操作尚未迁移到上方产出卡片
- **THEN** 下方旧资料清单 MUST 继续保持可访问和可操作
- **AND** 系统 MUST 继续使用现有资料状态、权限、附件和业务日志

#### Scenario: 迁移中旧资料清单显示兼容提示
- **WHEN** 某项通用资料操作已经迁移到上方产出卡片
- **THEN** 下方旧资料清单 MUST 降级为辅助或兼容区域
- **AND** 前端 MUST 提示用户以上方产出卡片为主入口，避免上下两个主入口并存
- **AND** 对已迁移到上方产出卡片的资料，下方旧资料清单 MUST NOT 再展示责任人分配/清空、附件上传、附件删除、提交资料、审核通过、审核退回、返工重提/返工完成、标记不适用或恢复适用等主操作按钮
- **AND** 下方旧资料清单 MAY 保留资料状态、责任人只读展示、附件摘要或附件只读列表、阻塞原因、兼容提示，以及“到上方产出卡片处理”的定位入口

#### Scenario: 旧资料清单读取同一套后端状态
- **WHEN** 用户从上方产出卡片或下方旧资料清单查看同一资料
- **THEN** 两处 MUST 展示同一套后端资料状态、责任人、附件、适用性、阻塞原因和权限结果
- **AND** 系统 MUST NOT 为下方旧资料清单保留第二套完成状态

#### Scenario: 删除前必须完成覆盖验收
- **WHEN** 团队准备隐藏或删除下方旧资料清单
- **THEN** 团队 MUST 先确认上方产出卡片覆盖全部通用操作、权限原因、状态说明、阻塞原因和验收项
- **AND** 未覆盖前 MUST NOT 删除旧资料清单组件

#### Scenario: 本 change 不隐藏或删除旧清单组件
- **WHEN** 本 change 处于第一版迁移阶段
- **THEN** 系统 MUST NOT 因本 change 隐藏或物理删除旧资料清单组件
- **AND** 系统 MUST NOT 完全移除旧资料清单入口
- **AND** 隐藏或删除旧清单 MUST 通过后续独立 change 明确确认
