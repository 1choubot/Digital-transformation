## 2. 字段说明

| 字段 | 说明 |
|---|---|
| 资料编号 | 与文件管理平台目录编号保持一致 |
| 资料项 | 第一版阶段资料名称 |
| 是否必需 | “是”为阶段齐套率统计项；“建议”为可选或视项目情况补充 |
| 提交方式 | 在线表单、文件上传、在线表单+文件上传 |
| 默认责任角色 | 第一版默认责任归口，具体责任人由项目经理分配 |
| 确认角色 | 负责确认/退回资料的角色 |
| 文件平台目标目录 | 文件管理平台中的目标目录名称 |

## 4. 资料状态建议

| 状态 | 说明 |
|---|---|
| 待提交 | 资料项已生成，但责任人尚未保存草稿或上传文件 |
| 草稿 | 在线表单已保存草稿，尚未提交 |
| 已提交 | 责任人已提交表单或上传文件，等待确认或归档状态刷新 |
| 归档中 | 数字化平台已请求文件平台归档，等待返回结果 |
| 已归档 | 文件平台已返回文件记录或归档成功状态 |
| 待确认 | 资料已提交/归档，等待项目经理或部门负责人确认 |
| 已确认 | 确认人已通过资料项，计入齐套 |
| 已退回 | 确认人退回资料项，责任人需重新提交 |
| 归档失败 | 调用文件平台失败或未获得目标目录/文件记录 |

## 5. 开发落地提示

- 数字化平台中的资料项应保存 `stage_key`、`document_key`、`required`、`submit_mode`、`default_role`、`confirm_role`、`target_folder_path`。
- 资料责任人是项目级分配结果，不应写死为固定姓名。
- 文件平台目录编号和资料编号应保持一致，便于目录映射、文件归档和问题排查。
- 资料齐套率建议只统计“是否必需 = 是”的资料项；“建议”资料可展示但不阻断阶段齐套。
