## 背景 / 原因

当前阶段资料清单已经能按项目展示资料项和基础状态，但状态只能初始化和只读展示，无法在不接入文件上传、在线表单和文件平台的情况下验证资料提交流程。需要先建立一个最小的手工状态流转闭环，让项目详情页可以标记提交、确认和退回，并为后续真实表单、上传、齐套率和阶段推进提供状态基础。

## 变更内容

- 为项目级阶段资料项新增受控状态操作：标记提交、确认、退回。
- 后端统一校验资料项状态机：
  - `not_submitted` / `returned` -> `submitted`
  - `submitted` -> `confirmed`
  - `submitted` -> `returned`
- 所有状态变更接口必须要求登录态，但本变更不做复杂权限、角色权限或轻角色校验。
- 资料项状态变更保存最小追溯字段：提交人/时间、确认人/时间、退回人/时间和退回原因。
- 已退回资料项重新标记提交时，清空本次退回原因、退回人和退回时间。
- 阶段资料清单查询返回提交、确认、退回追溯字段。
- 项目详情页阶段资料清单提供手工标记提交、确认、退回和退回原因输入，操作成功后刷新清单。
- 页面文案必须明确该能力只是“手工标记状态”，不代表已经上传文件或提交在线表单。
- 本变更不实现文件上传/下载、文件平台联动、在线表单、表单草稿、表单归档、资料齐套率、阶段推进、业务日志、责任人分配、个人待办、复杂权限、角色权限或轻角色校验。

## 能力范围

### 新增能力

- 无。

### 修改能力

- `stage-document-checklist`: 增加项目级资料项状态流转接口、状态机校验、状态追溯字段和查询返回字段。
- `project-core-frontend`: 扩展项目详情页阶段资料清单，从只读状态展示变为可手工标记资料项状态，并展示状态追溯信息。

## 影响范围

- 后端影响 `digital-platform-api` 的项目阶段资料清单查询、资料项状态更新路由、资料项仓储和数据库迁移。
- 前端影响 `digital-platform-web` 的项目详情页、项目 API 客户端、状态操作错误提示和资料清单刷新。
- 需要为 `project_stage_documents` 增加最小追溯字段，但不引入新文件存储、文件平台 API、业务日志或权限体系。
