## ADDED Requirements

### Requirement: 20260629 71 项候选模板确认边界

阶段资料清单能力 MUST 将 20260629 PDF 图面产出作为下一版目标资料模板基线，并 MUST 将图面 67 项产出 + 4 个流程图漏画草稿产出形成的 71 项资料清单视为候选模板确认输入；当前系统 MUST 继续以 20260625 64 项资料模板作为运行基线，除非后续独立 change 修改模板、初始化和迁移策略。

#### Scenario: 71 候选不改变当前初始化
- **WHEN** 系统创建项目、补初始化项目级资料清单或校验当前 active 模板
- **THEN** 系统 MUST 继续使用当前 20260625 64 项资料模板
- **AND** 系统 MUST NOT 因 71 项候选清单新增、删除、合并、改名、移动或重编号任何项目级资料项

#### Scenario: 64 67 71 口径必须区分
- **WHEN** 团队评审 20260629 资料模板候选
- **THEN** 团队 MUST 将 64 解释为当前系统运行模板
- **AND** 团队 MUST 将 67 解释为 20260629 PDF 图面红色产出统计和下一版目标基线
- **AND** 团队 MUST 将 71 解释为在 20260629 图面 67 项基础上补 4 个草稿产出后的目标候选
- **AND** 系统 MUST NOT 将 67 或 71 作为当前 active 模板数量

#### Scenario: 20260625 只作为运行和迁移对照
- **WHEN** 团队确认下一版目标资料模板
- **THEN** 团队 MUST 将 20260625 64 项仅作为当前运行模板和迁移对照
- **AND** 20260625 有但 20260629 没有的产出 MUST NOT 自动进入下一版目标模板
- **AND** 如业务要求保留这些产出，MUST 通过后续独立 change 明确保留理由、编号、阶段、completionMode 和迁移影响

#### Scenario: 新增候选必须独立确认
- **WHEN** 候选清单包含 `投标书`、`技术协议草稿（合同签订阶段）`、`销售合同草稿`、`技术协议草稿（生产制作阶段）`、`技术协议（生产制作阶段/供应商侧成品）`、`采购合同草稿`、`合格供应商评价表`、`生产记录表` 或 `资料移交清单`
- **THEN** 系统 MUST NOT 自动将这些候选加入阶段资料清单
- **AND** 团队 MUST 通过后续独立 change 确认资料编号、阶段、责任中心、必填性、适用条件、completionMode、权限和迁移影响

#### Scenario: 旧合同审核资料不得在本规划中删除
- **WHEN** 团队评审 `合同审核记录表（销售合同）` 或 `采购合同审核记录表`
- **THEN** 团队 MUST 将其标记为迁移对照和继续独立保留或并入对应草稿资料审核过程的业务确认项
- **AND** 系统 MUST NOT 因本规划 change 删除、隐藏、迁移或改名这两个当前运行期资料项
- **AND** 团队 MUST NOT 将这两个旧审核资料作为 71 项目标候选口径的核心推导依据

#### Scenario: 草稿和成品按阶段语义区分
- **WHEN** 后续实现技术协议、销售合同或采购合同相关候选模板
- **THEN** 团队 MUST 按阶段和业务语义区分草稿产出与成品产出
- **AND** 团队 MUST NOT 仅按中文名称合并 `技术协议草稿`、`技术协议` 或其他同名候选

#### Scenario: completionMode 变化必须独立实现
- **WHEN** 71 项候选清单建议某资料可能新增、合并、改名、阶段移动、必填性变化或 completionMode 变化
- **THEN** 系统 MUST 继续使用当前资料模板已定义的 completionMode
- **AND** 任何运行时 completionMode 调整 MUST 通过后续独立 change 补齐规格、实现、smoke、迁移策略和验收

#### Scenario: 第一版目标模板默认文件上传
- **WHEN** 团队确认 71 项候选资料的第一版实现方式
- **THEN** 除当前已实现的立项阶段 `1.1 项目需求表`、`1.2 项目立项审批表` 和 `1.3 项目立项通知` 继续沿用在线表单外，其他候选资料 MUST 默认按文件上传或附件上传能力处理
- **AND** 系统 MUST NOT 因本候选规划自动新增在线表单、专用审批流或复杂状态机
- **AND** 哪些资料需要在线表单、专用审批或复杂状态机 MUST 在后续逐阶段 implementation change 中确认

#### Scenario: 成本估算表为真实多节点协作例外
- **WHEN** 团队评审方案设计阶段成本估算和价格估算节点
- **THEN** 团队 MAY 将 `成本估算表` 保留为多人或多节点协作同一产出
- **AND** 该例外 MUST NOT 扩展为技术协议、销售合同或采购合同的准备/签订伪多对一规则
