# 9.18 立项阶段责任人分配权限修复

## 背景

立项阶段工作区已将 `1.1 项目需求表`、`1.2 项目立项审批表`、`1.3 项目立项通知` 收敛为在线表单产出。当前 review 发现 `1.1 / 1.2` 的责任人分配权限在后端返回字段与实际保存接口校验之间存在不一致：前端依据 `permissions.canManageResponsibility` 展示按钮，但保存接口可能采用另一套全局责任人分配规则。

## 修复目标

- `1.1` 和 `1.2` 的责任人分配权限统一为营销中心负责人专用规则。
- 阶段资料清单、项目工作区产出、工作台等通用权限字段返回的 `canManageResponsibility` 必须与责任人保存/清空接口的实际校验一致。
- `1.3` 不展示、不要求、不依赖单独责任人分配，默认由营销中心负责人处理。
- 不放宽其他阶段资料责任人分配权限，不改在线表单提交责任人规则。

## 权限口径

`1.1 / 1.2` 责任人分配允许条件：

- 资料编号为 `1.1` 或 `1.2`。
- 当前用户 `organizationRole = center_manager`。
- 当前用户 `department = marketing_center`。
- 当前用户不是 `system_admin`。
- 当前用户不是 `general_manager_assistant`。

明确不允许：

- 总经理助理。
- 系统管理员。
- 研发中心负责人。
- 非营销中心负责人。
- 仅因项目经理、创建人、总经理或项目可查看身份获得分配权。

## 实施边界

- 不新增数据库字段或 migration。
- 不处理第二阶段补录。
- 不处理 `file-platform-integration-v1` 或 `define-digital-platform-v1`。
- 不改变 `1.1 / 1.2` 在线表单提交权限：仍只有被分配责任人可填写/提交。
- 不改变 `1.3` 在线表单处理人规则。
