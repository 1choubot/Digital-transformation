# 阶段资料清单与责任角色表（20260610 草案）

版本：v0.2 草案  
来源：`智能制造项目管理流程图20260610.pdf`  
整理日期：2026-06-23  
适用范围：数字化管理平台后续资料模板升级、项目角色梳理、日报周报和文件平台联动前置分析

## 1. 编制说明

本文件用于承接领导更新的《智能制造类项目管理流程》产出文件清单，避免在后续长上下文讨论中丢失新版资料口径。

本文件暂不替换现有 `docs/9.2_阶段资料清单与责任角色表.md`。当前系统仍基于旧版 48 项资料模板运行，后续如要落地新版清单，应单独创建 OpenSpec change，并处理新项目模板、历史项目补初始化、齐套率、阶段推进门禁、责任人分配和文件平台目录映射。

### 2.4 责任归属变化

新版流程图对“谁组织、谁编写、谁汇总”描述更细：

- 项目需求表由营销人员填写。
- 项目立项审批表由营销人员填写，线下审批签字后扫描上传。
- 项目立项通知由营销总监编写，并发布至公司工作总群。
- 项目组成员由研发总监指定。
- 项目经理是项目第一责任人，并拥有项目管理最大权限。
- 方案设计工作计划、详细设计工作计划由项目经理制定。
- 项目经理和技术负责人/技术经理可以是同一人。
- 项目工作群由项目经理建立，群名为项目编号加项目名称。
- 成本估算表由各专业组成员填写本人部分成本估算零部件清单，技术负责人汇总后发送制造中心。
- 图纸审查由项目经理按流程组织，图纸审查人员由研发总监指定。
- 图纸审查不能代替图纸签字流程。
- 图纸和零部件清单一旦下发，产品修改需通过技术通知单执行。
- 预验收、终验收由营销中心按工作计划组织，制造中心人员填写对应验收资料。

## 4. 待确认问题

以下内容需要与领导或流程负责人确认后再进入正式模板：

1. 新版是否保留旧版 `客户输入文件`，还是其内容并入 `项目需求表` 或 `项目需求分析表`。
2. 旧版多阶段出现的 `产品反馈跟踪表` 是否取消，还是仍作为制造/调试问题闭环资料保留。
3. 旧版 `技术专项研究` 是否取消，还是作为非必需资料保留在详细设计阶段。
4. `方案评审记录表` 是否拆成内部方案评审和客户方案评审两项，还是同一资料项支持多轮/多类型记录。
5. `设计评审记录表` 是否拆成内部设计评审和客户设计评审两项，还是同一资料项支持多轮/多类型记录。
6. `安装调试记录` 在预验收阶段和终验收阶段是否使用同一模板，还是分别使用厂内安装调试和现场安装调试模板。
7. `UI界面设计PPT` 在新版流程图中为红色产出，是否从旧版“建议”改为必填。
8. `技术通知单` 在新版流程图中为红色产出，是否从旧版“建议”改为必填。
9. 新版生产制作阶段的采购、检验、入库、领料资料是否全部纳入阶段推进齐套门禁。
10. 是否需要为一个资料项支持多个协作子任务，例如成本估算表由各专业组成员分别填写，技术负责人汇总。

## 5. 对系统落地的影响

如采用本草案作为新版模板，至少需要处理以下系统影响：

- 阶段资料模板版本需要从当前 `v1` 升级为 `v2` 或其他稳定版本号。
- 后端当前 `EXPECTED_STAGE_DOCUMENT_ITEM_COUNT = 48` 需要调整。
- 新建项目应初始化新版资料项。
- 历史项目不能直接删除旧资料项，应提供补初始化或模板升级策略。
- 新增资料项会影响齐套率、缺失必填资料列表、阶段推进门禁、我的资料任务和项目总览。
- 改名资料项需要考虑旧数据展示、文件平台目录映射和业务日志可读性。
- 必填变建议的资料项会影响阶段推进结果，必须先确认口径。
- 责任归属变细后，需要引入项目内角色或资料项协作子任务，不能只依赖用户表中的部门和角色字符串。
