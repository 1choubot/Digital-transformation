## 2. 联动边界

第一版采用弱绑定方案：

- 数字化管理平台负责项目、阶段、资料项、表单、业务权限、业务日志和看板。
- 文件管理平台负责项目文件夹树、文件真实存储、文件下载、文件权限、文件搜索和文件日志。
- 数字化管理平台不复制文件平台复杂权限；查看文件按下载处理，由文件管理平台判断权限。
- 数字化平台归档文件时，必须传递真实提交人、项目、阶段、资料项和来源动作，避免文件日志只显示“平台上传”。

## 3. 现有文件平台能力可复用接口

以下接口来自文件管理平台当前能力说明，第一版可优先复用或封装。

| 能力 | 当前接口 | 方法 | 当前说明 | 数字化平台使用方式 |
|---|---|---|---|---|
| 登录 | `/api/login` | POST | 支持用户名或姓名登录 | 如两个平台暂不单点登录，可由适配层使用服务账号或映射用户 |
| 当前用户 | `/api/me` | GET | 返回用户、部门和角色 | 下载或用户映射时可校验文件平台身份 |
| 文件夹列表 | `/api/folders` | GET | 按角色返回可见文件夹 | 查询项目目录树、阶段目录映射 |
| 创建项目/文件夹 | `/api/folders` | POST | `is_project=1` 创建项目；带 `parent_id` 创建子文件夹 | 可封装为“创建项目文件夹树”能力 |
| 文件夹详情 | `/api/folders/:id` | GET | 获取文件夹详情 | 校验目标目录存在 |
| 上传文件 | `/api/files/upload` | POST | 上传到末级目录 | 可封装为“归档资料文件”能力 |
| 文件列表 | `/api/files/:folderId` | GET | 查询某文件夹下文件 | 查询资料项已归档文件 |
| 搜索文件 | `/api/files/search` | GET | 支持 `projectId` 筛选 | 查询项目文件，辅助归档检查 |
| 下载文件 | `/api/files/:id/download` | GET | 下载并记录日志 | 数字化平台“查看文件”时调用 |
| 查询日志 | `/api/admin/logs` | GET | 支持用户、动作、时间筛选 | 如权限允许，可用于追溯文件操作 |

## 4. 建议新增或封装的联动接口

### 4.1 创建项目文件夹树

| 项目 | 内容 |
|---|---|
| 建议接口 | `POST /api/integrations/digital/projects/folder-tree` |
| 调用方 | 数字化管理平台 |
| 承担方 | 文件管理平台或文件平台适配层 |
| 触发时机 | 数字化平台创建项目成功后 |
| 主要作用 | 按标准项目资料目录创建项目根目录、8 阶段目录和资料项目录 |

请求字段建议：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `project_code` | string | 是 | 项目编号，例如 `KRF26030` |
| `project_name` | string | 是 | 项目名称 |
| `customer_name` | string | 是 | 客户名称 |
| `department_code` | string | 是 | 文件平台部门编码，第一版建议研发中心 `RD` |
| `created_by_user_id` | string/number | 是 | 数字化平台操作人 ID |
| `created_by_user_name` | string | 是 | 数字化平台操作人姓名 |
| `source_system` | string | 是 | 固定 `digital_project_platform` |
| `source_action` | string | 是 | 固定 `create_project` |

响应字段建议：

| 字段 | 类型 | 说明 |
|---|---|---|
| `project_folder_id` | number | 文件平台项目根目录 ID |
| `project_folder_path` | string | 项目根目录路径 |
| `stage_folders` | array | 阶段目录映射 |
| `document_folders` | array | 资料项目录映射 |
| `sync_status` | string | `success`、`partial_failed`、`failed` |
| `message` | string | 错误或提示信息 |

### 4.2 查询项目目录映射

| 项目 | 内容 |
|---|---|
| 建议接口 | `GET /api/integrations/digital/projects/:projectCode/folder-tree` |
| 主要作用 | 数字化平台查询项目、阶段、资料项对应的文件平台目录 |
| 使用场景 | 项目详情、阶段工作台、归档前确认目标目录 |

查询参数建议：

| 参数 | 必填 | 说明 |
|---|---|---|
| `stage_key` | 否 | 只查询某阶段 |
| `document_key` | 否 | 只查询某资料项 |

响应字段建议：

| 字段 | 类型 | 说明 |
|---|---|---|
| `project_code` | string | 项目编号 |
| `project_folder_id` | number | 项目根目录 ID |
| `stage_folders` | array | 阶段目录 |
| `document_folders` | array | 资料项目录 |

| 使用场景 | 表单提交后归档、非模板文件上传后归档 |

请求字段建议：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `file` | file | 是 | 归档文件 |
| `target_folder_id` | number | 是 | 文件平台目标目录 ID |
| `project_code` | string | 是 | 项目编号 |
| `project_name` | string | 是 | 项目名称 |
| `stage_key` | string | 是 | 阶段标识 |
| `stage_name` | string | 是 | 阶段名称 |
| `document_key` | string | 是 | 资料项标识 |
| `document_name` | string | 是 | 资料项名称 |
| `source_system` | string | 是 | 固定 `digital_project_platform` |
| `source_action` | string | 是 | `submit_form`、`upload_file`、`archive_generated_file` |
| `source_user_id` | string/number | 是 | 数字化平台真实操作人 ID |
| `source_user_name` | string | 是 | 数字化平台真实操作人姓名 |
| `source_department` | string | 是 | 数字化平台真实操作人部门 |
| `form_submission_id` | string/number | 否 | 在线表单提交记录 ID |
| `business_log_id` | string/number | 否 | 数字化平台业务日志 ID |

响应字段建议：

| 字段 | 类型 | 说明 |

### 4.6 获取下载链接或触发下载

| 项目 | 内容 |
|---|---|

| 主要作用 | 数字化平台“查看文件”时调用文件平台下载能力 |
| 权限口径 | 文件管理平台判断下载权限 |

请求头或参数建议：

| 字段 | 必填 | 说明 |
|---|---|---|
| `file_platform_user_id` | 建议 | 文件平台用户映射 ID |
| `source_user_id` | 是 | 数字化平台用户 ID |
| `source_user_name` | 是 | 数字化平台用户姓名 |
| `source_system` | 是 | 固定 `digital_project_platform` |
| `source_action` | 是 | 固定 `download_file` |
| `business_log_id` | 否 | 数字化平台业务日志 ID |

响应建议：

| 场景 | 响应 |
|---|---|
| 有权限 | 返回文件流，或返回一次性下载链接 |
| 无权限 | 返回权限失败状态和原因 |

## 5. 来源元数据标准

| 字段 | 示例 | 说明 |
|---|---|---|
| `source_system` | `digital_project_platform` | 来源系统 |
| `source_action` | `submit_form`、`upload_file`、`archive_generated_file`、`download_file` | 来源动作 |
| `source_user_id` | `123` | 数字化平台用户 ID |
| `source_user_name` | `张三` | 真实操作人 |
| `source_department` | `研发中心` | 真实操作人部门 |
| `project_code` | `KRF26030` | 项目编号 |
| `project_name` | `某自动化项目` | 项目名称 |
| `stage_key` | `detailedDesign` | 阶段标识 |
| `stage_name` | `详细设计阶段` | 阶段名称 |
| `document_key` | `4.12` | 资料项标识或编号 |
| `document_name` | `设计评审记录表` | 资料项名称 |
| `target_folder_id` | `456` | 文件平台目标目录 ID |
| `business_log_id` | `789` | 数字化平台业务日志 ID |

## 6. 错误状态建议

| 错误码 | 场景 | 数字化平台处理建议 |
|---|---|---|
| `FOLDER_NOT_FOUND` | 目标目录不存在 | 标记归档失败，允许项目经理或管理员重试同步目录 |
| `PERMISSION_DENIED` | 文件平台拒绝下载或上传 | 展示权限失败，不绕过文件平台权限 |
| `ARCHIVE_FAILED` | 上传文件失败 | 资料项标记为归档失败，保留用户提交记录 |
| `FILE_NOT_FOUND` | 文件不存在 | 展示文件不存在或已被删除 |
| `USER_MAPPING_MISSING` | 未找到文件平台用户映射 | 可提示联系管理员维护映射；第一版可由服务账号归档但必须记录真实提交人 |
| `PARTIAL_FOLDER_CREATED` | 项目目录部分创建成功 | 保存已返回映射，允许重试补齐 |

| P0 | 创建项目文件夹树 | 是 |

## 8. 开发落地提示

- 如果不直接改造文件管理平台原接口，建议在文件平台服务端增加 `/api/integrations/digital/*` 适配层。
- 第一版可先用服务账号完成归档，但请求和日志必须保存真实提交人元数据。
- 数字化平台应保存文件平台返回的 `folder_id`、`file_id`、`log_id`，用于状态刷新和追溯。
- 文件查看不做在线预览，统一按下载处理。
