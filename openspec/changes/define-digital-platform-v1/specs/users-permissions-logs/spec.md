## ADDED Requirements

### Requirement: 数字化平台用户

系统 MUST（必须）维护数字化平台用户，用户信息至少包括账号、姓名、部门、平台角色、启用状态和可选的文件平台用户 ID 映射。

#### Scenario: 创建启用用户

- **WHEN** 管理员创建用户
- **THEN** 系统必须保存该用户的部门、平台角色、启用状态和可选文件平台用户 ID

#### Scenario: 禁用用户

- **WHEN** 管理员禁用用户
- **THEN** 系统必须阻止该用户执行新的业务操作，同时保留历史日志

### Requirement: 第一版业务角色

系统 MUST（必须）支持第一版业务角色，至少包括管理层、项目经理、部门负责人、普通成员和系统管理员。

#### Scenario: 管理层查看看板

- **WHEN** 管理层用户打开数字化管理平台
- **THEN** 系统必须允许其访问业务规则允许的全部项目总览和看板信息

#### Scenario: 普通成员查看分配工作

- **WHEN** 普通成员打开数字化管理平台
- **THEN** 系统必须展示该成员被分配的资料工作和允许查看的项目信息

### Requirement: 业务权限控制

系统 MUST（必须）控制项目创建、项目维护、资料分配、表单编辑、文件上传、资料确认、资料退回、阶段推进、看板查看和平台配置等业务权限。

#### Scenario: 无权限推进阶段

- **WHEN** 没有阶段推进权限的用户尝试推进项目阶段
- **THEN** 系统必须拒绝该操作，并且不能记录阶段已推进

#### Scenario: 有权限上传文件

- **WHEN** 具备上传权限的用户向分配给自己的资料项提交文件
- **THEN** 系统必须接受该业务操作，并继续执行文件平台归档

### Requirement: 文件权限分离

系统 MUST（必须）将数字化平台业务权限与文件管理平台查看/下载权限分开处理。

#### Scenario: 业务用户查看资料状态

- **WHEN** 用户具备查看某项目资料状态的业务权限
- **THEN** 系统必须展示资料状态，但不能因此授予文件下载权限

#### Scenario: 用户没有文件下载权限

- **WHEN** 用户可以查看资料状态，但文件管理平台拒绝下载
- **THEN** 系统必须遵守文件平台的拒绝结果

### Requirement: 业务操作日志

系统 MUST（必须）记录项目创建、项目修改、责任人分配、表单草稿保存、表单提交、文件上传、资料确认、资料退回、阶段推进、项目暂停、项目恢复和下载请求等业务日志。

#### Scenario: 记录表单提交日志

- **WHEN** 用户提交标准表单
- **THEN** 系统必须记录操作人、操作类型、项目、阶段、资料项、时间和结果

#### Scenario: 记录阶段推进日志

- **WHEN** 项目经理推进阶段
- **THEN** 系统必须记录操作人、原阶段、目标阶段、资料齐套状态、时间和结果

### Requirement: 跨平台追溯

系统 MUST（必须）将数字化平台业务日志与文件管理平台文件日志关联起来，用于追溯从数字化平台归档或下载的文件。

#### Scenario: 追溯归档文件

- **WHEN** 管理员查看从数字化平台提交的归档文件
- **THEN** 系统必须提供足够元数据，用于识别业务操作、真实提交人、项目、阶段、资料项和文件平台日志记录

#### Scenario: 追溯下载请求

- **WHEN** 用户从数字化平台请求下载文件
- **THEN** 系统必须记录业务侧下载请求，并依赖文件管理平台记录文件侧下载结果
