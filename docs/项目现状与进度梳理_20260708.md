# 数字化管理平台 — 前后端代码现状与进度梳理

> 梳理时间：2026-07-08
> 范围：`digital-platform-api`（后端）+ `digital-platform-web`（前端）+ `openspec/` + `docs/`
> 目的：理解项目当前情况、已实现功能、技术架构与进度风险

---

## 一、项目概况

- 面向「智能制造项目管理」的**内网 Web 数字化管理平台**（Digital Transformation）。
- 定位：项目全过程主入口（项目创建、8 阶段流程、阶段资料、状态看板）；文件管理平台作为文件底座（**当前阶段暂停联动**）。
- 主线代码两条：`digital-platform-api`（后端）与 `digital-platform-web`（前端）。
- 配套：`docs/`（大量规划/验证文档 + 建库 SQL）、`openspec/`（57 个已归档 change + 1 个 active change）、`智能制造项目管理文件模板/`（Excel/Word 模板）、`daily_and_weekly_files/`。
- 两个工程均已在 2026-07-08 当天被构建/运行（`dist/`、`dev-server.out.log` 时间戳为 07-08），属**活跃项目**。

## 二、技术栈

| 层 | 技术 |
|----|------|
| 后端 | Node.js (≥20) + Express 4 + mysql2（连接池，无 ORM）；dotenv、busboy（附件上传）；纯 ES Module |
| 前端 | Vue 3.5 + Element Plus 2.14 + Vite 6；**无 vue-router**，自研 hash 路由（`src/router.js`） |
| 测试 | 后端用 Node 内置 `node:test`（仅日报/周报/中心日报有覆盖）；前端无测试 |
| 构建 | 后端 `node src/server.js`（端口 3001）；前端 `vite`（dev 5174，代理 `/api → 3001`），已产出 `dist/` |
| 部署 | 内网同服务器部署，独立服务/端口边界；不依赖文件平台数据库 |

## 三、后端现状（`digital-platform-api`，85 个 JS 文件）

分层严格、结构清晰：

- **入口**：`src/server.js` + `src/app.js`。挂载 `/api` 下 8 个路由前缀（`auth` / `me` / `users` / `navigation` / `projects` / `daily-reports` / `center-daily-reports` / `weekly-reports`），`/health`，统一 `notFound + errorHandler`（覆盖 20+ 自定义错误类型）。无静态文件中间件，文件走 `res.download` 流式下载。
- **路由层**（薄控制器，仅参数解析/鉴权）→ **仓储层**（`src/repositories`，`projects/*`、`stageDocuments/*` 等）→ **领域层**（`src/domain`：组织/角色/权限谓语、阶段资料状态机、精准返工、日报周报、工作日计算、导航）。
- **服务层**（`src/services`）：4 个 Excel/docx 导出服务（自研 ooxml renderer）、周报评分/预填服务、周报 AI 预填（直连 DeepSeek，需 `DEEPSEEK_API_KEY`）、导航服务。
- **数据层**：`src/db/pool.js`（连接池 10）；`src/db/*Schema.js` 运行时幂等补列；`migrations/` 27 个有序 SQL（数据库源头）；脚本齐全（`init-auth`、`init-stage-documents`、5 个 `migrate-*` 数据修复、`check`/`diagnose`）。
- **鉴权**：**基于数据库 Session（非 JWT / 非 cookie）**，Bearer token（32B 随机），`token_hash` 用 sha256 存储，TTL 12h；密码 PBKDF2-SHA256 12 万次迭代；5 种组织角色 × 4 业务部门，细粒度谓语封装成 `requireAuth` / `requirePlatformAdmin` / `requireDailyReportWriter` 等中间件。

**已实现模块（约 85–90% 文档化功能）**：

1. 用户 / 认证 / 权限
2. 项目 CRUD + 8 阶段 + 阶段审批四态（提交/通过/退回/重提）+ 阶段推进门禁（齐套 + 审批通过）
3. 阶段资料全链路：清单、状态机、适用性、责任人、附件（上传/下载/删除，50MB 内）、齐套摘要、精准返工、发起评审节点、生成文件、在线表单（含 1.2 专用营销/技术评价 + 总经理审批）、业务日志
4. 日报 / 周报 / 中心日报：CRUD、导出 Excel、AI 预填、评分、审批、终核
5. 我的工作台 / 资料待办、项目总览看板、导航树

**受限 / 停用**：

- 中心日报自动调度器（`centerDailyReportScheduler`）已实现但被显式停用。
- 周报 AI 预填依赖外部 `DEEPSEEK_API_KEY`，未配即报错（无降级）。
- 时区硬编码 `Asia/Shanghai`。
- **核心模块（projects / stageDocuments / auth / users / workbench）缺自动化测试**，仅日报/周报有测试——回归风险高。

## 六、进度评估与关键风险

**整体进度：约 85–90% 规划功能已落地。** 主线（项目 → 8 阶段 → 阶段资料 → 齐套 → 推进 → 看板）闭环可用；立项阶段（1.1/1.2/1.3）在线表单 + 多节点审批 + 模板文件生成已贯通；日报周报体系完整。

**关键风险 / 待办**：

| 优先级 | 项 | 说明 |
|--------|----|------|
| 🔴 高 | 测试缺口 | 核心业务模块无自动化测试，仅日报/周报有 `node:test` 覆盖，发版/重构回归风险高 |
| 🔴 高 | 文档脱节 | 前端 README 严重落后于代码；建议以 openspec/specs 为准同步 |
| 🟡 中 | 全局样式 | `styles.css` 4151 行、无设计令牌/主题变量，技术架构已要求收敛组件边界 |
| 🟡 中 | 自研路由 | 用 hash 路由而非 vue-router，缺少集中守卫、参数校验、懒加载（影响首屏与可维护性） |
| 🟡 中 | 外部依赖脆弱 | 周报 AI 预填直连 DeepSeek，无 key 即报错、无降级；中心日报调度器已停用 |
| 🟢 低 | 文件平台联动 | `file-platform-integration-v1` 按架构仍为暂停/后续独立 change，当前附件走在线平台存储，符合边界 |
| 🟢 低 | 新旧项目并存 | 旧项目（20260625 64 项）与新项目（v20260629 71 项）兼容策略已明确，未强制迁移 |

**当前最可能在推进的方向**（基于最新归档 change 时间线 2026-07-06/07）：立项阶段在线表单模板文件生成运行时、1.2 新版模板与通知编号流程修正。后续大方向：其他 7 阶段逐阶段工作区迁移、文件平台联动、复杂权限/审批。
