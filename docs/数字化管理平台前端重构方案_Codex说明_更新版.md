# 数字化管理平台前端导航及项目详情页重构方案

## 1. 文档目的

本文档用于指导 Codex 对数字化管理平台前端进行重构。

目标：

-   重构主页面布局；
-   实现动态导航体系；
-   重构项目详情页面为项目工作区模式；
-   统一使用 Element Plus UI 风格；
-   保持当前后端业务模型，不进行数据库改造。

------------------------------------------------------------------------

# 2. 当前问题

## 2.1 App.vue 职责过重

当前 App.vue 可能承担：

-   页面布局；
-   导航显示；
-   路由出口；
-   页面逻辑。

需要拆分。

App.vue 只保留应用入口：

``` vue
<template>
  <RouterView />
</template>
```

------------------------------------------------------------------------

## 2.2 项目详情页面耦合严重

当前项目详情页面包含：

-   项目阶段
-   文档列表
-   审批
-   附件
-   操作日志
-   时间线

多个业务混合。

目标：

项目详情页面只负责：

1.  左侧项目流程导航；
2.  右侧节点页面容器。

------------------------------------------------------------------------

# 3. 核心设计原则

## 导航层级原则

只有一级目录固定。

二级及以下全部动态。

结构：

    一级目录（固定）
        |
        +-- 二级目录（JSON配置）
                |
                +-- 三级目录（项目数据）
                        |
                        +-- 四级目录（项目阶段）
                                |
                                +-- 五级目录（业务节点）

------------------------------------------------------------------------

# 5. 二级导航设计

-   后续可以平滑迁移数据库。

# 18. 项目详情页面节点状态展示需求（新增）

## 18.1 设计目标

项目详情页面不是普通详情展示页面，而是：

> 项目执行工作区（Project Workspace）

左侧导航不仅负责页面跳转，同时需要展示项目执行状态。

最终效果：

    项目流程导航 = 流程树 + 状态看板

用户进入项目后，可以快速了解：

-   当前项目执行到哪个阶段；
-   哪些节点已完成；
-   哪些节点正在处理；
-   哪些节点存在异常。

------------------------------------------------------------------------

左侧导航结构：

    项目名称

        |

                |
                |

                        +-- 节点状态

# 18.3 节点状态模型

前端不要直接使用数据库状态。

后端统一转换为业务状态。

统一状态：

``` javascript
{
  PENDING,
  PROCESSING,
  COMPLETED,
  WAIT_APPROVAL,
  RETURNED,
  FAILED
}
```

状态含义：

  状态            说明
  --------------- ----------
  PENDING         未开始
  PROCESSING      进行中
  COMPLETED       已完成
  WAIT_APPROVAL   待审核
  RETURNED        退回整改
  FAILED          异常

# 18.8 阶段状态计算规则

阶段状态不直接保存。

通过节点状态计算。

规则：

    如果所有节点完成

        =>
    阶段 COMPLETED


    如果存在处理中节点

        =>
    阶段 PROCESSING


    如果全部未开始

        =>
    阶段 PENDING

# 18.12 Codex实现要求

实现时遵循：

1.  不新增数据库表；
2.  不修改已有业务表结构；
3.  使用现有状态字段；
4.  新增 NavigationService 聚合导航数据；
5.  前端使用 Element Plus Tree 展示；
6.  节点页面保持独立 Vue 页面；
7.  状态转换逻辑放后端，不放前端。

最终目标：

    固定平台导航

            +

    动态业务导航

            +

    项目流程树

            +

    节点状态可视化

            +

    独立业务页面

形成完整项目工作区架构。
