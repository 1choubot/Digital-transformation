# 9.28 v20260629 新项目运行基线稳定验证规划

## 背景

`enable-v20260629-template-for-new-projects-v1` 已将新建项目默认资料模板切换为 `v20260629` 71 项，并明确旧项目继续保持 20260625 64 项不迁移。切换完成后，需要用独立 change 对真实运行链路做稳定验证，确认新项目创建、项目工作区、阶段资料、我的工作台、阶段推进和兼容资料区在 64/71 并存状态下可用。

本 change 目标是稳定运行基线，不是新增业务规则。

## 目标

- 验证新建项目默认生成 71 项资料，且 `template_version = v20260629`。
- 验证新项目不包含 `3.3`、`5.4`、`LC33`、`LC54`。
- 验证旧项目仍保持 20260625 64 项，不迁移、不补初始化、不改状态、责任人或附件。
- 验证新项目项目工作区展示 8 阶段、蓝色模块和 71 项产出卡片。
- 验证旧项目仍能看到 `LC33 / LC54` 旧模板兼容卡片，新项目不显示 `LC33 / LC54`。
- 验证 `1.1 / 1.2 / 1.3` 在线表单不回退，`1.2` 专用评价审批继续可用。
- 验证非立项资料在上方产出卡片复用现有责任人、附件上传、提交、审核、退回、返工、不适用能力。
- 验证我的工作台能展示 v20260629 新项目待办。
- 验证阶段推进按项目实际资料集合判断，不混用 64/71。
- 验证兼容资料区在新项目中文案不误导，在旧项目中仍可展开核对。

## 非目标

- 不新增业务规则。
- 不新增复杂在线表单。
- 不新增第二套上传、提交、审核、退回、返工或不适用规则。
- 不接文件平台，不恢复文件平台目录、归档或下载联动。
- 不迁移旧项目，不补初始化旧项目。
- 不改数据库 schema，不写 migration。
- 不删除、不隐藏、不物理移除兼容资料区。
- 不处理 `file-platform-integration-v1` 或 `define-digital-platform-v1`。

## 验证策略

### 1. API smoke

优先用 API smoke 验证数据层和权限层：

- 新建测试项目。
- 校验项目阶段资料数量为 71。
- 校验项目阶段资料模板版本为 `v20260629`。
- 校验新项目不包含 `3.3`、`5.4`、`LC33`、`LC54`。
- 校验旧项目仍保持 64 项资料。
- 校验阶段资料清单、workspace、我的工作台和阶段推进门禁按项目实际资料集合判断。

### 2. 人工浏览器验收

再用浏览器人工验收主要页面和交互：

- 项目总览。
- 项目详情页。
- 上方项目工作区。
- 兼容资料区。
- 我的工作台。
- 立项在线表单和 `1.2` 专用评价审批。
- 非立项产出卡片通用资料操作。

浏览器自动化不是强制项。如果 agent 自动化卡住，可以由人工测试会话执行，并在验证记录中注明执行方式、账号、项目、结果和问题。

### 3. Bugfix 边界

本 change 允许修复验证发现的 v20260629 新项目运行阻塞 bug，但修复必须遵守：

- 优先修复现有入口、现有状态字段、现有权限判断和现有刷新链路。
- 不创建第二套资料状态机。
- 不创建第二套上传、提交、审核、退回、返工或不适用规则。
- 不新增复杂流程引擎、付款流、发票流或项目模式分支。
- 不迁移旧项目，不补初始化旧项目。
- 不接文件平台。
