## Context

本 change 对齐 20260610 版智能制造项目管理主流程。依据文件为：

- `智能制造项目管理流程图20260610.pdf`
- `docs/9.7_智能制造项目整体推进流程_20260610.md`
- `docs/9.2_阶段资料清单与责任角色表_20260610.md`

当前系统仍处于开发阶段，项目和阶段资料均为模拟数据。旧版 48 项资料模板在本 change 后不再作为程序运行依据，不考虑旧项目迁移、旧资料项兼容展示或历史项目补初始化。

## Goals / Non-Goals

**Goals:**

- 将项目流程依据明确为 20260610 新项目管理流程图。
- 保持现有 8 阶段名称和顺序不变。
- 将阶段资料模板来源切换为正式文档 `docs/9.2_阶段资料清单与责任角色表_20260610.md`。
- 后端模板版本改为 `v20260610`。
- `EXPECTED_STAGE_DOCUMENT_ITEM_COUNT` 后续按新版资料项实际数量设置；正式文档当前为 54 项。
- 新建项目直接初始化新版资料项。
- 正式资料清单必须包含 `文件平台目标目录` 字段，后端解析并保存为 `targetFolderPath`。
- `targetFolderId` 第一版继续为空。
- 齐套摘要继续只统计“适用 + 必填”资料。
- 非必需资料继续展示，但不影响齐套率和阶段推进。
- 阶段推进门禁继续要求当前阶段适用必填资料均为 `confirmed`。
- 资料状态机、适用/不适用、责任人分配、附件能力保持现有行为，只替换资料项模板。

**Non-Goals:**

- 不兼容旧模拟项目数据。
- 不做历史项目补初始化。
- 不做旧 48 项资料模板与新版资料模板共存。
- 不做资料模板 `v1` / `v20260610` 双版本运行。
- 不做日报周报。
- 不做文件管理平台联动。
- 不做消息提醒。
- 不做复杂审批流。
- 不做跳阶段、阶段回退或阶段推进审批。
- 不新增业务日志能力。

## Decisions

### 20260610 流程图作为主流程依据

系统项目管理主流程以后以 `智能制造项目管理流程图20260610.pdf` 为依据，`docs/9.7_智能制造项目整体推进流程_20260610.md` 作为流程文字整理说明。8 阶段主干继续沿用现有阶段名称和顺序，不在本 change 中新增、删除、重命名或重排阶段。

### 阶段资料模板切换为 20260610 新版清单

新版资料模板来源为正式文档 `docs/9.2_阶段资料清单与责任角色表_20260610.md`。后端实现应从该正式文档解析或固化同口径常量数据，不再引用非正式资料清单作为 MUST 来源。

正式资料清单每个资料项必须包含 `文件平台目标目录` 字段。后端应将该字段解析为项目级资料项模板快照中的 `targetFolderPath`，并在第一版继续保持 `targetFolderId` 为空。

旧版 48 项资料模板直接废弃。由于当前数据为开发阶段模拟数据，后续实现不提供旧模板兼容、不保留旧模板运行分支、不为历史项目做补初始化。

### 模板版本和数量校验

后端模板版本改为 `v20260610`。`EXPECTED_STAGE_DOCUMENT_ITEM_COUNT` 按正式文档中的新版资料项实际数量设置；当前正式文档列出 54 项。

数量校验仍用于防止资料清单解析不完整，但校验对象改为新版清单，不再校验旧 48 项。

### 新建项目初始化口径

新建项目仍初始化标准 8 阶段，同时直接初始化 `v20260610` 新版阶段资料项。旧项目和旧资料项不做兼容，不做补初始化，不做迁移。

### 齐套摘要和阶段推进门禁

齐套摘要沿用现有口径：只统计 `isRequired = true` 且 `isApplicable = true` 的资料项；`confirmed` 计为完成；`not_submitted`、`submitted`、`returned` 计为未完成；非必需资料和不适用资料不影响齐套率。

阶段推进门禁继续读取当前阶段齐套摘要，只有当前阶段适用必填资料均为 `confirmed` 时允许推进。非必需资料不阻塞阶段推进。

### 既有资料能力保持

资料状态机、适用/不适用、责任人分配、我的资料任务、项目总览、附件上传/列表/下载/删除保持现有行为。后续实现只改变资料项模板集合、资料项名称、必填口径、默认责任角色等模板数据，不改变这些能力的状态机、权限边界、错误码或接口路径。

## Risks / Trade-offs

- 旧模拟数据不可继续按新版资料项展示 -> 接受该结果；当前项目仍为开发阶段模拟数据。
- 正式资料清单后续如发生业务确认调整 -> 后续实现必须同步更新正式文档和 `EXPECTED_STAGE_DOCUMENT_ITEM_COUNT`，不得重新引用非正式资料清单作为 MUST 来源。
- 新版资料项数量和名称变化会影响我的资料任务、项目总览和前端详情展示 -> 后续实现按后端返回数据动态展示，不允许前端写死旧资料名称或旧数量。
- 非必需资料不影响齐套率可能导致阶段推进不等待部分产出 -> 这是本 change 指定口径，继续以“适用 + 必填 + confirmed”为门禁。
