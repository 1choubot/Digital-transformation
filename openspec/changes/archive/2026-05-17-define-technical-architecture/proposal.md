## Why

数字化管理平台第一版已经明确业务边界，但还需要固定正式系统的技术架构，避免后续实现继续沿用前端 Demo 的临时数据结构或误用文件管理平台数据库。

本变更用于明确第一版技术栈、部署方式、数据边界和两平台联动方式，为后续项目核心、在线表单、文件联动等实现提供统一架构约束。

## What Changes

- 确定前端采用 Vue + Vite。
- 确定后端采用 Node.js + Express，作为数字化平台独立服务端。
- 确定数据库采用 MySQL，并由数字化平台独立维护业务数据。
- 确定部署方式为内网 Web，并与文件管理平台部署在同一服务器。
- 确定文件仍由文件管理平台存储，数字化平台不直接存储正式归档文件。
- 确定数字化平台与文件管理平台通过 HTTP API 联动。
- 明确数字化平台不能直接共用或直接读写文件管理平台数据库。
- 明确前端 Demo 只作为页面参考，不作为正式数据架构、接口结构或持久化方案。

## Capabilities

### New Capabilities

- `technical-architecture`: 覆盖数字化管理平台第一版的前端、后端、数据库、部署、文件存储边界、HTTP API 联动和 Demo 使用边界。

### Modified Capabilities

- 无。

## Impact

- 后续数字化平台实现必须按 Vue + Vite、Node.js + Express、MySQL 的技术架构落地。
- 后续文件联动必须通过 HTTP API 与文件管理平台交互，不能绕过服务接口直接访问文件平台数据库。
- 部署方案需要支持数字化平台与文件管理平台在同一内网服务器上以独立服务方式运行。
- 前端 Demo 可继续作为页面布局和交互参考，但不能作为正式数据模型、接口模型或运行时架构来源。
