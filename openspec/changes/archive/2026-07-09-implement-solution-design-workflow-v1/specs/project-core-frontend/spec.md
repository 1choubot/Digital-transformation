## ADDED Requirements

### Requirement: 方案设计阶段子节点展示
项目核心前端 MUST 在既有 8 大阶段导航中的“方案设计阶段”下展示方案设计内部 9 个子节点。

#### Scenario: 展示内部节点列表
- **WHEN** 用户打开项目工作区的方案设计阶段
- **THEN** 前端 MUST 展示方案设计准备、项目方案分析、方案设计、内部方案评审、客户方案评审、研发成本估算、制造成本估算、财务成本估算和报价/投标 9 个子节点
- **AND** 前端 MUST NOT 将这些子节点展示为新的大阶段

#### Scenario: 展示节点状态
- **WHEN** 后端返回方案设计内部节点状态
- **THEN** 前端 MUST 至少能展示未开始、待提交、待审批、已退回、已通过、已跳过/不适用和已结束
- **AND** 前端 MUST 使用后端返回状态，不得用本地静态规则硬猜状态

#### Scenario: 方案设计阶段不影响其他阶段导航
- **WHEN** 用户切换立项、合同签订、详细设计、生产制作、预验收、终验收或结题阶段
- **THEN** 前端 MUST 保持既有 8 大阶段导航可用
- **AND** 前端 MUST NOT 因方案设计子节点规划改变其他阶段入口

### Requirement: 方案设计项目内角色和待办展示
项目核心前端 MUST 展示后端返回的方案设计项目内角色、可处理动作和个人工作台待办。

#### Scenario: 研发中心负责人角色分配入口
- **WHEN** 后端返回当前用户可处理方案设计准备角色分配
- **THEN** 前端 MUST 展示分配项目经理、技术负责人、商务负责人、采购负责人、财务会计和财务负责人的入口
- **AND** 前端 MUST NOT 展示主管机械工程师、主管电气工程师或主管软件工程师分配入口

#### Scenario: 项目经理展示复用项目现有字段
- **WHEN** 前端展示或提交方案设计准备节点的项目经理分配
- **THEN** 前端 MUST 将项目经理表达为项目现有项目经理用户关联的设置或更新
- **AND** 前端 MUST NOT 展示第二套与项目详情项目经理相冲突的方案设计专用项目经理
- **AND** 分配成功后前端 MUST 刷新项目详情、项目经理待办和方案设计工作计划入口

#### Scenario: 工作台待办来自后端
- **WHEN** 用户打开个人工作台或项目工作区待办区域
- **THEN** 前端 MUST 展示后端返回的方案设计阶段待办
- **AND** 前端 MUST NOT 仅凭用户部门、姓名、项目角色文案或本地规则生成可处理待办

#### Scenario: 项目内角色操作入口
- **WHEN** 后端返回当前用户具备项目经理、技术负责人、采购负责人、财务会计或商务负责人项目内角色的处理权限
- **THEN** 前端 MUST 显示对应的上传、在线表单、提交或重提入口
- **AND** 前端 MUST 在无权限时隐藏或禁用对应入口并展示可读原因

### Requirement: 方案设计节点操作体验
项目核心前端 MUST 按节点上下文展示方案设计阶段的在线表单、上传、审批、退回原因和重提入口。

#### Scenario: 项目方案分析操作展示
- **WHEN** 用户查看项目方案分析节点
- **THEN** 前端 MUST 展示项目方案分析表在线填写入口和产品功能框图上传入口
- **AND** 前端 MUST 展示研发中心负责人审批状态、审批意见和退回原因
- **AND** 审批退回后前端 MUST 展示整体重新提交入口，不得引导精确重提单个产出

#### Scenario: 方案设计 8 个产出展示
- **WHEN** 用户查看方案设计节点
- **THEN** 前端 MUST 展示工艺时序图、节拍表、布局图、3D模型、演示动画、电气功能框图、软件功能框图和项目方案PPT 8 个上传产出
- **AND** 前端 MUST 将工艺时序图展示在方案设计节点目标归属下

#### Scenario: 方案评审记录展示
- **WHEN** 用户查看内部方案评审或客户方案评审节点
- **THEN** 前端 MUST 分别展示 C15 方案评审记录表（内部方案评审）和 C16 方案评审记录表（客户方案评审）的在线填写入口
- **AND** 前端 MUST 区分内部方案评审和客户方案评审上下文
- **AND** 前端 MUST 展示最新有效记录、审批状态和退回原因
- **AND** 前端 MUST 提供历史记录入口或至少展示后台保留历史可审计的提示
- **AND** 前端 MUST NOT 将 C15 和 C16 合并成一个资料项或一个不可区分的记录列表

#### Scenario: 成本估算三段展示
- **WHEN** 用户查看成本估算节点
- **THEN** 前端 MUST 分别展示研发成本估算、制造成本估算和财务/运营成本估算的上传、审批、退回和历史状态
- **AND** 前端 MUST 展示后一部门可下载前一部门文件的入口，但只能在后端返回允许时展示

### Requirement: 财务成本估算文件保密展示
项目核心前端 MUST 在财务成本估算节点遵守后端保密结果，不向非授权用户展示财务文件细节。

#### Scenario: 非授权用户财务文件脱敏
- **WHEN** 后端返回当前用户无权查看财务/运营成本估算文件，或当前用户是研发中心负责人、技术负责人、项目经理、商务负责人、采购负责人、制造中心负责人、非授权运营中心人员、总经理助理、系统管理员或其他无关用户
- **THEN** 前端 MUST 只展示财务成本估算节点状态和审批结果
- **AND** 前端 MUST NOT 展示文件名、附件明细、预览入口或下载入口

#### Scenario: 授权用户查看财务文件
- **WHEN** 后端返回当前用户是总经理或运营中心授权处理人且有权查看财务/运营成本估算文件
- **THEN** 前端 MUST 按后端返回展示文件名、附件明细、预览或下载入口
- **AND** 前端 MUST NOT 在无后端授权字段时自行拼接下载地址

#### Scenario: 财务文件所有入口均遵守后端脱敏
- **WHEN** 前端展示节点详情、资料清单、附件列表、业务日志、预览入口或下载入口
- **THEN** 前端 MUST 以后端过滤后的结果为准
- **AND** 前端 MUST NOT 从缓存、历史记录或本地拼接内容恢复无权查看的财务文件细节

#### Scenario: 直接访问失败提示
- **WHEN** 用户通过历史链接或直接地址访问无权查看的财务文件
- **THEN** 前端 MUST 展示可读无权提示
- **AND** 前端 MUST NOT 显示已缓存的文件名或附件明细

### Requirement: 报价投标分支 UI
项目核心前端 MUST 将报价/投标展示为一个节点，并按后端分支状态展示未选择、报价流程或投标流程。

#### Scenario: 未选择分支状态
- **WHEN** 报价/投标节点处于未选择状态且后端返回总经理可选择
- **THEN** 前端 MUST 展示报价流程和投标流程的分支选择入口
- **AND** 前端 MUST 在选择后刷新节点状态和待办

#### Scenario: 报价流程展示
- **WHEN** 报价/投标节点处于报价流程
- **THEN** 前端 MUST 展示报价单上传入口和商务负责人报价结果处理状态
- **AND** 报价单 MUST 表达为文件上传产出
- **AND** 前端 MUST NOT 展示报价单模板生成承诺

#### Scenario: 报价未被客户接受选择展示
- **WHEN** 商务负责人记录客户不同意报价
- **THEN** 前端 MUST 展示商务负责人线下与总经理讨论后的退回研发成本估算和项目结束二选一处理入口
- **AND** 选择项目结束时前端 MUST 清楚提示合同签订及后续阶段将被阻止

#### Scenario: 投标流程展示
- **WHEN** 报价/投标节点处于投标流程
- **THEN** 前端 MUST 展示投标商务标和投标技术标两个上传槽
- **AND** 前端 MUST 展示两个上传槽都完成后才能提交总经理审批
- **AND** 前端 MUST NOT 将报价流程和投标流程拆成两个固定大节点

#### Scenario: 投标退回展示
- **WHEN** 总经理审批不通过投标书
- **THEN** 前端 MUST 展示投标节点退回原因
- **AND** 前端 MUST 展示商务标和技术标重新提交入口
- **AND** 前端 MUST NOT 默认引导回成本估算节点

### Requirement: 方案设计在线表单和生成状态展示
项目核心前端 MUST 按方案设计阶段节点展示在线表单入口和生成文件状态，但不得伪造文件平台或 PDF 能力。

#### Scenario: 项目方案分析表生成状态
- **WHEN** 项目方案分析表在线表单提交后后端返回生成文件状态
- **THEN** 前端 MUST 展示对应的生成状态和可用下载或查看入口
- **AND** 前端 MUST NOT 在后端未返回生成文件时伪造下载入口

#### Scenario: 方案评审记录表生成状态
- **WHEN** 内部方案评审或客户方案评审记录表提交后后端返回生成文件状态
- **THEN** 前端 MUST 按 C15 内部方案评审或 C16 客户方案评审节点上下文展示对应生成状态
- **AND** 前端 MUST NOT 混淆内部评审记录和客户评审记录

#### Scenario: 不展示文件平台或 PDF 承诺
- **WHEN** 用户查看方案设计阶段任一节点
- **THEN** 前端 MUST NOT 展示已接入文件平台、已归档到文件平台、生成 PDF 或可下载 PDF 的承诺
- **AND** 前端 MUST NOT 在浏览器端填充 Excel 或 Word 模板

### Requirement: 方案设计专用流程入口展示
项目核心前端 MUST 对纳入方案设计内部状态机的资料项和上传槽展示专用节点流程入口，不得展示普通资料审核入口。

#### Scenario: 隐藏普通资料审核入口
- **WHEN** 用户查看项目方案分析、C15 内部方案评审、C16 客户方案评审、C17 成本估算、报价或投标相关资料
- **THEN** 前端 MUST 隐藏或禁用普通资料确认、普通资料退回和普通资料审核入口
- **AND** 前端 MUST 展示方案设计专用节点流程入口和后端返回的可操作动作

#### Scenario: 普通审核直接调用失败展示
- **WHEN** 用户通过历史入口或异常路径触发普通资料审核且后端返回专用流程错误
- **THEN** 前端 MUST 展示必须使用方案设计专用节点流程的可读提示
- **AND** 前端 MUST 刷新节点状态和待办

#### Scenario: 待办不重复展示
- **WHEN** 后端为方案设计专用流程对象返回待办
- **THEN** 前端 MUST 只展示专用节点提交或审批待办
- **AND** 前端 MUST NOT 同时展示普通资料审核待办和专用节点审批待办
