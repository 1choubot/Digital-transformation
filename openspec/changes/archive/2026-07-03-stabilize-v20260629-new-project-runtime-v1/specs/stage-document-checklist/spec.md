## ADDED Requirements

### Requirement: v20260629 阶段资料清单运行验收

阶段资料清单能力 MUST 支持 v20260629 新项目和 20260625 旧项目并存运行验收；新项目 MUST 使用 71 项资料集合，旧项目 MUST 保持既有 64 项资料集合。

#### Scenario: 新项目阶段资料清单返回 71 项
- **WHEN** 用户或 API smoke 查询 v20260629 新项目阶段资料清单
- **THEN** 阶段资料清单 MUST 按 8 阶段返回 71 项项目级资料
- **AND** 每项资料 MUST 返回资料编号、资料名称、模板版本、completionMode、状态、适用性、责任人、附件、权限、阻塞原因和派生完成状态所需字段

#### Scenario: 旧项目阶段资料清单保持 64 项
- **WHEN** 用户或 API smoke 查询 20260625 旧项目阶段资料清单
- **THEN** 阶段资料清单 MUST 继续返回该项目已有 64 项项目级资料
- **AND** 系统 MUST NOT 因 v20260629 新项目默认模板启用而补初始化旧项目资料

#### Scenario: 旧兼容资料不进入新项目清单
- **WHEN** 系统查询 v20260629 新项目阶段资料清单
- **THEN** 清单 MUST NOT 返回 `3.3 合同审核记录表（销售合同）`、`5.4 采购合同审核记录表`、`LC33` 或 `LC54`
- **AND** 这些兼容项 MUST 继续只用于旧项目 workspace 兼容展示

#### Scenario: 新项目非立项资料沿用文件上传能力
- **WHEN** v20260629 新项目处理非立项阶段资料
- **THEN** 阶段资料清单能力 MUST 继续使用现有文件上传或附件上传能力承载
- **AND** 系统 MUST NOT 因运行稳定验证新增复杂在线表单、专用审批流或流程引擎

#### Scenario: 产出卡片和清单状态一致
- **WHEN** 用户从上方产出卡片或兼容资料区查看同一项目资料
- **THEN** 两处 MUST 展示同一套后端资料状态、责任人、附件、适用性、阻塞原因和权限结果
- **AND** 系统 MUST NOT 为 v20260629 新项目创建第二套完成状态

#### Scenario: 立项资料普通接口保护继续生效
- **WHEN** v20260629 新项目包含 `1.1`、`1.2` 或 `1.3`
- **THEN** 普通资料提交、普通审核或普通退回接口 MUST 继续保护这些在线表单资料
- **AND** 系统 MUST 提示调用方使用对应在线表单或 `1.2` 专用评价审批能力
