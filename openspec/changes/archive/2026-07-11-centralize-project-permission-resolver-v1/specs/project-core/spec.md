## ADDED Requirements

### Requirement: 项目核心权限判断统一表达
项目核心权限判断 MUST 逐步通过统一 helper/resolver 表达基础身份和项目上下文，并在迁移过程中保持现有权限结果不变。

#### Scenario: 基础身份和项目关系判断复用统一 helper
- **WHEN** 项目核心需要判断项目经理、中心负责人、总经理、总经理助理、系统管理员或资料责任人
- **THEN** 后端 SHOULD 复用统一 helper/resolver 表达该判断
- **AND** 重复的本地身份判断 SHOULD 在对应迁移批次中被替换为统一 helper

#### Scenario: 立项权限迁移保持现有结果
- **WHEN** 后续迁移立项 `1.1 / 1.2 / 1.3` 在线表单、评价/审批和精准返工相关权限
- **THEN** 系统 MUST 保持现有可查看、可填写、可提交、可评价、可审批和阻塞原因结果不变
- **AND** 迁移 MUST NOT 改变立项审批状态机或工作台待办口径

#### Scenario: 方案设计权限迁移保持现有结果
- **WHEN** 后续迁移方案设计角色分配、节点提交、节点审批、上传槽、报价/投标或财务保密权限
- **THEN** 系统 MUST 保持现有角色判断、节点状态 gate、上传/下载权限和自动推进触发结果不变
- **AND** 迁移 MUST NOT 改变方案设计复杂节点状态机或 C04-C19 派生齐套语义

#### Scenario: 阶段资料和阶段推进权限迁移保持现有结果
- **WHEN** 后续迁移阶段资料提交/确认/退回、manual fallback API 阶段推进或相关工作台待办权限
- **THEN** 系统 MUST 保持现有资料状态规则、中心负责人项目相关性规则、manual fallback API 权限和自动推进语义不变
- **AND** 本 change MUST NOT 改变当前权限结果，只建立收敛计划和第一批迁移边界
