## Why

当前前端功能基本打通，但项目总览、项目工作区、我的工作台、旧资料清单、用户管理等界面风格和操作层级不统一。若继续叠加第二阶段补录和文件平台功能，会放大页面混乱、组件边界不清和维护成本。

## What Changes

- 统一项目入口和项目工作区视觉层级：项目总览作为项目主入口，项目工作区作为单项目主操作区。
- 固化项目工作区层级：左侧 8 阶段导航，右侧按阶段说明、蓝色节点列表、节点产出区、在线表单/评价审批动作区组织。
- 弱化旧资料清单：下方旧清单作为辅助兼容区，立项阶段主操作继续位于上方节点产出卡片。
- 统一我的工作台深链与任务入口语义：深链定位到项目工作区的阶段、节点或产出，但不自动打开在线表单。
- 统一通用 UI 组件和样式口径：页面头部、按钮、状态、空态、错误提示和操作区使用一致语义。
- 收敛前端组件边界，减少 `styles.css` 继续无序膨胀。
- 保持旧 `ProjectListPage.vue` 不作为用户可见入口。

## Capabilities

### New Capabilities

- None.

### Modified Capabilities

- `project-core-frontend`: 增加前端统一项目入口、项目工作区体验、旧资料清单辅助区和工作台深链体验要求。
- `technical-architecture`: 增加前端信息架构统一的技术边界，明确 UI 统一不得改变后端业务状态机、权限来源或接口边界。

## Impact

- 主要影响后续 `digital-platform-web/src/**` 的页面布局、组件边界和样式组织。
- 影响 OpenSpec frontend / architecture specs。
- 不影响 API、数据库、migration 或权限规则。
- 不处理第二阶段补录、其他 7 阶段完整蓝色节点、文件平台联动、日报周报、通知推送、账号管理或通用审批流。
