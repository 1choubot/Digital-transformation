## Why

当前阶段资料项只能维护手工状态、适用性、责任人和齐套摘要，尚不能保存资料项对应的实际附件。第一版阶段资料附件能力用于补齐“资料项可上传、查看、下载、删除附件”的最小闭环，并为后续文件管理平台联动预留清晰边界。

## What Changes

- 新增阶段资料附件数据结构，附件关联到具体 `project_stage_document`，记录文件名、存储标识、类型、大小、上传人、上传时间和软删除追溯字段。
- 新增资料项附件上传、列表、下载和删除接口，均要求 `requireAuth`，不引入复杂权限、项目成员权限或资料责任人权限。
- 明确不适用资料项第一版拒绝上传附件。
- 上传和删除附件成功后写入项目业务操作日志；下载第一版不写日志，避免日志噪声。
- 项目详情页阶段资料清单新增附件区域，支持查看附件列表、上传、下载和删除附件。
- 明确附件操作不改变资料状态、适用性、齐套摘要或阶段推进状态，也不代表文件管理平台归档。
- 更新 API 和 Web README，说明阶段资料附件接口、页面能力和边界。

## Capabilities

### New Capabilities

- 无。

### Modified Capabilities

- `stage-document-checklist`: 增加阶段资料附件模型、上传、列表、下载、删除、日志和业务边界要求。
- `project-core-frontend`: 增加项目详情页资料附件展示和操作要求，并调整项目详情页排除能力口径，允许本变更定义的资料附件能力。
- `business-operation-log`: 增加附件上传和删除的业务日志动作要求，明确下载不写业务日志。

## Impact

- 后端：新增附件表或等价持久化结构、附件存储目录策略、附件上传/列表/下载/删除接口、错误码和业务日志写入。
- 前端：项目详情页阶段资料清单新增附件区域和附件操作交互。
- 文档：更新 `digital-platform-api/README.md` 和 `digital-platform-web/README.md`。
- 依赖：第一版不引入对象存储、S3、OSS、文件管理平台联动、病毒扫描或附件预览；如实现 multipart 上传需要使用现有后端技术栈内的上传处理中间件，不新增无关运行时能力。
