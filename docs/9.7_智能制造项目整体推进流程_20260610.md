# 智能制造项目整体推进流程（20260610）

版本：v0.1 草案  
来源：`智能制造项目管理流程图20260610.pdf`  
整理日期：2026-06-23  
适用范围：数字化管理平台后续项目流程、项目角色、阶段任务、日报周报、资料归档和文件管理平台联动规划

## 1. 编制目的

本文件用于沉淀 20260610 版智能制造类项目管理流程的整体业务主线，说明一个项目从市场调研、立项、方案设计、合同签订、详细设计、生产制作、验收到结题的完整推进方式。

本文件关注“项目怎么从开始走到结束”，与 `docs/9.2_阶段资料清单与责任角色表_20260610草案.md` 分工如下：

- 本文件：说明阶段、中心职责、流程节点、推进关系和系统落地方向。
- 资料清单草案：说明每个阶段具体需要哪些产出文件、谁填写、是否必需、是否影响齐套。

本文件暂不直接替换现有系统 8 阶段配置或资料模板。后续如要落地，应通过 OpenSpec change 分阶段实施。

## 3. 中心职责主线

流程图按中心泳道组织，项目推进不是单一部门线性完成，而是多个中心在不同阶段承担不同职责。

| 中心/角色 | 主要职责 |
|---|---|
| 营销中心 | 市场调研、项目需求表、立项审批表、项目立项通知、客户方案评审、销售合同、客户设计评审、预验收组织、终验收组织、产品培训组织 |
| 研发中心 | 项目组成员指定、方案设计计划、需求分析、方案设计、技术协议、详细设计计划、详细设计、内部评审、图纸审查、技术通知单、作业指导和技术资料编写 |
| 制造中心 | 项目启动会、成本估算接收、采购、比价、采购合同、检验、入库、领料、安装调试、自验收、预验收单、发货、终验收单 |
| 项目经理 | 项目第一责任人，负责项目工作群、工作计划、流程组织、资料收口、阶段推进和项目管理协调 |
| 技术负责人 | 负责方案和详细设计阶段的技术产出、成本估算汇总、图纸和零部件清单汇总、技术资料编写或收口 |
| 总经理/管理层 | 参与关键审批和全局监督，关注项目进度、风险、延期、资料完整性和结题情况 |

## 4. 阶段推进说明

### 4.1 立项阶段

立项阶段从市场调研开始，由营销中心发现或承接客户需求，并形成项目需求表。

关键流程：

1. 营销人员完成项目市场调研。
2. 营销人员填写项目需求表，明确项目基本信息、工作环境条件、现工艺和期望目标。
3. 营销人员填写项目立项审批表，并将项目需求表作为审批附件。
4. 商务、技术、总经理等按流程线下审批签字。
5. 审批完成后扫描上传电子档。
6. 营销总监编写项目立项通知，并发布至公司工作总群。
7. 研发总监指定项目组成员。
8. 项目经理成为项目第一责任人，并建立项目工作群。
9. 项目经理制定方案设计阶段工作计划。

系统落地含义：

- 项目创建不应只是填写项目编号和客户，还应能记录项目来源、需求、立项审批状态、项目经理和项目组。
- 项目经理是项目级核心角色，不应只作为文本字段长期存在。
- 项目启动前已有多个关键资料需要归档和追溯。

### 4.2 方案设计阶段

方案设计阶段由研发中心主导，营销中心组织客户侧评审，制造中心参与成本估算衔接。

关键流程：

1. 项目经理组织方案需求分析，必要时组织客户人员参加。
2. 技术负责人编写项目需求分析表和产品功能框图。
3. 机械、工艺、电气、软件等人员完成 3D 模型、布局图、工艺时序图、节拍表、演示动画、电气功能框图、软件功能框图等产出。
4. 项目经理组织内部方案评审，评审次数视方案成熟度而定，直到评审合格。
5. 各专业组成员编写本人部分成本估算零部件清单。
6. 技术负责人汇总成本估算表后发送制造中心。
7. 技术负责人编写项目方案 PPT。
8. 营销人员按项目流程组织客户方案评审。
9. 技术负责人填写方案评审记录表。

系统落地含义：

- 一个资料项可能需要多人协作，例如成本估算表。
- 评审类资料可能存在多轮记录，不应只理解为一次提交。
- 工艺时序图、节拍表、演示动画属于客户要求相关的非必需资料，齐套规则需要可配置。

### 4.3 合同签订阶段

合同签订阶段完成技术协议、销售合同和项目启动。

关键流程：

1. 技术负责人编写技术协议。
2. 技术协议经双方签字盖章后作为销售合同的一部分。
3. 先签技术协议，再签销售合同。
4. 营销人员编写并签订销售合同。
5. 制造中心组织召开项目启动会。
6. 制造中心形成项目启动书。
7. 项目经理制定详细设计阶段工作计划。
8. 项目经理建立或延续项目工作群。

系统落地含义：

- 合同签订阶段不是只有“技术协议 + 销售合同”，还包含项目启动书和详细设计计划。
- 制造中心从合同后正式进入项目执行主线。
- 项目启动会可作为后续任务分配和制造资料生成的业务节点。

### 4.4 详细设计阶段

详细设计阶段由研发中心主导，形成可下发制造的完整设计资料。

关键流程：

1. 机械、电气、软件等成员完成详细设计资料。
2. 产出包括 3D 模型、电气原理图、电气接线图、电气布置图、控制逻辑流程图、自动化程序、软件开发说明文档、UI 界面设计 PPT、软件代码等。
3. 项目经理按详细设计工作计划组织内部设计评审。
4. 内部评审次数视详细设计成熟度而定，直到评审合格。
5. 营销人员按项目流程组织客户设计评审。
6. 技术负责人填写设计评审记录表。
7. 各专业组成员编写本人部分零部件清单，技术负责人汇总产品零部件清单。
8. 项目经理按流程组织图纸审查。
9. 图纸审查人员由研发总监指定。
10. 图纸审查不能代替图纸签字流程。
11. 客户图纸会签由营销人员组织，客户会签记录由项目经理扫描存档。
12. 技术负责人汇总各专业组图纸和零部件清单，形成整套完整图纸。
13. 图纸和零部件清单通过钉钉采购流程下发至制造中心。
14. 图纸和零部件清单一旦下发，后续修改必须通过技术通知单执行。

系统落地含义：

- 详细设计阶段需要区分“设计资料提交”“内部评审”“客户评审”“图纸审查”“客户会签”“图纸下发”这些节点。
- 技术通知单不只是一个普通资料项，而是图纸下发后的受控变更入口。
- 后续可以把技术通知单作为变更管理能力的前置。

### 4.5 生产制作阶段

生产制作阶段由制造中心主导，围绕采购、检验、入库、领料和生产准备展开。

关键流程：

1. 制造中心根据下发图纸和零部件清单寻找供应商。
2. 制造中心完成比价表。
3. 制造中心完成采购合同审批和签订。
4. 制造中心完成来料检验并形成检验单。
5. 制造中心完成材料入库并形成入库单。
6. 制造中心完成领料并形成领料单。

系统落地含义：

- 旧资料清单对生产制作阶段覆盖不足，新版新增了明显的制造中心资料。
- 这些资料是否全部影响阶段推进，需要与领导确认。
- 如果未来要与金蝶或采购系统集成，生产制作阶段是重点边界。

### 4.6 预验收阶段

预验收阶段包含技术资料准备、厂内安装调试、自验收和客户预验收。

关键流程：

1. 技术负责人编写作业指导书。
2. 技术负责人编写产品使用说明书和产品维护保养手册。
3. 技术负责人编写产品培训 PPT。
4. 制造中心进行厂内安装调试并形成安装调试记录。
5. 制造中心完成自验收并形成自验收报告。
6. 营销中心按工作计划组织产品预验收。
7. 制造中心人员填写预验收单。

系统落地含义：

- 预验收阶段同时涉及研发资料准备和制造中心验收资料。
- 预验收单与旧版验收记录表需要做映射确认。
- 厂内安装调试和现场安装调试可能使用同名或相近模板，系统中需要通过阶段区分。

### 4.7 终验收阶段

终验收阶段从发货、现场安装调试开始，最终完成客户终验收和培训。

关键流程：

1. 制造中心完成发货并形成发货单。
2. 制造中心进行现场安装调试并形成安装调试记录。
3. 营销中心按工作计划组织产品终验收。
4. 制造中心人员填写终验收单。
5. 营销人员按工作计划组织产品培训。
6. 培训主讲人员填写培训记录表。

系统落地含义：

- 终验收阶段不应只保留一个“验收记录表”，新版已经拆出发货、安装调试、终验收、培训记录。
- 发货和现场安装调试可能是后续项目进度看板的重要节点。

### 4.8 结题阶段

结题阶段由项目经理组织资料整理归集，技术负责人配合，最终形成项目结题报告。

关键流程：

1. 项目经理组织产品资料整理归集。
2. 技术负责人配合资料归集。
3. 资料整理归集重点检查项目过程资料的完整性。
4. 资料整理归集重点检查资料与实物的一致性。
5. 资料归集按《技术资料管理制度》执行。
6. 项目经理编写项目结题报告。
7. 项目完成结题。

系统落地含义：

- 结题不是简单推进第 8 阶段，而是需要资料完整性和一致性检查。
- 后续可以在结题阶段增加资料归集核查清单。
- 文件管理平台联动归档在结题阶段会成为关键验收项。

## 5. 对数字化平台的总体建模建议

根据 20260610 流程图，系统后续应避免只用“部门 + 角色字符串”支撑所有业务。建议至少区分以下模型：

### 5.1 组织角色

组织角色表示人在公司中的长期身份：

- 总经理
- 系统管理员
- 中心负责人
- 员工

组织部门包括：

- 营销中心
- 研发中心
- 制造中心
- 运营中心
- 数字化管理组或系统管理组

### 5.2 项目角色

项目角色表示人在某个项目中的职责，可能与组织角色不同：

- 项目经理
- 技术负责人
- 制造负责人
- 营销负责人
- 图纸审查人
- 培训主讲人
- 资料责任人
- 审核人

### 5.3 阶段资料任务

阶段资料任务表示具体产出物由谁负责：

- 所属项目
- 所属阶段
- 资料项
- 是否必需
- 是否适用
- 责任人
- 组织人
- 审核人
- 状态
- 退回原因
- 附件或在线表单
- 文件平台归档映射

### 5.4 协作子任务

部分资料项不是单人完成，例如成本估算表、产品零部件清单。后续可考虑资料项下的协作子任务：

- 成本估算表：各专业组成员填写本人部分，技术负责人汇总。
- 产品零部件清单：各专业组成员填写本人部分，技术负责人汇总。
- 评审记录：项目经理或营销人员组织，技术负责人填写记录。

## 6. 日报周报融入方式

日报周报应融入项目执行，而不是单独作为文本填报功能存在。

建议第一版关系：

```text
项目
  ├─ 当前阶段
  ├─ 阶段资料任务
  ├─ 项目角色
  ├─ 日报
  └─ 周报
```

员工填写日报时，应至少关联：

- 项目
- 日期
- 当前工作内容
- 关联阶段，可选
- 关联资料项或任务，可选
- 问题和风险
- 明日计划

员工填写周报时，应至少关联：

- 项目
- 周期
- 本周完成内容
- 本周问题和风险
- 下周计划
- 可引用或汇总本周日报

负责人和总经理查看日报周报时，应按项目、人员、中心、周期进行查询，并能看到日报与周报的基础一致性对照。

## 7. 文件管理平台联动位置

根据整体流程，文件管理平台不应替代数字化平台的业务流程，而应在以下节点作为文件底座参与：

1. 项目创建或立项完成后，创建或绑定项目文件夹树。
2. 阶段资料项生成后，建立资料项与文件平台目标目录的映射。
3. 资料确认通过后，归档附件或生成文件到文件平台。
4. 技术通知单、图纸下发、合同、验收资料等关键文件归档后，记录来源系统、来源动作、真实提交人、项目、阶段和资料项。
5. 用户从数字化平台查看或下载文件时，下载权限仍由文件管理平台判断。
6. 结题阶段检查资料归集完整性时，应结合数字化平台资料状态和文件平台归档状态。

## 8. 后续落地路线建议

建议后续按以下顺序推进：

1. `align-stage-document-template-20260610`：将新版流程图资料清单正式纳入 OpenSpec，处理新增、改名、必填变化和历史项目升级策略。
2. `add-organization-project-roles`：补组织角色和项目角色模型，明确项目经理、技术负责人、制造负责人、营销负责人等项目级职责。
3. `add-project-daily-weekly-reports`：在项目和项目角色基础上实现日报、周报、项目查看和基础对比。
4. `add-project-work-items`：把非资料型任务、协作子任务、多人成本估算、评审多轮记录等纳入项目任务模型。
5. `add-file-platform-project-sync`：实现项目创建或立项后联动文件管理平台创建/绑定目录。
6. `add-document-archive-to-file-platform`：实现资料确认通过后的文件平台归档、目录映射和日志追溯。
7. `add-closeout-document-verification`：结题阶段增加资料完整性和实物一致性检查。

## 9. 待确认问题

1. 运营中心在该流程图中没有明显泳道，是否仅作为支持部门，还是后续需要纳入项目角色。
2. 项目经理由谁指定，是研发总监、营销中心、总经理，还是系统管理员在平台中维护。
3. 技术负责人和项目经理可以是同一人，系统是否允许一人多项目角色。
4. 生产制作阶段的比价、采购合同、检验、入库、领料是否全部纳入阶段推进门禁。
5. 预验收和终验收的安装调试记录是否使用同一模板。
6. 评审记录是否需要支持多轮记录，而不是单一资料状态。
7. 技术通知单是否需要作为变更流程单独管理。
8. 日报周报是否必须关联具体资料项/任务，还是第一版只要求关联项目。
9. 文件平台目录结构是否需要按新版产出物重新调整。
