## Why

v20260629 71 项目标模板、workspace card 主操作区、64/64 覆盖核查和兼容资料区过渡已经完成。现在需要规划一个独立 implementation change，将“新项目默认使用 v20260629 71 项模板，旧项目不迁移”的运行边界固化下来。

## What Changes

- 后续 implementation 将新建项目默认初始化模板从当前 20260625 64 项切换到 v20260629 71 项。
- 旧项目继续保持现有 64 项资料记录，不迁移、不补初始化、不改写资料状态、责任人、附件或阶段推进结果。
- `3.3 合同审核记录表（销售合同）` 和 `5.4 采购合同审核记录表` 继续只作为旧项目兼容卡片，不进入 v20260629 71 项模板。
- 非立项阶段默认文件上传或附件上传。
- `1.1 / 1.2 / 1.3` 继续沿用现有在线表单和 `1.2` 专用评价审批。
- 兼容资料区继续保留，不删除、不隐藏。
- 实现前必须固化 v20260629 71 项字段映射清单，明确 documentCode 是否使用 targetOutputCode，并封口 completionMode、submitMode、isRequired。
- 兼容资料区文案必须适配 20260625 旧项目和 v20260629 新项目，不能固定写“当前 64 项资料已迁移”等旧项目专属说明。
- 回滚边界为默认模板版本可切回旧模板，但已创建项目不自动回滚。

不做：

- 不迁移旧项目，不补初始化旧项目。
- 不改数据库 schema，不写 migration。
- 不处理文件平台联动。
- 不新增通用流程引擎、BPM、付款流、发票流或项目模式分支。
- 不删除、不隐藏兼容资料区。

## Capabilities

### New Capabilities

- 无。

### Modified Capabilities

- `stage-document-checklist`: 新项目默认资料模板启用 v20260629 71 项，旧项目继续保持既有 64 项资料。
- `project-core`: 项目创建默认初始化 v20260629 71 项，旧项目不迁移，阶段推进按项目实际资料版本判断。
- `project-core-frontend`: 前端按后端返回的项目资料版本展示 v20260629 新项目和旧项目兼容区，不本地拼接模板。
- `technical-architecture`: 架构边界限定为默认模板版本切换，不改 schema、不写 migration、不处理文件平台、不删除兼容区。

## Impact

- 本轮只新增规划文档和 OpenSpec change artifacts，不修改业务代码。
- 后续 implementation 预计会影响新项目资料初始化入口、模板版本常量、模板字段映射、模板加载/校验逻辑、兼容资料区文案和 smoke 断言。
- 后续 implementation 不应影响数据库 schema、旧项目资料记录、文件平台联动或兼容资料区删除。
