## Why

20260629 自研模式流程图已被归档为后续阶段节点迁移输入，但团队还缺少一份可评审的下一版资料模板候选清单。需要先明确下一版目标资料模板以 20260629 PDF 图面产出为目标基线，并在 67 项图面产出基础上补 4 个流程图漏画草稿产出形成 71 项候选，避免继续以 20260625 64 项运行模板倒推下一版目标。

## What Changes

- 新增 `docs/9.21_20260629资料模板71项确认规划_20260702.md`，输出 64 / 67 / 71 口径和 71 项候选资料清单草案。
- 明确本 change 只确认资料模板候选，不实现任何模板、接口、数据库、前端、权限、状态机或 completionMode 变更。
- 明确下一版目标资料模板以 20260629 图面产出为目标基线，补 4 个草稿产出形成 71 项候选。
- 明确 20260625 64 项只作为当前运行和迁移对照，不作为下一版目标模板来源；当前项目创建和阶段资料初始化仍使用 64 项运行模板。
- 明确 9 个新增候选：`投标书`、`技术协议草稿（合同签订阶段）`、`销售合同草稿`、`技术协议草稿（生产制作阶段）`、`技术协议（生产制作阶段/供应商侧成品）`、`采购合同草稿`、`合格供应商评价表`、`生产记录表`、`资料移交清单`。
- 明确 `合同审核记录表（销售合同）` 和 `采购合同审核记录表` 不在本 change 删除，只作为迁移对照/待确认保留项，不作为 71 口径的核心推导依据。
- 明确 `成本估算表` 是真实多人/多节点协作同一产出的例外；技术协议、销售合同、采购合同的准备/签订应按“草稿 + 成品”拆分评审。
- 明确第一版目标模板默认文件上传/附件上传，不默认新增在线表单；立项阶段 `1.1 / 1.2 / 1.3` 运行期继续沿用已实现在线表单。
- 明确项目工作区迁移目标是将旧资料清单主操作逐步迁移到上方蓝色模块产出卡片，旧资料清单在对应阶段迁移稳定前作为兼容区保留。

### Modified Capabilities

- `stage-document-checklist`: 增加 20260629 图面产出 + 4 个草稿修正形成 71 项目标候选的确认边界，要求新增、删除、合并、改名、阶段移动、必填性和 completionMode 调整必须后续独立实现。
- `project-core`: 明确项目创建和阶段资料初始化仍以 20260625 64 项为运行基线。
- `project-core-frontend`: 明确前端不得因 71 候选直接展示新资料主操作入口，并固化蓝色模块产出卡片为未来主操作入口。
- `technical-architecture`: 明确本 change 不产生 migration、不初始化新资料、不改变状态机、权限、接口或在线表单体系。
