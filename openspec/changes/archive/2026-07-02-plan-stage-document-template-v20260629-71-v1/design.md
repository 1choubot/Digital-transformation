## Context

`review-self-developed-process-flow-20260629-v1` 已经确认：当前系统仍以 20260625 64 项资料模板运行，20260629 PDF 图面红色产出为 67 项，业务修正后逻辑产出候选为 71 项。该归档 change 只完成流程图影响分析，本 change 进一步明确下一版目标资料模板以 20260629 图面产出为目标基线，并把 67 项图面产出 + 4 个漏画草稿产出形成的 71 项候选模板草案写成可评审清单。

## Goals / Non-Goals

**Goals:**

- 固化 64 / 67 / 71 三种数量口径，并明确 20260629 图面产出是下一版目标基线。
- 输出 71 项候选资料清单草案，包含阶段、资料名称、当前 64 项映射、来源节点、新增状态、草稿/成品、建议责任中心、建议 completionMode、在线表单建议、必填性和待确认问题。
- 显式标注新增候选、可能合并或保留的旧项，以及草稿/成品拆分项。
- 明确第一版 71 项目标模板默认文件上传，不默认新增在线表单。
- 明确项目工作区上方蓝色模块产出卡片是未来主操作入口，旧资料清单降级为兼容区。
- 为后续模板实现、completionMode 调整、蓝色节点迁移和验收用例提供输入。

**Non-Goals:**

- 不修改当前 20260625 64 项运行模板。
- 不新增、删除、合并、改名或移动任何运行期资料项。
- 不修改 `completionMode`、权限、状态机、工作台、阶段推进或在线表单规则。
- 不实现后 7 阶段蓝色节点。
- 不改 API、前端代码、数据库或 migration。
- 不处理 `file-platform-integration-v1` 或 `define-digital-platform-v1`。

## Decisions

### Decision 1: 20260629 作为下一版目标产出基线

下一版目标资料模板以 20260629 PDF 图面红色产出为目标基线。71 项候选不是从 20260625 64 项倒推出来，而是在 20260629 图面 67 项基础上补充 4 个流程图漏画草稿产出形成。

### Decision 2: 20260625 只作为当前运行和迁移对照

20260625 64 项继续作为当前系统运行模板和迁移对照。20260625 有但 20260629 没有的产出，默认不进入下一版目标模板，除非业务另行确认保留。当前项目创建、阶段资料初始化、齐套统计、阶段推进门禁、工作台待办和前端入口仍读取现有 64 项项目级资料。

### Decision 3: 第一版 71 项目标模板默认文件上传

71 项目标模板第一版全部默认走文件上传/附件上传，不默认新增在线表单。立项阶段 `1.1 / 1.2 / 1.3` 已实现在线表单，运行期继续沿用；其他阶段资料是否需要在线表单、专用审批或复杂状态机，等按阶段重构时再独立确认。

### Decision 4: 项目工作区上方蓝色模块成为主操作入口

目标 UI 结构是：项目总览 -> 项目工作区 -> 8 阶段 -> 蓝色模块 -> 产出卡片 -> 责任人/上传/提交/审核。当前下方旧资料清单只是兼容区，后续目标是把旧资料清单中的通用能力逐步迁移到上方蓝色模块产出卡片；旧资料清单在对应阶段迁移稳定前不得物理删除。

### Decision 5: 同名资料按阶段和业务语义区分

`技术协议草稿`、`技术协议`、`3D模型`、`安装调试记录` 等同名或近似名称必须按阶段和业务语义区分。后续实现不得只按中文名去重。

### Decision 6: 合同审核旧项只做迁移对照和待确认保留项

`合同审核记录表（销售合同）` 和 `采购合同审核记录表` 可能继续作为独立资料，也可能并入 `销售合同草稿` / `采购合同草稿` 的审核过程。本 change 只记录这个选择点，不删除旧项，不调整运行时数量，不改初始化；它们不作为 71 口径的核心推导依据。

### Decision 7: 草稿/成品拆分优先于伪多对一

技术协议、销售合同和采购合同的准备/签订节点应按“草稿 + 成品”评审。`成本估算表` 是当前唯一保留的真实多人/多节点协作同一产出例外。

## Risks / Trade-offs

- [Risk] 把 71 候选误认为已实现模板，会破坏当前项目初始化和阶段推进。 -> Mitigation: spec 明确项目创建仍使用 64 项，71 只能作为后续确认输入。
- [Risk] 继续以 20260625 64 项为下一版目标来源，会拖住 20260629 工作区重构和目标模板确认。 -> Mitigation: 明确 20260629 图面产出是目标基线，20260625 只作为当前运行和迁移对照。
- [Risk] 过早做在线表单或复杂审批，会让阶段迁移同时牵动模板、状态机、权限和前端流程。 -> Mitigation: 第一版目标模板默认文件上传，在线表单和复杂状态机按阶段另开 change。
- [Risk] 同名资料按中文名去重会合并不同阶段的技术协议或 3D 模型。 -> Mitigation: 候选表必须记录阶段和业务语义。
- [Risk] 新增候选暗示 completionMode 或审批规则变化。 -> Mitigation: 所有 completionMode、权限、状态机和审批/返工变化必须后续独立 change。

## Migration Plan

本 change 不执行系统迁移。后续如确认模板变化，必须另开 implementation change，至少覆盖：

1. 最终资料编号和阶段归属。
2. 模板初始化和项目级资料补初始化策略。
3. completionMode、必填性、责任人、权限、返工和齐套统计。
4. 工作台、项目工作区、上方蓝色模块产出卡片和旧资料清单兼容区展示影响。
5. API smoke、前端 build、OpenSpec validate 和浏览器验收。
