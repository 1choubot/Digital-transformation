- 1.1 项目需求表：责任人分配、在线表单保存/提交、无责任人阻塞、空内容阻塞。

- 1.3 项目立项通知：必须等 1.2 最终通过且 1.1 无关联返工后，营销中心负责人才能提交。
- 旧入口：1.1/1.2/1.3 普通提交、普通完成返工、1.2 普通 confirm/return 均被拒绝。
- 权限：系统管理员、总助、中心负责人、项目经理、总经理、责任人的差异。

## 7. 结论

本轮验证未发现产品缺陷。此前重点疑点的验证结论：

- 1.3 在 1.2 最终通过前不能提交。
- 无资料责任人时不能提交完成。
- 有责任人但无内容时不能提交完成，必填字段校验生效。
- 中心负责人、项目经理、总经理不能越权代提交 1.1/1.2。
- 审核权限和提交权限未混用。
- 有审核权限但没有上传/提交责任时，不能自己制造提交完成。
- 1.2 普通 confirm/return 接口被拒绝，必须走专用节点审批。
- 总经理退回后触发 1.1 返工；返工未清除时阻塞 1.2 重提、1.3、项目编号和阶段推进。

备注：浏览器验证过程中曾出现一次测试服务被前台超时命令终止导致页面网络错误，重启为独立后台进程后最终浏览器验证全部通过；该问题为测试执行环境问题，不计为产品缺陷。
