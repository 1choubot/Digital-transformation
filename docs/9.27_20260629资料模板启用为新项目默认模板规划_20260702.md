# 9.27 20260629 资料模板启用为新项目默认模板规划

日期：2026-07-02

## 1. 背景

此前已完成 v20260629 71 项目标模板 shell 配置、项目工作区产出卡片通用操作迁移、64/64 覆盖核查，以及旧资料清单默认折叠为兼容资料区。下一步可以规划将 v20260629 71 项资料模板启用为新项目默认模板。

本规划只固化实施边界和后续任务拆分，本轮不修改业务代码、不改数据库、不提交、不归档。

## 2. 目标

`enable-v20260629-template-for-new-projects-v1` 的目标是：让后续 implementation 将新建项目默认初始化为 v20260629 71 项资料模板，同时旧项目继续保持既有 20260625 64 项资料记录，不迁移、不补初始化。

关键口径：

- 新建项目默认初始化为 v20260629 71 项资料模板。
- 旧项目继续保持 64 项资料，不迁移、不补初始化、不改写资料状态、责任人或附件。
- `3.3 合同审核记录表（销售合同）`、`5.4 采购合同审核记录表` 只作为旧项目兼容卡片，不进入 71 项目标模板。
- 非立项阶段默认文件上传或附件上传。
- `1.1 / 1.2 / 1.3` 继续沿用现有在线表单与 `1.2` 专用评价审批，不回退为普通上传。
- 兼容资料区继续保留，不删除、不隐藏。
- 实现前必须固化 71 项字段映射清单，至少包括 documentCode、documentName、stageOrder、stageKey、documentOrder、completionMode、submitMode、isRequired，并明确是否使用 targetOutputCode 作为 documentCode。
- 兼容资料区文案必须适配 20260625 旧项目和 v20260629 新项目，不能固定显示“当前 64 项资料已迁移”等旧项目专属说明。

## 3. 非目标

- 不迁移旧项目，不补初始化旧项目。
- 不把 `LC33 / LC54` 或旧合同审核资料加入新项目 71 项模板。
- 不改数据库 schema，不写 migration。
- 不处理文件平台联动，不恢复 file-platform 归档。
- 不新增通用流程引擎、BPM、付款流、发票流或项目模式分支。
- 不删除、不隐藏兼容资料区。

## 4. 实施设计口径

后续 implementation 需要先找出现有新项目初始化模板入口，预计包括：

- 当前 active 模板版本常量。
- 新项目创建事务中的阶段资料初始化逻辑。
- 阶段资料模板加载和校验逻辑。
- smoke 中对新项目资料数量、模板版本和阶段分布的断言。

切换方式应是受控地把新项目默认模板版本从当前 20260625 64 项切换到 v20260629 71 项，而不是迁移数据库中的旧项目资料。

切换前置条件：

- 先产出 71 项字段映射清单，字段至少包含 documentCode、documentName、stageOrder、stageKey、documentOrder、completionMode、submitMode、isRequired。
- 明确 documentCode 是否直接使用 targetOutputCode；若不使用，必须列出稳定业务编号映射。
- completionMode、submitMode、isRequired 不允许在编码时临时决定。

旧项目保持方式：

- 旧项目已经存在的 `project_stage_documents` 记录继续作为事实来源。
- 查询、阶段推进、工作台、项目工作区和兼容资料区按项目已有资料记录和模板版本判断。
- 不对旧项目执行补初始化、重置或资料重编号。

兼容输出边界：

- `LC33 / LC54` 继续只服务旧项目或仍存在 64 项资料的项目工作区兼容展示。
- 新建 v20260629 项目不初始化 `3.3 / 5.4`，也不把它们计入 71 项模板。

兼容资料区文案边界：

- 兼容资料区继续保留，但标题、摘要和提示必须使用模板版本中性表达。
- 不得在 v20260629 新项目中固定显示“当前 64 项资料已迁移”等只适用于旧项目的说明。
- 文案应表达该区域仅用于旧模板兼容查看，不是新项目资料主操作区。

回滚方式：

- 默认模板版本可通过后续代码回滚切回旧模板。
- 已创建项目不自动回滚；已按 v20260629 创建的项目继续保留其 71 项资料记录。
- 若需要迁移或回滚已创建项目，必须通过后续独立 change 定义数据策略和验收。

## 5. 验收口径

后续 implementation 至少需要验证：

- 新建项目生成 v20260629 71 项资料。
- 新建项目不包含 `3.3 / 5.4`。
- 旧项目继续保持 64 项，不迁移、不补初始化。
- 立项 `1.1 / 1.2 / 1.3` 在线表单和 `1.2` 专用评价审批继续可用。
- 非立项资料默认文件上传或附件上传。
- 兼容资料区继续保留，且文案适配新旧模板，不固定显示“当前 64 项资料已迁移”。
- OpenSpec 严格校验、API check 和必要前端 build 通过。

## 6. 后续边界

旧项目迁移、文件平台联动、阶段在线表单化、复杂流程引擎、兼容资料区隐藏或删除，均不在本 change 中处理。
