# Digital Platform Web

数字化管理平台正式前端工程，使用 Vue + Vite。

## Scope

当前已实现项目核心前端和基础登录：

- 登录页：`POST /api/auth/login`
- 退出登录：`POST /api/auth/logout`
- 登录态恢复：`GET /api/auth/me`
- 当前用户状态保存 `organizationRole`、`department`、`role` 和 `isPlatformAdmin`；仅 `system_admin + isPlatformAdmin` 展示“用户管理”入口
- 用户管理页：展示用户列表、新增用户、编辑组织角色、部门、岗位/职务文本、启用/禁用用户和重置密码
- 用户管理表单请求字段使用 `displayName`，展示用户姓名时使用响应中的 `name`
- 用户管理中 `role` 是岗位/职务文本，不是组织角色；总经理、系统管理员、总经理助理不选择部门，中心负责人和员工必须选择四个业务部门之一
- 项目列表页：`GET /api/projects`，必须携带登录态，只展示当前用户可见项目；总经理和总经理助理可查看全局，系统管理员不因系统管理身份自动看到业务项目，项目经理、资料责任人、中心负责人和普通员工按后端项目可见范围展示
- 新建项目页：携带登录态调用 `POST /api/projects`
- 新建项目页选择项目模式、项目经理用户和参与部门，提交 `projectMode`、`projectManagerUserId` 和 `participatingDepartments`。参与部门使用四个业务部门复选框，页面显示中文，接口提交 `operations_center`、`marketing_center`、`manufacturing_center`、`rd_center` 枚举数组，不提交自由文本。
- 项目详情页：`GET /api/projects/:projectId`
- 展示当前登录用户、项目模式、项目经理用户、项目基础信息、当前阶段、8 阶段基础状态和项目创建人追溯字段
- 项目详情页展示阶段资料清单，携带当前登录态调用 `GET /api/projects/:projectId/stage-document-checklist`
- 阶段资料清单按 8 阶段分组展示资料项基础信息、状态和提交/确认/退回追溯字段
- 项目详情页按后端接口返回动态展示 20260610 版 54 项阶段资料，不在前端维护旧 48 项清单、旧资料名或旧编号范围
- 阶段资料清单支持手工标记提交、确认、退回和退回原因输入，操作成功后刷新清单
- 页面明确提示“手工标记状态”不代表已上传文件或已提交在线表单
- 阶段资料清单展示资料项适用/不适用、不适用原因、标记不适用和恢复适用追溯字段
- 阶段资料清单支持手工标记不适用、恢复适用和不适用原因输入，操作成功后刷新清单和齐套摘要
- 项目详情页展示阶段适用必填资料齐套摘要和缺失适用必填资料列表，并明确该摘要基于当前手工状态和人工适用性判断，不代表文件已上传或已归档
- 页面明确提示“不适用”是人工业务判断，用于说明该项目不需要该资料，不代表资料已提交、已确认或已归档
- 项目详情页支持阶段资料项责任人手工分配，调用 `GET /api/users/responsibility-candidates` 获取启用的中心负责人/员工候选列表，并可在权限允许时分配或清空资料责任人
- 阶段资料清单展示当前责任人、未分配状态、已禁用责任人状态和责任人最近一次变更追溯字段
- 页面明确提示“资料责任人”是手工分配，不代表权限控制、个人待办或文件权限
- 项目详情页阶段资料清单展示资料附件区域，支持附件列表、上传、下载和删除
- 页面明确提示“附件上传”只是资料项附件管理，不等于资料已确认，不自动推进阶段，不代表文件管理平台归档
- 我的资料任务页：从主导航进入，登录用户可集中查看分配给自己的适用阶段资料项
- 我的资料任务页支持状态筛选和项目关键字筛选，点击任务可跳转到对应项目详情页
- 项目总览页：从主导航进入，登录用户可查看跨项目汇总指标、当前阶段齐套摘要、未完成适用必填资料和项目详情跳转
- 项目详情页展示当前阶段“手工推进阶段”入口，基于当前用户组织角色/项目经理身份和当前阶段适用必填资料齐套摘要提示能否推进
- 当前阶段未齐套时展示缺失适用必填资料并阻止推进；推进成功后刷新项目详情、8 阶段状态和阶段资料清单
- 第 8 阶段推进完成后展示项目已完成或当前阶段为空的合理状态
- 项目详情页展示只读“业务日志”区域，调用 `GET /api/projects/:projectId/operation-logs` 展示项目最近业务操作日志
- 业务日志区域处理加载中、失败和空日志状态，并展示时间、操作人字段、动作类型和中文摘要；资料责任人变更成功后同步刷新业务日志
- 项目详情页已按区域模块化拆分为项目基础信息、阶段状态、阶段资料清单、资料项操作、责任人、追溯、阶段推进和业务日志等前端组件；拆分仅调整前端结构，不改变业务能力、接口、权限或用户可见行为

不包含日报周报、文件管理平台联动、自动通知、复杂 RBAC、项目成员表、技术负责人表、项目参与人表、完整审批流引擎、项目类型模板、自动批量标记不适用、阶段回退、跳阶段、批量推进、在线表单填写、表单草稿、表单生成归档文件、文件管理平台上传/下载、附件预览、断点续传、多版本管理、病毒扫描、对象存储、全局审计日志、系统配置日志、登录日志、用户管理操作日志、文件平台日志、文件下载日志、管理层大屏图表、日志筛选导出、复杂分页、日志权限矩阵、基于文件上传/归档状态的齐套率、阶段推进操作人完整日志、复杂菜单权限、文件平台用户同步、权限同步、下载权限判断、部门权限继承、SSO、细粒度按钮权限或日志指标。

用户管理入口由当前用户 `organizationRole = system_admin` 且 `isPlatformAdmin = true` 控制，保护范围仅限用户管理页面本身。非系统管理员不展示入口，直接访问用户管理页时展示无权限提示；中心负责人管理本中心员工的用户维护入口本 change 暂不实现。页面展示和编辑 `filePlatformUserId`，但不读取文件管理平台用户表、不校验文件平台用户、不做文件平台权限联动。后端会保证至少保留一个启用的系统管理员且具备平台管理员权限，前端仅展示对应错误提示，不实现复杂权限矩阵。

组织角色固定为总经理、系统管理员、总经理助理、中心负责人、员工；部门固定为运营中心、营销中心、制造中心、研发中心。系统管理员必须同时具备平台管理员开关，但不默认获得业务项目查看、审批、推进或责任人分配权限；总经理助理只读/汇总，不显示资料确认、退回、阶段推进、资料责任人分配、不适用或恢复适用入口。前端隐藏入口不是权限边界，后端会对直接调用返回 `FORBIDDEN_OPERATION`。

项目模式固定为自研模式和供应链/外包模式，两种模式共用同一 8 阶段和 20260610 版 54 项资料。项目经理来自项目经理用户关联 `projectManagerUserId` / `projectManagerUser`；前端不以旧 `projectManager` 文本判断权限。项目经理可以查看自己负责项目全量进度、在权限允许时分配资料责任人并在齐套后推进阶段，但不会仅因项目经理身份显示资料确认或退回入口。中心负责人只在本中心相关项目或资料范围内显示确认、退回、责任人分配、不适用、恢复适用或阶段推进入口。项目经理相关错误码包括 `INVALID_PROJECT_MANAGER_USER_ID`、`PROJECT_MANAGER_USER_NOT_FOUND_OR_DISABLED` 和 `PROJECT_MANAGER_USER_ROLE_NOT_ALLOWED`，页面会展示中文提示。

阶段推进只调用 `POST /api/projects/:projectId/stages/advance` 推进当前阶段，不提供目标阶段选择、跳阶段、回退、批量推进、审批流或权限配置入口。齐套门禁要求当前阶段资料清单已初始化；历史项目尚未补初始化资料清单时不能推进。齐套门禁基于当前手工状态和人工适用性判断，不代表文件已上传或已归档。

业务日志只读展示项目维度关键业务状态变化，包括项目创建、资料提交/确认/退回、资料标记不适用/恢复适用、阶段推进和项目完成。日志来自当前登录用户触发的新成功业务动作，不回填历史日志；前端不提供日志新增、编辑、删除、筛选导出、复杂分页、通知、个人待办或权限配置入口。

资料责任人分配只用于项目详情页阶段资料清单中的人工责任说明。阶段资料清单因返回责任人信息，加载时必须携带当前登录态并受项目可见范围保护；责任人候选列表只返回启用的中心负责人或员工，且必须隶属于四个业务部门之一。页面分配时可在后端允许的权限边界内选择候选用户或清空责任人；中心负责人候选列表会按本中心收窄展示，后端仍执行最终校验。如果已分配责任人后来被禁用，页面仍展示该责任人并提示禁用状态。该能力不创建项目成员表、技术负责人表或项目参与人表，不调用文件管理平台。

阶段资料附件只用于项目详情页资料项内部附件管理。页面调用附件上传、列表、下载和删除接口，附件列表展示文件名、大小、上传人和上传时间，列表按后端 `uploadedAt DESC, id DESC` 排序。四个附件接口都必须携带登录态，并受后端项目可见范围保护；无权用户直接调用会返回 `FORBIDDEN_OPERATION`。上传入口只在适用资料项展示，单文件 50MB 以内，0 字节文件会被拒绝；不适用资料项显示“不能新增附件”提示，但已有未删除附件仍展示，并可下载或删除。上传和删除成功后刷新该资料项附件列表和业务日志；下载不写业务日志。附件操作不改变资料状态、适用性、齐套摘要或阶段推进，不代表资料已确认，不代表文件管理平台归档。第一版不提供文件管理平台同步、在线表单、附件预览、断点续传、多版本管理、病毒扫描、复杂权限、项目成员权限、资料责任人权限或跨项目附件统计入口。

项目详情页模块化拆分后，路由仍为既有项目详情页，仍由页面容器调用 `GET /api/projects/:projectId`、`GET /api/projects/:projectId/stage-document-checklist`、`GET /api/users/responsibility-candidates`、`GET /api/projects/:projectId/operation-logs` 以及既有资料状态、适用性、责任人和阶段推进接口。子组件只通过 props 接收数据、通过事件通知页面容器执行既有操作，不自行扩展后端请求或刷新规则。

20260610 流程对齐后，项目详情页、我的资料任务页和项目总览页均以后端返回的阶段资料项、齐套摘要和缺失必填资料为准动态展示。前端不维护旧 48 项资料模板，不做旧模拟项目资料兼容展示，也不提供模板版本切换入口。

我的资料任务页调用 `GET /api/me/stage-document-tasks`，只展示当前登录用户被分配负责的适用阶段资料项。页面默认展示待办状态，即待提交、已提交和已退回；也可筛选待提交、已提交、已退回、已确认或全部状态。项目关键字筛选在当前任务列表中按项目编号和项目名称过滤，并保留后端返回的排序。任务点击后第一版只跳转到项目详情页，不强制定位到具体资料项。

我的资料任务页展示的是“分配给我的资料项”查询视图，资料状态仍为手工标记状态，不代表文件已上传，也不代表在线表单已填写。该页面不提供文件上传/下载、在线表单、消息提醒、超期提醒、截止日期、批量操作、复杂权限、项目成员权限、文件平台联动或跨项目统计入口。

项目总览页调用 `GET /api/projects/overview-dashboard`，必须携带登录态，并只展示当前用户可见项目。主导航显示“项目总览”入口。页面顶部展示项目总数、进行中项目数、已完成项目数、风险/延期项目数和“我的待办资料”数量；“我的待办资料”指标可点击进入我的资料任务页。该指标是当前登录用户全局待办资料数量，不随项目状态、当前阶段或关键字筛选变化。

项目总览页支持项目状态筛选、当前阶段筛选和关键字筛选，并由后端返回筛选后的汇总指标和项目列表。项目列表展示项目编号、项目名称、客户、项目模式、项目经理用户、项目状态、当前阶段、当前阶段齐套率、未完成适用必填资料数量和计划时间；当前阶段异常时展示缺失当前阶段、多个当前阶段或资料清单未初始化等提示。点击项目后跳转项目详情页。

项目总览页中的齐套率基于当前手工状态和人工适用性判断，不代表文件已上传，也不代表在线表单已填写。第一版项目总览是只读页面，不提供文件上传/下载、在线表单、消息提醒、超期提醒、截止日期、大屏图表、分页、导出、批量操作、复杂权限或项目成员权限入口。

## API

开发期默认使用 Vite proxy，将 `/api` 转发到 `http://127.0.0.1:3001`。

如需指定后端地址，复制 `.env.example` 为 `.env` 后配置：

```bash
VITE_API_BASE_URL=http://127.0.0.1:3001
```

基础登录需要先在后端执行：

```bash
npm run init-auth
```

默认初始化账号为 `admin`，默认密码为 `Admin@123456`，可通过后端 `.env` 中的 `INITIAL_USER_*` 配置覆盖。

`digital-demo` 只作为布局和样式参考，本工程不使用 Demo 静态项目数据。
