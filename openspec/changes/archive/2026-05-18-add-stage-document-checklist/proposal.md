## Why

项目已经具备主数据、8 阶段和基础登录能力，但后续在线表单、文件上传、资料齐套率和阶段推进都需要先有项目级阶段资料清单作为基础数据。当前变更先把阶段资料项模板和项目级清单初始化落到规格层，避免后续能力继续依赖静态说明或人工口径。

本变更只实现资料清单基础能力和只读展示，不实现提交、上传、确认、退回、齐套率或阶段推进。

## What Changes

- 新增阶段资料项模板能力，模板来源参考 `docs/9.1_8阶段流程与阶段定义表.md` 和 `docs/9.2_阶段资料清单与责任角色表.md`。
- 模板字段包括阶段、资料项名称、是否必填、默认责任部门或责任角色、提交方式、文件管理平台目标文件夹路径；第一版 `targetFolderId` 为空，待文件平台真实联动时再回填目录 ID。
- 新增项目级阶段资料清单初始化能力：项目创建后自动按模板生成该项目的资料项清单。
- 兼容历史项目：已有项目允许通过后端脚本或命令补初始化资料清单；第一版不做前端按钮，也不开放给普通用户的补初始化接口。
- 新增查询接口，按项目查询阶段资料清单，并按阶段分组返回资料项。
- 新增资料项基础状态字段，第一版系统枚举为 `not_submitted`、`submitted`、`confirmed`、`returned`；前端分别展示“待提交”、“已提交”、“已确认”、“已退回”，初始化默认 `not_submitted` 并展示为“待提交”。
- 新增为后续文件上传、在线表单、资料齐套率和阶段推进预留的字段，但不实现这些后续能力。
- 前端项目详情页增加“阶段资料清单”区域，只读展示资料项基础信息和状态，不提供上传、填写、提交、确认、退回、补初始化或阶段推进按钮。
- 模板数据必须完全来自 `docs/9.2_阶段资料清单与责任角色表.md`；如果实现时无法可靠解析文档，必须暂停并说明，不得自行编造资料项。
- 明确不实现文件上传、文件下载、文件管理平台真实联动、在线表单填写、表单生成归档文件、资料齐套率计算、阶段推进、退回/确认操作、复杂权限或业务日志。

## Capabilities

### New Capabilities

- `stage-document-checklist`: 覆盖阶段资料项模板、项目级阶段资料清单初始化、历史项目补初始化、资料清单查询和阶段分组返回。

### Modified Capabilities

- `project-core`: 项目创建成功后除初始化 8 阶段外，还需要初始化项目级阶段资料清单。
- `project-core-frontend`: 项目详情页需要展示阶段资料清单基础信息和状态。

## Impact

- 后续实现会影响 `digital-platform-api` 的 MySQL 表结构、项目创建流程和新增资料清单查询接口。
- 后续实现会影响 `digital-platform-web` 的项目详情页和项目 API client。
- 数字化平台第一版只保存文件管理平台目标路径，`targetFolderId` 保持为空；本变更不得调用文件管理平台 API，不得上传/下载文件，也不得判断文件平台权限。
- 本变更不得引入在线表单填写、文件归档、资料齐套率计算、阶段推进、退回/确认操作、复杂权限或业务日志。
