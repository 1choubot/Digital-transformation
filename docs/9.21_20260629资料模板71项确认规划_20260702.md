## 3. 草稿/成品拆分原则

真实多节点协作同一产出只保留 `成本估算表`。研发、制造、运营和总经理可以围绕同一张表协作补充成本、价格、税费、利润和审批意见。

技术协议、销售合同、采购合同的准备/签订不得继续按多对一建模。后续应按以下原则评审：

- 准备节点优先对应草稿产出。
- 签订节点优先对应成品产出。
- 草稿产出承载拟制、内部审核、财务法务审查、总经理批准等过程。
- 成品产出承载客户或供应商签署后的正式文件。
- 同名资料必须按阶段和业务语义区分，不能只按中文名去重。

## 4. 第一版目标模板默认文件上传

71 项目标模板第一版全部默认走文件上传/附件上传，不在本轮默认新增在线表单。

- 立项阶段 `1.1 / 1.2 / 1.3` 已实现在线表单，运行期继续沿用现有在线表单、责任人、评价审批和返工规则。
- 后续新增阶段资料默认先按文件上传、附件上传、通用提交/审核能力承载。
- 哪些资料需要在线表单、专用审批或复杂状态机，等按阶段重构时再由独立 change 确认。
- `成本估算表`、草稿技术协议、草稿销售合同、草稿采购合同等候选项不因本规划被定义为当前必须在线表单。

## 5. 工作区迁移基线

目标 UI 结构是：项目总览 -> 项目工作区 -> 8 阶段 -> 蓝色模块 -> 产出卡片 -> 责任人/上传/提交/审核。

- 当前下方旧资料清单只是兼容区。
- 后续目标是把旧资料清单里的通用能力逐步迁移到上方蓝色模块产出卡片。
- 迁移顺序按阶段推进，不一次性实现所有在线表单或复杂业务流。
- 旧资料清单在对应阶段迁移稳定前不得物理删除。
