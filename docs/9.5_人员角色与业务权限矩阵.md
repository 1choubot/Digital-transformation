## 2. 权限设计原则

- 数字化管理平台控制业务权限：建项目、维护项目、分配资料责任人、填表、上传、确认、退回、推进阶段、查看看板。
- 文件管理平台控制文件权限：文件查看、下载、文件夹访问、文件日志。
- 第一版不做复杂跨系统权限同步，只保留数字化平台用户与文件平台用户的映射字段。
- 数字化平台可以展示资料状态，但不能因为业务可见而自动授予文件下载权限。
- 文件下载或查看必须走文件管理平台权限判断。

## 3. 第一版用户字段

| 字段 | 建议字段名 | 类型 | 必填 | 说明 |
|---|---|---|---|---|
| 用户 ID | `id` | number/string | 是 | 数字化平台用户主键 |
| 登录账号 | `username` | string | 是 | 登录名 |
| 显示姓名 | `display_name` | string | 是 | 页面展示姓名 |
| 所属部门 | `department_name` | string | 是 | 如营销中心、研发中心、制造中心 |
| 平台角色 | `platform_role` | enum | 是 | 管理层、项目经理、部门负责人、普通成员、系统管理员 |
| 文件平台用户 ID | `file_platform_user_id` | number/string | 否 | 用于下载或追溯时映射文件平台用户 |
| 是否启用 | `is_active` | boolean | 是 | 禁用后不能执行新业务操作 |
| 创建时间 | `created_at` | datetime | 是 | 用户创建时间 |
| 更新时间 | `updated_at` | datetime | 否 | 用户更新信息时间 |

## 4. 第一版角色定义

| 角色 | 角色定位 | 典型人员 | 数据范围 |
|---|---|---|---|
| 管理层 | 查看项目全局进度、风险、暂停、延期和资料齐套情况 | 总经理、分管领导 | 默认全部项目业务状态；文件下载仍按文件平台权限 |
| 项目经理 | 项目全过程主责，负责建项目、维护信息、分配资料、确认资料、推进阶段 | 项目经理、项目负责人 | 自己负责或参与的项目；第一版可配置为全部智能制造项目 |
| 部门负责人 | 负责本部门相关资料确认和项目状态查看 | 研发负责人、制造负责人、营销负责人 | 本部门相关项目或本部门资料项 |
| 普通成员 | 完成个人待办、填写表单、上传资料、查看提交状态 | 机械、电气、软件、制造、营销、商务成员 | 分配给自己的资料项；允许查看的项目资料状态 |
| 系统管理员 | 维护数字化平台基础配置，不参与业务推进 | 平台管理员 | 用户、角色、阶段模板、资料模板等配置 |

## 6. 文件权限分离矩阵

| 场景 | 数字化平台判断 | 文件管理平台判断 | 第一版结论 |
|---|---|---|---|
| 用户能否看到项目状态 | 判断业务角色和项目范围 | 不参与 | 数字化平台决定 |
| 用户能否看到资料是否已提交 | 判断业务角色和项目/资料范围 | 可提供文件状态 | 数字化平台决定展示 |
| 用户能否查看文件名 | 可展示资料关联文件名 | 可提供文件列表 | 第一版可展示文件名和状态 |
| 用户能否下载文件 | 只发起下载请求 | 判断文件权限 | 文件管理平台最终决定 |
| 下载失败如何处理 | 展示失败并记录业务侧请求 | 记录文件侧拒绝或失败 | 不绕过文件平台 |
| 用户是否可上传归档文件 | 判断业务权限和责任人 | 判断目标目录/服务账号写入权限 | 两边都需通过 |

## 7. 普通成员进入平台后的可见内容

普通成员第一版不建议进入复杂项目管理视图，主要聚焦待办和提交状态。

| 模块 | 普通成员可见内容 |
|---|---|
| 我的待办 | 分配给自己的资料项、退回资料、待补充表单 |
| 我的提交 | 已保存草稿、已提交、已归档、已退回、已确认资料 |
| 项目基础信息 | 与自己资料相关项目的项目编号、项目名称、客户、当前阶段 |
| 阶段资料 | 自己负责的资料项；可按权限查看阶段其他资料状态 |
| 文件下载 | 只能发起下载请求，是否成功由文件管理平台判断 |

## 8. 业务日志权限

| 日志类型 | 记录内容 | 默认查看角色 |
|---|---|---|
| 项目日志 | 创建项目、修改项目、暂停、恢复、标记风险 | 管理层、项目经理 |
| 阶段日志 | 阶段推进、推进时资料齐套状态 | 管理层、项目经理 |
| 资料日志 | 分配责任人、保存草稿、提交、确认、退回 | 项目经理、部门负责人、资料责任人可看本人相关 |
| 归档日志 | 数字化平台请求文件平台归档的结果 | 项目经理、系统管理员 |
| 下载请求日志 | 从数字化平台发起的下载请求 | 项目经理、管理层；文件侧结果以文件平台为准 |
| 配置日志 | 用户、角色、阶段模板、表单模板变更 | 系统管理员 |

## 9. 与文件管理平台角色的映射建议

文件管理平台当前角色包括系统管理员、总经理、中心负责人、员工。数字化平台第一版不强制一一等同，但建议建立映射关系。

| 数字化平台角色 | 可映射的文件平台角色 | 说明 |
|---|---|---|
| 管理层 | 总经理 | 下载权限仍由文件平台总经理权限或显式权限判断 |
| 项目经理 | 中心负责人/员工 | 视项目经理在文件平台中的账号类型而定 |
| 部门负责人 | 中心负责人 | 可对应文件平台中心负责人 |
| 普通成员 | 员工 | 可对应文件平台员工 |
| 系统管理员 | 系统管理员 | 数字化系统管理员不一定进入文件域 |

## 10. 开发落地提示

- 业务权限建议先做角色级控制，后续再扩展到更细粒度的项目成员、部门、资料项权限。
- 所有关键操作都需要服务端校验，前端按钮隐藏只能作为体验优化。
- 禁用用户不能执行新的业务操作，但历史业务日志必须保留。
- 下载文件时必须带上数字化平台真实操作人信息，便于和文件平台日志关联。
- 第一版不要把文件平台权限复制到数字化平台，避免两套权限不一致。
